## 좋아요 버튼 클릭 → 모달 지연 원인 분석

좋아요 버튼 클릭 후 모달이 늦게 뜨는 원인은 **3단계 순차 지연**이 누적되기 때문입니다.

### 병목 흐름

```
클릭
 ↓
① hasSpotRecommendations API 응답 대기 (network-only) ⚠️ ~200-500ms
 ↓
② 드로어 컴포넌트 dynamic import 로드 ⚠️ ~100-300ms
 ↓
③ 드로어 내부 추천 스팟 Suspense 쿼리 실행 ⚠️ ~200-500ms
 ↓
모달/드로어 표시
```

### 주요 원인 3가지

**1. API 응답 후에야 드로어가 열림** (`useSpotRecommendDrawer.ts`)
```typescript
const hasRecommendations = await hasSpotRecommendations(spotCode); // 여기서 blocking
if (hasRecommendations) {
  openDrawer(); // API 끝나야 열림
}
```
추천 스팟이 있는지 먼저 확인하고 나서야 드로어를 여는 구조입니다.

**2. `network-only` 캐시 정책** (`useHasSpotRecommendations.ts`)
- 캐시를 쓰지 않고 항상 네트워크 요청 → 매번 대기 발생

**3. 드로어 컴포넌트 동적 로드** (`GeneralSpotDetail.tsx` 등)
```typescript
const DynamicSpotDetailSpotRecommendationDrawer = dynamic(

```
