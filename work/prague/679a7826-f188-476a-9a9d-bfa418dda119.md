**Assistant**: 아뇨, 정확히는 **만료 여부가 아니라 발급 기간(issuancePeriod)에 의해 자동 관리**되는 거예요.

### OnlineCoupon.isActive 동작 방식

- **별도 상태 enum은 없고**, `isActive` boolean 하나만 있음
- **스케줄러가 자동으로 토글**:
  - `issuancePeriod.start` 도달 → `isActive = true` (발급 가능)
  - `issuancePeriod.end` 경과 → `isActive = false` (발급 중단)
- **수동으로도 비활성화 가능**: `unsetActiveStatus()` mutation (단, 활성화하는 mutation은 없음)

### 구분

| | 대상 | 상태 관리 |
|---|---|---|
| **OnlineCoupon.isActive** | 쿠폰 자체 | 발급 기간 기반 스케줄러 + 수동 비활성화 |
| **IssuedOnlineCoupon.status** | 유저에게 발급된 쿠폰 | PENDING → NOT_USED → USED/EXPIRED/WITHDRAWN |

그래서 `isActive`는 "이 쿠폰을 지금 발급할 수 있느냐"를 뜻하고, 발급된 쿠폰의 만료는 `IssuedOnlineCoupon`의 `availablePeriod.end` 기준으로 별도 스케줄러가 `EXPIRED` 처리합니다.
