## 인사이트 D2 (NEW). "예약내역" 사이드바가 대시보드를 갉아먹는다

283/391 = 72.4% 세션이 대시보드에 들어와서 **사이드바 "예약내역"만 한 번 클릭하고 떠난다**. 클릭은 307건이며, "예약내역" 자체가 대시보드에서 가장 많이 눌리는 라벨.

문제 구조:
- 대시보드는 **승인대기 테이블**이 페이지 내에 있는데도 사용자는 사이드바 "예약내역"으로 이동.
- 가설 1: 승인대기 테이블의 가시성/스크롤 위치가 낮아 발견되지 않음.
- 가설 2: 사이드바 "예약내역"이 익숙해서 습관적 클릭(인지 비용 0).
- 가설 3: "예약내역" 페이지에 있는 필터/검색·페이지네이션 기능이 대시보드 테이블엔 없어 결국 떠남.

**Why it matters**: D1과 합쳐 보면, 매장 운영자가 대시보드에 도착하지만 **대시보드 안에서 작업을 끝낼 수단이 부족**해 결국 `/reservation`으로 이동한다. 위젯을 더 추가하기보다 **사이드바 "예약내역"과 동등한 수준의 필터/검색을 대시보드 내 테이블에 부여**하면 떠나는 비율을 줄일 수 있다.

## 인사이트 D3 (NEW). 대시보드 내 예약 상세 모달은 작동하나 노출이 부족하다

테이블 행 클릭 → 예약 상세 모달 열기 = 23세션 (5.9%). 그중 본문(div) 클릭으로 모달 내 컨트롤을 살피는 세션 7세션. 인라인 승인(승인/확정하기 등)까지 가는 세션 8세션.

성공 사례: 뉴욕연세치과(9 대시보드 세션 / 7 테이블 행 클릭 / 4 스크롤 / 1 인라인 승인) — 대시보드를 "처리 도구"로 활용하는 유일한 매장 운영자.

**Why it matters**: 모달 + 인라인 승인 동선은 **잘 만들었지만 발견되지 않는다**. (a) "처리하러 가기 →" 버튼의 시인성 강화, (b) 대시보드 헤더에 "오늘 X건 처리됨 / X건 남음" 진행률 노출, (c) 행 hover 시 액션 버튼(승인/거절) 직접 노출이 후보.

## 한 줄 결론

**대시보드는 잘 만들어졌지만 외부 파트너의 핵심 워크플로(알림 → 직접 진입 → 빠른 처리 → 종료)에 닿지 못한다.** 헤비 처리자는 우회하고, 매장 운영자는 진입하지만 사이드바 "예약내역"으로 이탈한다. 추가 위젯이 아니라 **대시보드 내 테이블의 필터·검색 강화 + 행 단위 직접 액션**이 대시보드 효용성을 끌어올리는 가장 가성비 좋은 개선이다.
