## 왜 안 걸리는가

Standard analyzer 기준으로 "Ultherapy Prime 300 shots"는 다음과 같이 토큰화됩니다:

```
["ultherapy", "prime", "300", "shots"]
```

검색어 "Ulthera"도 토큰화되면 `["ulthera"]`. 그런데 인덱스에는 `"ulthera"` 토큰이 없고 `"ultherapy"`만 있음 → **완전 일치가 아니라서 매칭 실패**.

즉 부분 문자열(prefix, substring) 매칭은 **기본 동작이 아님**. 별도 설정 필수입니다.

## 걸리게 하려면 네 가지 길 중 하나

| 방법 | 동작 | 비용 |
|---|---|---|
| **Edge n-gram analyzer** | 인덱싱 시 "ultherapy"를 `["u","ul","ult","ulth","ulthe","ultheR","ultherap","ultherapy"]`로 저장 → 접두사 검색 모두 매칭 | 인덱스 크기 증가, 타 필드에도 영향 검토 필요 |
| **match_phrase_prefix 쿼리** | 쿼리 시점에 접두사 매칭 | 쿼리 성능 약간 저하, 긴 접두사에 불리 |
| **Fuzzy matching** | 편집 거리(edit distance)로 유사 토큰 허용 ("ulthera" ≈ "ultherapy", 거리 2) | 오매칭 증가 가능 |
| **동의어 사전** | `ulthera → ultherapy`, `울쎄라 → ultherapy` 수동 매핑 | 유지보수 필요, 정확도 높음 |

|---|---|---|
| **A. 그대로 두기** | 진행 중 요청은 옛 스냅샷대로 처리. 신규 요청부터 새 흐름 적용 | 구현 단순, 위험 0. 단 검수 화면에서 옛/새 구조 분기 처리 필요 |
