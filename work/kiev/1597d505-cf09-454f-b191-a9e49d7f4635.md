### 설계 요약

#### 백엔드: 서비스 계층 공유

현재 `sendReservationDateChangeRequestByAdmin` 내부 로직을 보면 이미 **인증 무관한 공용 로직**과 **auth 분기**가 섞여 있습니다. 이를 분리하면:

```
[공용 로직 — 이미 존재, 재사용 가능]
├─ validationService.checkBaseConditionForSendRequest()   ← 예약 상태/날짜 검증
├─ validationService.validateDateChangeLimit()            ← 180일 범위 검증
├─ reservationMailService.sendMailToRequestChangeReservationDate()  ← 메일 발송
└─ ReservationDateChangeRequestHistory.createDomainForAdmin()       ← 이력 생성

[Auth 분기 — Usecase별로]
├─ Admin Usecase:  @CurrentUser() member → writerCode
└─ Token Usecase:  actionToken 검증 → 예약에서 partnerId 추출 → writerCode
```
