DBA MCP Server
Creatrip 내부 Aurora MySQL 데이터베이스에 대한 읽기 전용 MCP(Model Context Protocol) 서버.

Claude Code에서 DB 스키마 조회 및 SELECT 쿼리 실행을 지원합니다.

제공 도구
Tool	설명
list_tables	데이터베이스의 모든 테이블 목록 조회
describe_table	테이블의 컬럼 정의 및 인덱스 정보 조회
show_create_table	테이블의 CREATE TABLE DDL 문 조회
select	SELECT 쿼리 실행 (EXPLAIN/EXPLAIN ANALYZE 지원)
환경
모든 도구는 env 파라미터로 대상 환경을 지정합니다:

dev: 개발 DB
prod: 운영 DB
select 도구 옵션
옵션	타입	기본값	설명
explain	enum	none	none: 쿼리 실행, explain: 실행 계획만, explain_analyze: 실제 통계 포함
truncate	boolean	true	긴 문자열 필드 자르기. true: 500자, false: 10,000자 하드리밋
응답 포맷 (TOON)
select와 describe_table 도구는 TOON(Token-Oriented Object Notation) 포맷으로 응답합니다.

장점:

JSON 대비 ~60% 토큰 절감
LLM 정확도 향상 (73.9% vs JSON 69.7%)
예시:

[3]{id,name,email}:
  1,Alice,alice@example.com
  2,Bob,bob@example.com
  3,Charlie,charlie@example.com
문자열 Truncate
긴 텍스트 필드(HTML, JSON 등)는 자동으로 잘려서 토큰을 절감합니다.

content: "<p>앞부분 내용...</p>...[TRUNCATED: 8234 chars]"
truncate=true (기본): 500자 제한
truncate=false: 10,000자 하드리밋
보안
1. JWT 인증
Authorization: Bearer <JWT>
알고리즘: HS256 (명시적 지정)
시크릿: 기존 Creatrip 서비스와 동일한 JWT_SECRET 사용
필수 검증:
서명 유효성
만료 시간 (exp)
이메일 도메인 (@creatrip.com만 허용)
2. Member 검증
JWT 검증 후 추가로 prod DB에서 실제 멤버 확인:

member 테이블에서 code로 조회
JWT의 이메일과 DB의 이메일 일치 확인
level >= 90 검증 (관리자 권한)
3. SQL 검증
검증 항목	설명
SELECT만 허용	INSERT, UPDATE, DELETE, DROP 등 차단
단일 문장만 허용	세미콜론으로 구분된 다중 쿼리 차단
LIMIT 강제	최대 100행 (미지정 시 자동 추가, 초과 시 축소)
위험 함수 차단	sleep, benchmark, get_lock, release_lock, load_file
파일 접근 차단	INTO OUTFILE, INTO DUMPFILE 패턴
크로스 스키마 차단	다른 데이터베이스 접근 불가 (파싱 실패 시 거부)
4. 테이블명 검증
show_create_table 도구에서 SQL injection 방지:

유효한 테이블명: /^[a-zA-Z_][a-zA-Z0-9_]*$/
백틱, 세미콜론, 주석 등 특수문자 포함 시 거부.

5. 쿼리 타임아웃
세션 레벨 max_execution_time = 5000ms
DB 서버가 직접 쿼리 중단
6. 인프라 보안
VPC 내부 실행: Lambda가 private subnet에서 실행
동시성 제한: ReservedConcurrentExecutions = 5
DB 연결: 환경별 읽기 전용 계정 사용 권장
로깅
AWS Lambda Powertools Logger를 사용한 구조화된 JSON 로그.

로그 필드
필드	설명
member_code	인증된 사용자 코드
email	인증된 사용자 이메일
env	대상 환경 (dev/prod)
sql	실행된 SQL (select 도구)
duration_ms	쿼리 실행 시간
row_count	반환된 행 수
로그 종류
auth_success / auth_failed: 인증 결과
query_executed / query_failed: 쿼리 실행 결과
list_tables, describe_table, show_create_table: 메타데이터 조회
show_create_table_invalid_name: 잘못된 테이블명 시도
Claude Code 설정
참고: 이 MCP는 개인별 인증 토큰이 헤더에 포함되어야 하므로, 각 사용자의 로컬 환경에 설정해야 합니다.

기존 설치 제거 (해당 시)
이전에 설치한 적이 있다면 먼저 제거합니다:

claude mcp remove dba-mcp
Step 1. Authorization 토큰 획득
크리에이트립 운영환경에 로그인
브라우저 개발자 도구 열기 (F12)
Network 탭에서 아무 API 요청 선택
Request Headers에서 Authorization: Bearer eyJ... 값 복사
Step 2. 장기 토큰 발급 (1년)
복사한 토큰으로 1년짜리 장기 토큰을 발급받습니다. 브라우저 주소창에 직접 입력하거나 curl을 사용합니다:

https://xcetliunbjxu3vfakhaigroq3a0pfvgg.lambda-url.ap-northeast-2.on.aws/token/extend?token=<복사한_토큰>
token= 뒤에는 Bearer 를 제외한 토큰 값만 입력합니다. (예: eyJhbGci...)

응답:

{"token": "eyJhbGci...(1년 유효 토큰)"}
응답의 token 값을 복사해둡니다.

Step 3. MCP 서버 등록
발급받은 장기 토큰으로 MCP를 등록합니다:

claude mcp add -s user --transport http dba-mcp \
  "https://xcetliunbjxu3vfakhaigroq3a0pfvgg.lambda-url.ap-northeast-2.on.aws/" \
  --header "Authorization: Bearer <발급받은_장기_토큰>"
-s user 옵션을 사용하면 모든 프로젝트에서 사용할 수 있습니다. 특정 프로젝트에서만 사용하려면 -s local로 지정하세요.

Step 4. 설정 확인
Claude Code 세션에서 MCP가 활성화되었는지 확인합니다:

/mcp
목록에 dba-mcp가 enabled 상태로 표시되면 설정 완료입니다.

사용 예시
Claude Code에서 자연어로 요청:

dev DB에서 member 테이블 구조 보여줘
prod에서 최근 가입한 회원 10명 조회해줘
user 테이블의 인덱스 확인하고 SELECT 쿼리 실행 계획 분석해줘
토큰 갱신
JWT 토큰이 만료되면 위 과정을 다시 진행합니다:

claude mcp remove dba-mcp
