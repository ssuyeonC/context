## 0. 한 문장 요약

> 파트너가 어드민에서 셀프로 처리할 수 있는 업무 범위를 확장하여, 제휴팀의 반복 소통 부담을 구조적으로 줄인다. 카톡은 대체하지 않고, 자연스럽게 빈도를 낮춘다.
> 

---

## 1. 배경 & 목적

### 1-1. 현황

- 제휴팀과 파트너 간 소통 및 요청을 처리하는 업무가 운영 리소스의 큰 비중을 차지함
    - 4/20 운영팀 인터뷰 결과, 이 소통의 상당 부분이 **파트너 어드민으로 흡수 가능한 반복 업무**(가격·운영시간·주의사항·이벤트 등 운영 변경 사항)임이 확인됨
- 어드민 활용도를 높여 제휴팀의 반복 소통 부담을 구조적으로 감소시키는 것이 본 방향성의 목적

### 1-2. 인터뷰로 본 현황 수치

| 항목 | 수치 |
| --- | --- |
| 제휴팀 담당 파트너 | 혜원 ~250개 / 지윤 60개·80스팟 / 성민 **500개·1000+스팟** |
| 주 소통 채널 | **3명 모두 카톡 압도적** (슬랙·잔디는 극히 미미함) |
| 수정 요청 처리 리소스 | 건마다 다르나, n주 동안 잡고 있어야 하는 경우 존재 |
| 운영팀 수동 예약 확정 | 52.4% (11,833건 / 3개월), 하루 평균 130건+ |
| Type D 미사용 파트너 | 11개 스팟 / 1,689건이 100% 운영 의존 |

## 2. 현황 — 제휴팀 ↔ 파트너 소통의 실제 구조

### 2-1. 제휴팀의 실제 부담

| 업무 유형 | 주 채널 | 제휴팀 체감 비중 | 비고 |
| --- | --- | --- | --- |
| 스팟/블로그 내용 수정 요청 | 카톡 | **최대** | 가격·운영시간·주의사항·이벤트 |
| 리뷰 컴플레인 | 카톡 | 중 |  |
| 복잡한 예약/클레임 | 카톡 | 저 | 일반 예약은 CX팀 이관 완료 |

**핵심 인사이트**

- 예약 처리는 이미 CX로 이관되어 제휴팀 직접 부담은 상대적으로 낮음
- 제휴팀이 실감하는 부담은 "운영 변경 사항의 반영 소통"에 집중됨

### 2-2. 파트너 어드민의 현재 커버리지

| 기능 영역 | 어드민 지원 | 파트너 실제 활용 |
| --- | --- | --- |
| 예약 처리 (승인/변경) | O | 19.2% (어드민+알림톡 합산) |
| 스팟 정보 수정 요청 | O (요청→관리자 승인) | 저조 (카톡 우회 多) |
| 운영시간/가격 변경 요청 | O (요청→관리자 승인) | 저조 |
| 블로그 수정 | **X** | 제휴팀+인턴이 대행 |
| 리뷰 답글 | 일부 (권한 제약) | 제한적 |

→ 기능은 있으나 **파트너 인지도·활용도 낮음**

*c.f., 혜원님: "알림톡 버튼 누르고서야 어드민 존재를 알게 됨."*

## 3. 개선 방향

### 3-1. 예약 처리 UX 개선

어드민 내 예약 기능에 대한 접근성·편의성을 강화하여 파트너 직접 처리율을 높인다.

- **3-1-A. 파트너 대시보드 신설**
    - 어드민 인덱스 페이지를 대시보드로 재구성
    - 승인 대기, 오늘 방문 예정, 이번 달 확정/사용완료 카드 + 일괄 승인 테이블
    - 파트너 로그인 시 기본 랜딩 페이지
    - 반영 진행 중
- **3-1-B. 알림톡 기능 편의성 강화**
    - '예약 변경 요청' 흐름을 토큰 기반 무로그인 진입으로 전환 (현재 '예약 승인'과 동일 방식)
    - 유저에게 발송되는 이메일 일원화 & 예약 변경 요청 시간 선택 등 UX 개선
- **3-1-C. 알림톡 제공 정보 강화**
    - 현재 유저 예약 내역을 제한적으로만 표시 → 파트너의 즉각 판단에 필요한 정보까지 확대

---

### 3-2. 스팟/블로그 수정 UX 개선

제휴팀 최대 부담인 "수정 요청 소통"을 파트너 셀프로 전환한다.

- **3-2-A. 어드민 알림 기능**
    - 파트너의 수정 요청 시, 담당 관리자에게 어드민 내 알림 + 슬랙 알림
    - 비동기 코멘트 스레드 포함 (경량 — 채팅 UI 아님, 액션 히스토리 기반)
    - Why: 현재는 파트너가 요청해도 관리자가 인지할 수단이 없어 처리 지연 → 파트너가 카톡으로 우회 → 파트너 어드민 신뢰도 하락의 악순환
- **3-2-B. 파트너의 블로그 수정 기능**
    - 스팟과 연결된 블로그를 파트너가 템플릿 제한 내에서만 수정 가능하도록 개편
    - 워크플로우 확립: 파트너 수정 → 제휴팀 확인 → (필요 시) 콘텐츠팀 검수 → AI 번역 → 반영
    - 선행 작업: 블로그 구조 파악을 통한 반영 가능 여부 판단 / 수정 허용 범위를 정의하는 템플릿 정형화 / 컨텐츠팀 협의
- **3-2-C. 파트너 가격 변경 기능 강화**
    - 스팟마다 수수료율을 저장, 파트너가 원가/판매가 변경 시 자동 계산하여 반영
    - 조건부 로직 필수: 정산타입이 "정산가 정산"인 경우 수수료 미입력/미적용 처리 (성민 제안)
    - 현재 이슈: 수수료 입력란은 있으나 작성되지 않는 케이스 존재, 정산타입별 분기 부재, 세금계산서 미발행 예외 1건
    - 선행 조사 필요: 정산타입 전체 케이스 매핑, 수수료율 필수값 여부 재정의

---

### 3-3. 어드민 UI 개선 (전반)

파트너가 어드민을 열었을 때의 인지적 부담을 낮춘다.

- **3-3-A. 파트너에게 불필요한 컴포넌트 제거**
    - 어드민 공용 페이지에서 파트너 권한으로 보이는 불필요 UI 정리
    - 네비게이션 재구성 (파트너 전용 메뉴 트리 분리 — 파트너 대시보드와 연계)
- **3-3-B. 고령 파트너 친화 UI**
    - 글자 크기, 버튼 가시성, 핵심 액션 단순화, 에러 메시지 명료화
    - 대상 근거: 성민 담당 500개 파트너 풀(체험/식당/의상대여/한의원 등)은 고령 담당자 다수 + 블로그-스팟 복잡도가 결합된 영역. 이 풀이 어드민 접근이 실질적으로 어려운 상태.
    - Why (전환 전략과의 관계): 지윤 인사이트 "젊은 담당자에서 전환 성공"(메디큐브·뷰티블라썸)은 "쉬운 파트너의 조건"을 보여준다. 본 개선은 그 전환 가능 풀을 고령 다수 파트너까지 확장하는 축.
