사용자가 회의록 파일에 대해 추가 논의를 요청합니다.

입력 형식: `{회의록 파일명}, {추가 논의 내용}`
인자: $ARGUMENTS

## 실행 절차

1. `$ARGUMENTS`에서 첫 번째 콤마(,)를 기준으로 파일명과 추가 논의 내용을 분리합니다.
   - 콤마 앞: 회의록 파일명 (앞뒤 공백 제거)
   - 콤마 뒤: 추가 논의할 내용 (앞뒤 공백 제거)

2. `.context/outputs/meetings/` 폴더에서 해당 파일을 찾습니다.
   - 정확한 파일명 매칭을 먼저 시도하고, 없으면 부분 매칭으로 찾습니다.
   - 파일을 찾을 수 없으면 `.context/outputs/meetings/` 폴더의 파일 목록을 보여주고 사용자에게 확인합니다.

3. 회의록 파일 전체를 읽고 내용을 파악합니다.

4. 사용자가 제시한 추가 논의 내용을 바탕으로, 회의록의 맥락(참석자, 논의 주제, 기존 결론 등)을 고려하여 추가 논의를 진행합니다:
   - 기존 회의 내용과의 연관성을 분석합니다.
   - 사용자가 언급한 새로운 사실/관점이 기존 논의에 미치는 영향을 정리합니다.
   - 이를 바탕으로 구체적인 추가 논의 포인트, 액션 아이템, 수정이 필요한 기존 결론 등을 도출합니다.

5. 회의록 파일 끝에 아래 형식으로 추가 논의 내용을 append합니다:

```
---

## 추가 논의 (YYYY-MM-DD)

### 배경
> {사용자가 제시한 추가 논의 내용 원문}

### 기존 논의와의 연관성
{기존 회의 내용 중 관련된 부분 요약 및 연결}

### 추가 논의 사항
{새로운 사실/관점을 반영한 심화 논의}

### 수정/보완이 필요한 기존 결론
{기존 결론 중 변경이 필요한 부분이 있다면 명시}

### 추가 액션 아이템
- [ ] {도출된 후속 작업들}
```

6. 완료 후, 추가된 내용의 핵심을 간략히 요약하여 사용자에게 보고합니다.

## 주의사항
- 기존 회의록 내용은 절대 수정하지 않고, 항상 파일 끝에 append만 합니다.
- 추가 논의는 회의록의 톤과 형식에 맞춰 작성합니다.
- 날짜는 오늘 날짜를 사용합니다.
