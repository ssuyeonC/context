### 6-bis.2 방식 B: 여행 풀 한정 후 NTILE 적용

### 6-bis.3 두 방식 비교

| 세그먼트 | 방식 A (전체 풀) | 방식 B (여행 풀) | 관점 차이 |

- **타 블로그 카테고리와의 상대 비교** → 방식 A (동일 기준선)

> 세그먼트별 전체 블로그 ID 목록:

### 6-bis.4 결과 해석 및 인사이트

#### (1) 여행 블로그는 전체 대비 "상위층에 쏠려 있다" — 방식 A가 증명

전체 블로그 10,904개 기준 25%인 killer에 여행 블로그 710개(방식 A), 히든 젬에 863개가 들어감. 이는 여행 블로그 2,229개의 **70.57%가 전체 블로그 상위 25%**에 진입한다는 의미 (무작위 기대값 50%를 크게 초과).

- **해석**: `MAIN_REVIEW` 카테고리는 모집단 평균 대비 **체계적으로 고품질**. 전략 보고서의 "여행 리뷰 블로그 타겟 선정"은 데이터로 정당화됨.
- **반대 증거의 부재**: 방식 A에서 여행 블로그 중 "전체 상위 25% 조회수"(killer+passing = 832)는 전체 10,904 × 25% = 2,726 중 30.5%를 차지 → 여행 블로그가 전체 조회량의 약 1/3을 점유하는 지배적 카테고리임.

#### (2) 여행 블로그 내부에서는 "조회수와 좋아요율이 독립적" — 방식 B가 증명

방식 B의 세그먼트 비중(5.70/19.34/19.34/55.63)은 이론적 독립값(6.25/18.75/18.75/56.25)과 거의 일치. Section 4.3에서 전체 풀에서 관찰된 **양의 상관관계가 여행 블로그 내에서는 사라짐**.

- **해석**: 전체 풀에서의 상관관계(킬러 8.91% > 6.25%)는 **카테고리 간 차이**에서 기인 (예: 문화/뉴스는 조회↑·좋아요↓ 패턴). 여행 블로그만 떼어놓으면 "조회가 많다고 반드시 좋아요율이 높지는 않다".
- **시사점**: 여행 블로그 내에서 **"많이 읽히는 글"과 "반응이 좋은 글"은 서로 다른 집합**이므로, 둘 다 따로 최적화할 필요가 있음.

#### (3) 두 방식의 교차표 — 127개의 "진짜 킬러"와 492개의 숨은 리스크

| 방식 A \ 방식 B | B:killer | B:hidden | B:passing | B:low | 합계 |
|---|---:|---:|---:|---:|---:|
| A:killer | **127** | 65 | 355 | 166 | 713 |
| A:hidden | 0 | **366** | 0 | 492 | 858 |
| A:passing | 0 | 0 | **76** | 43 | 119 |
| A:low | 0 | 0 | 0 | **539** | 539 |
| **합계** | 127 | 431 | 431 | 1,240 | 2,229 |

주요 포인트:
- **127개 "진짜 킬러"**: 두 방식 모두 killer로 분류. 앱 설치 배너 A/B 최우선 적용 대상 (confidence 최고).
- **166개 "전체 기준 killer지만 여행 내 저성과"**: 전체 블로그 대비 좋아 보이지만, 여행 블로그끼리 비교하면 하위권. 전략 보고서에서 "killer 우선 적용" 대상으로 포함하면 ROI가 낮을 수 있음 → 방식 B 기준으로 걸러야 함.
- **355개 "전체 killer인데 여행 내 스쳐가는 콘텐츠"**: 조회수는 여행 내에서도 높은데 좋아요율이 낮음 → **CTA 개선 최우선 대상**. 방식 A만 보면 "이미 잘됨"으로 오판하기 쉬움.
- **492개 "전체 hidden인데 여행 내 저성과"**: 좋아요율은 전체 평균 대비 높지만, 여행 블로그 평균이 높아 상대적으로 가라앉음. 노출 확대(SEO) 투자 대비 효과가 불확실.
- **539개 "두 방식 모두 low"**: 명확한 아카이브/리뉴얼 후보.
