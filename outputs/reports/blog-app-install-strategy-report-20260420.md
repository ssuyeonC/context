## Executive Summary

**문제**: 앱 DAU가 1월 19명 → 4월 96명으로 5배 성장했으나, 웹 대비 여전히 14.9% 수준. 현행 띠배너 전략만으로는 성장 가속에 한계.

**기회**: 블로그 상세 페이지(/blog/{id})가 월 **538,587 활성 사용자**로 전체 페이지 중 압도적 1위. 2위 스팟 상세(228K) 대비 2.4배, 메인홈(69K) 대비 7.8배.

**전략**:
- **단기(5월)**: 앱 전용 쿠폰 배포 + 블로그 띠배너 메시지를 혜택 기반으로 전환
- **중기**: 블로그 스팟 '저장 → 여행 중 위치 기반 푸시 리마인더' 연결 (앱 전용 가치)
- **콘텐츠 최적화**: 블로그 4분면 세그먼트에 기반한 노출/품질 개선

**한 문장 정리**: *"블로그에서 여행을 '세팅'하고, 앱에서 여행을 '실행'하게 한다."*

- 그러나 웹 대비 절대 규모는 여전히 작음 → 추가 성장 동력 필요

### 1-2. 문제 진단

| 문제 | 근거 |
|------|------|
| 띠배너만으로는 성장 한계 | 블로그 내 현재 CTR이 낮은 상태로 추정됨 |

### 1-3. 기각된 대안: 메인홈 → 지도(/map) 유도

- 메인홈 모바일웹(한국) 활성 사용자: 1,253명/월
- 예상 퍼널 전환: **월 2~6명** (메인홈 → 피드 클릭 → /map → POI 상세 → 앱 설치)
- **결론**: 모수가 너무 작아 임팩트 부족 → 기각

## 3. 전략 방향

### 3-1. 전체 전략 구조

```
블로그 유저 (538K/월)
  ├─ 단기(5월): 띠배너 메시지 전환 → 앱 전용 쿠폰 혜택 소구 [외적 동기]
  │
  └─ 중기: 스팟 저장 → 여행 중 위치 기반 푸시 리마인더 [내적 동기]
        (블로그의 '세팅' 가치 → 앱의 '실행' 가치로 연결)
```

### 3-2. 단기 액션 (2026년 5월)

| # | 액션 | 내용 | 시점 |
|---|------|------|------|
| 1 | **앱 전용 쿠폰 배포** | 앱에서만 사용 가능한 쿠폰 제공 | 5월 중 |
| 2 | **띠배너 메시지 전환** | 현재 일반 메시지 → 앱 전용 쿠폰 혜택 메시지 | 쿠폰 배포 시점 |

**성과 측정**: 기존 GA 이벤트(`AppInstallBannerImpression`, `AppInstallBannerClick`)로 블로그 vs 스팟 페이지 CTR 비교.

### 3-3. 중기 아이데이션 (우선순위)

#### 1순위: 위치 기반 푸시 리마인더 (B안)

**퍼널 구조**:
```
블로그에서 여행 정보 탐색
  → 마음에 드는 스팟 저장/좋아요
    → "여행 중 이 장소 근처에서 알림 받으려면 앱 설치"
      → 앱 설치
        → 여행 중 위치 기반 리마인더 발동
```

**선정 이유**:
- 블로그 유저 JTBD("여행 세팅")와 앱 가치("여행 실행")가 자연스럽게 연결
- 기존 행동(저장)에 이어지는 가치 제안 → "맥락 있는 설치 유도"
- **인프라가 이미 존재**: 스팟 저장 기능 + 푸시 알림 시스템 모두 앱에 있음 → 연결 로직만 추가

**기술 검토 필요**: 위치 권한, 배터리 영향, 지오펜싱 정밀도

#### 2~4순위: 기타 아이데이션 후보

| 아이디어 | 앱 전용 가치 | 실현 난이도 |
|---------|------------|-----------|
| 블로그 업데이트 알림 (시즌 콘텐츠) | 재방문 동기 | **낮음** |
| 여행 준비 진척도 대시보드 | 상태 지속 + 재방문 | 중 |
| 블로그 읽은 장소 자동 플랜 제안 | 플래너 연계 원탭 | 중~높음 |
| 오프라인 블로그 저장 | 로밍 없는 환경 정보 접근 | 중 |

## 4. 핵심 인사이트

### Insight 1. 모수가 큰 곳에서 자연스러운 퍼널을 만드는 것이 낫다

메인홈(1,253명) → /map → POI → 앱 설치는 월 2~6명.
블로그(538K) → 저장 → 앱 설치는 퍼널이 짧고 모수가 **400배 이상**.
**같은 전환율이라도 모수에서 승부가 난다.**

### Insight 2. 띠배너의 한계는 "맥락 없는 제안"이다

정보 탐색 중인 오가닉 유저에게 무관한 설치 유도는 CTR이 낮다.
- **쿠폰 메시지(단기) = 외적 동기**
- **위치 기반 리마인더(중기) = 내적 동기**
둘 다 필요하지만, 장기적으로는 **내적 동기가 리텐션에 유리**.

### Insight 3. 블로그 유저의 JTBD는 "세팅", 앱의 가치는 "실행"이다

블로그에서 여행을 세팅하고 → 앱에서 여행을 실행한다.
이 **단계 전환**이 앱 설치의 자연스러운 이유가 된다.
앱에서도 같은 콘텐츠를 보여주는 것이 아니라, **"세팅한 것을 여행 중에 활성화해주는"** 기능이 핵심.

### Insight 4. 인프라가 있다는 것은 실현 가능성이 높다는 뜻이다

B안(위치 기반 푸시)은 스팟 저장 + 푸시 알림 시스템이 이미 앱에 있다.
**연결 로직만 추가**하면 되므로, 아이데이션 중 가장 빠른 프로토타입 가능.

### Insight 5. 블로그 콘텐츠 간 편차가 크다 — 타겟팅이 필요하다

전체 6,157개 블로그 중 **킬러 콘텐츠(972개, 8.91%)에 조회와 좋아요가 집중**.
앱 설치 유도 포인트를 모든 블로그에 일률적으로 적용하기보다, **킬러 콘텐츠 + 히든 젬 우선 적용**이 효율적.

### Insight 6. 블로그 engagement는 2019년 피크 이후 감소 추세

연도별 좋아요 수: 2019년 117K → 2025년 9K (YTD 2026: 834).
- 단순 재증가 기대보다 **앱 연계 재활성화 전략**으로 접근해야 함.
- "좋아요/저장" 행동 자체를 앱 설치 가치 제안과 연결하면 행동 회복 가능성.

## 7. 결론

블로그는 현재 웹 트래픽의 최대 모수(538K)를 보유하며, **여행 정보 탐색 → 여행 실행**이라는 유저 여정의 초입에 위치한 전략적 채널이다.

단기적으로는 **앱 전용 쿠폰 + 띠배너 메시지 전환**으로 즉시 전환율을 끌어올리고, 중기적으로는 **저장 행동 → 위치 기반 푸시 리마인더**라는 앱 전용 가치를 제안하여 "앱을 깔아야 하는 이유"를 구조화한다.

동시에 블로그 콘텐츠 세그먼트(킬러 8.91% / 히든 젬 16.09%)를 활용해 **앱 설치 유도 포인트의 타겟팅 효율을 극대화**한다.

> **"블로그에서 여행을 '세팅'하고, 앱에서 여행을 '실행'하게 하자."**
