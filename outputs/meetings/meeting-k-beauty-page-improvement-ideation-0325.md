## 0. 오늘 회의의 초점

**PO**: 3/22에 4-Phase 구조를 합의하고, 3/24에 데이터로 우선순위를 보정했습니다. 이제 리서치 인사이트까지 확보된 상태이니, 오늘은 **"각 Phase 안에 정확히 무엇을, 어떤 순서로 배치하는가"**를 확정합니다.

오늘의 핵심 질문은 하나입니다:

> **"18.9%인 스팟 클릭율을 25~30%로 끌어올리려면, 유저가 스크롤하면서 만나는 콘텐츠의 순서와 내용이 어떻게 설계되어야 하는가?"**

그 전에 리서치에서 나온 핵심 인사이트를 공유하고, 이걸 설계의 전제 조건으로 깔고 가겠습니다.

## 1. 리서치 인사이트 — 설계의 전제 조건

**Marketer**: 외국인 고객 인지도 리서치를 분석한 결과, 페이지 설계에 직접 영향을 미치는 핵심 인사이트가 3가지 있습니다.

### 전제 1: 고객은 '목적형 방문자'다

> 피부미용을 위해 한국을 찾는 외국인 관광객은 **'한국 여행 온 김에 피부 미용을 받는다'보단 '피부 미용을 받기 위해 한국에 방문한다'에 가깝다.**

- 피부과 진료가 외국인 환자 방문의 56.6%로 압도적 1위
- 리쥬란 엠보싱 회복 기간을 고려해 여행 일정 자체를 조정하는 수준
- "한국에서의 울쎄라는 비행기 값을 뽑는 시술"이라는 인식

**설계 시사점**: 이 유저들에게 "한국에서 뷰티 시술을 받을 수 있다"는 인지 단계를 길게 가져갈 필요가 없다. **인지(Phase 1)는 짧고 강하게 → 탐색(Phase 2)으로 빠르게 전환**해야 한다.

### 전제 2: 고객은 이미 '무엇을 받아야 하는지' 알고 있다

> 피부미용을 위해 한국을 찾는 외국인 관광객은 **피부에 대한 본인들의 니즈를 충족시키기 위해 어떤 시술을 받아야 하는지 정도는 인지하고 있다.**

- PN 성분, HIFU 원리 등 성분/기술 수준까지 사전 학습
- 리쥬란 종류별 차이를 구분하고 목적에 맞게 선택
- 콤비네이션 테라피를 고객이 먼저 제안하는 수준

**설계 시사점**: "피부과란 무엇인가"를 설명하는 초보자용 콘텐츠가 아니라, **"이 시술을 어디서, 얼마에, 어떤 조합으로 받을 수 있는가"에 빠르게 도달**하는 구조가 필요하다.

### 전제 3: 국가별로 고객 여정이 완전히 다르다

| 국적 | 핵심 동기 | 의사결정 기준 | 선호 시술 |
|------|----------|-------------|----------|
| 일본 | 가성비 + 빠른 시술 | 가격, 소요 시간 | 보톡스, 레이저, 점 빼기 |
| 중국 | 정품 신뢰 | 정품 인증, 의료진 경력 | 울쎄라, 써마지 |
| 미국/유럽 | 과학적 근거 | 성분 원리, 임상 결과 | 리쥬란, 쥬베룩 |
| 동남아 | K-뷰티 동경 | 비주얼 결과, 트렌드 | 리쥬란, 엑소좀 |

**설계 시사점**: 단일 구조로 모든 국가를 커버하되, **콘텐츠 레이어에서 국가별 차별화**가 가능한 구조를 만들어야 한다. 현재 `useLanguage`로 언어 감지가 되므로 이를 활용할 수 있다.

---

**PO**: 이 3가지 전제를 깔고 Phase별로 들어갑시다. 특히 전제 1과 2가 중요합니다 — **우리 고객은 초보자가 아니라 고관여 소비자**라는 점. 이걸 잊으면 "왜 크리에이트립인가"를 설명하는 데 페이지의 절반을 쓰게 됩니다.

## 2. Phase 1: 인지 — "여기서 예약하면 이득이다"

### 2-1. Phase 1의 목표 재정의

**PM**: 3/22에서 Phase 1의 목표를 "여기서 뷰티 시술을 받을 수 있다"로 정의했는데, 리서치 인사이트를 반영하면 **이 목표가 수정되어야 합니다.**

전제 1에 따르면, 유저는 이미 "한국에서 뷰티 시술을 받겠다"고 결정하고 온 사람들입니다. "받을 수 있다"는 건 이미 알아요. **이 유저들에게 인지 단계에서 전달해야 할 것은 "여기서(크리에이트립에서) 예약하면 왜 좋은가"**입니다.

| 3/22 정의 | 수정 정의 |
|-----------|----------|
| "여기서 뷰티 시술을 받을 수 있다" (시술 가능 여부 인지) | **"여기서 예약하면 이득이다"** (플랫폼 가치 인지) |

**Marketer**: 정확합니다. 목적형 방문자에게 "한국에서 뷰티 가능해요"는 이미 알고 있는 정보의 반복이고, 이 유저가 궁금한 건 **"왜 크리에이트립을 통해야 하는가"**예요. 이걸 3초 안에 전달해야 합니다.

**PO**: 동의합니다. Phase 1의 목표를 수정합니다.

### 2-2. Phase 1 콘텐츠 배치

**PM**: Phase 1의 콘텐츠를 위에서 아래로 논의합시다.

#### [1-A] Hero 섹션

**Marketer**: Hero는 **감성적 훅(hook)**입니다. 유저가 페이지에 착지했을 때 0.5초 안에 "맞는 페이지에 왔다"는 확신을 줘야 해요.

```
┌─────────────────────────────────────────────┐
│                                             │
│  [풀 블리드 비주얼]                           │
│   시술 중인 클리닉 내부 or 시술 전후 비교      │
│                                             │
│  "K-Beauty, Curated for You"                │
│   한국 뷰티 전문가가 엄선한 클리닉            │
│                                             │
│  100+ 클리닉 · 50K+ 방문 · 98% 만족          │
│                                             │
└─────────────────────────────────────────────┘
```

**Growth Manager**: 여기서 논의할 게 있습니다. **Social Proof 숫자(100+, 50K+, 98%)를 Hero 안에 넣을지, 별도 섹션으로 분리할지.** 3/22에서는 별도 `[Social Proof]` 섹션으로 분리했었는데.

**PM**: 리서치 전제 1을 적용하면, **Phase 1을 최대한 짧게 가져가야** 합니다. 별도 섹션으로 분리하면 스크롤 한 번 더 해야 하고, 목적형 방문자는 그 사이에 탐색으로 넘어가고 싶어해요. **Hero 안에 숫자를 인라인으로 포함시키고, Social Proof 독립 섹션은 삭제**하는 게 맞습니다.

**PO**: 동의합니다. Hero에 Social Proof를 통합합니다. 3/22의 `[Social Proof]` 독립 섹션은 제거.

**Marketer**: Hero 비주얼에 대해 한 가지 더. 리서치에서 국가별로 선호 시술이 다르다고 나왔는데, **Hero 비주얼을 언어별로 다르게 보여줄 수 있나요?** 일본 유저에게는 가벼운 레이저/보톡스 이미지, 중국 유저에게는 프리미엄 클리닉 내부, 서구권에는 Glass Skin 비포/애프터 같은 식으로.

**PM**: 기술적으로 가능합니다. `useLanguage`로 언어 감지가 되고, Hero 배너를 어드민화(P0-1)하면 언어별 배너 설정이 가능해요. 다만 **첫 출시에는 단일 비주얼로 가고, 어드민화 이후 언어별 분기를 P2로** 잡는 게 현실적입니다.

**PO**: 맞습니다. Hero 언어별 분기는 Phase 0(어드민화) 이후 과제로 기록합니다.

#### [1-B] Beauty Cashback 섹션

**Marketer**: 3/22 추가 논의에서 합의한 대로, 캐시백은 Hero 바로 아래에 독립 섹션으로 배치합니다. 이건 **"왜 크리에이트립인가"에 대한 가장 직접적인 답**이에요.

```
┌─────────────────────────────────────────────┐
│ 💰 뷰티캐시백                                │
│                                             │
│ 크리에이트립으로 예약하면                      │
│ 시술비의 일부를 포인트로 돌려받아요             │
│                                             │
│ 예약 → 시술 → 영수증 인증 = 캐시백!           │
│                                             │
│ [캐시백 자세히 보기 →]                        │
└─────────────────────────────────────────────┘
```

**Growth Manager**: 캐시백 섹션에 **실제 환급 사례 숫자**를 넣으면 더 강력합니다. "이번 달 평균 환급액: ₩XX,XXX" 같은. 3/24에서 P2-10으로 잡았고 P1 우선순위니까, 데이터 파이프라인이 Phase 0에서 준비되어야 해요.

**PM**: 캐시백 데이터가 아직 충분하지 않을 수 있으니, **초기에는 제도 설명 중심으로 시작하고, 데이터가 쌓이면 실제 환급 사례를 추가**하는 2단계 접근이 안전합니다.

#### [1-C] Trust 섹션 (Why Creatrip)

**PM**: 3/22에서 합의한 대로, 기존 Trust Highlights에서 캐시백 카드를 분리하고 나머지 3개를 유지합니다.

```
┌─────────────────────────────────────────────┐
│ Why Creatrip?                               │
│                                             │
│ [🏥 검증된 클리닉]  [💰 합리적 가격]  [💬 24시간 지원] │
│  100개 이상의       현지 시세 기반     다국어 상담     │
│  파트너 클리닉      투명한 가격 제공   연중무휴 운영    │
└─────────────────────────────────────────────┘
```

**Marketer**: 리서치에서 가격 투명성이 신뢰의 전제 조건이라는 인사이트가 나왔습니다. Trust 카드 중 **"합리적 가격" 카드를 "투명한 가격"으로 리프레이밍**하는 게 좋겠어요. "싸다"가 아니라 "숨기지 않는다"가 이 고객층에게 더 소구됩니다.

| 현재 카피 | 수정 제안 | 근거 |
|----------|----------|------|
| 합리적 가격 | **투명한 가격** | 리서치: 고객이 레딧으로 시세를 꿰뚫고 있고, 서면 견적서를 요구하는 수준 |

**PM**: 좋은 포인트입니다. "현지 시세 기반 투명한 가격 제공" — 이 카피가 레딧에서 공유되는 "외국인 바가지 방지" 니즈에 정확히 대응합니다.

**Growth Manager**: Trust 카드에 **정품 인증** 관련 메시지도 추가할 필요가 있지 않을까요? 리서치에서 중국 고객이 정품 여부를 최우선으로 본다고 나왔고, 서구권도 정품 팁 인증 스티커를 요구하는 수준입니다.

**PM**: 3개 카드에 정품 인증을 추가하면 4개가 되는데, 캐시백을 뺀 이유가 "4개가 많아서"였으니 다시 4개로 돌아가면 안 됩니다. **"검증된 클리닉" 카드의 서브 카피에 정품 인증을 포함**시키는 게 낫습니다. "100개 이상의 파트너 클리닉 · 정품 장비 인증"처럼.

**PO**: 동의합니다. Trust 카드는 3개 유지, "검증된 클리닉"에 정품 인증 메시지 포함.

### 2-3. Phase 1 최종 배치

```
Phase 1: 인지 — "여기서 예약하면 이득이다"
│
├─ [1-A] Hero (풀 블리드 비주얼 + 핵심 카피 + Social Proof 인라인)
│       └─ 언어별 비주얼 분기는 어드민화 이후 P2

├─ [1-B] Beauty Cashback (독립 섹션, 제도 설명 + CTA)

│
└─ [1-C] Trust — Why Creatrip (3개 카드)

        ├─ 투명한 가격 (리프레이밍)
        └─ 24시간 다국어 지원
```

**PO**: 3/22 대비 변경점을 정리합니다.

| 변경 | 내용 | 근거 |
|------|------|------|
| Phase 1 목표 수정 | "시술 가능 여부 인지" → **"플랫폼 가치 인지"** | 리서치 전제 1: 고객은 목적형 방문자 |
| Social Proof 독립 섹션 삭제 | Hero에 인라인 통합 | Phase 1 압축 → 빠른 탐색 전환 |
| Trust "합리적 가격" 리프레이밍 | → **"투명한 가격"** | 리서치: 가격 투명성이 신뢰의 전제 조건 |
| Trust "검증된 클리닉"에 정품 인증 추가 | 서브 카피에 포함 | 리서치: 중국·서구권 고객의 정품 민감도 |

## 3. Phase 2: 탐색 — "나에게 맞는 시술과 클리닉 찾기"

### 3-1. Phase 2의 설계 원칙

**PM**: Phase 2는 **스팟 클릭율 18.9% → 25~30%**를 달성해야 하는 핵심 구간입니다. 3/24 데이터에서 873명 중 708명이 스팟을 클릭하지 않고 이탈한 구간이 바로 여기예요.

리서치 전제 2를 적용하면, 이 고객들은 **"어떤 시술을 받아야 하는지"는 이미 알고 있습니다.** 그러니까 Phase 2에서 해줘야 하는 건 "시술 교육"이 아니라 **"이미 알고 있는 시술을 크리에이트립에서 예약 가능한 스팟으로 연결"**해주는 것이에요.

**Growth Manager**: 그렇다면 Phase 2의 콘텐츠 배치 원칙은 이렇습니다:

| 원칙 | 설명 |
|------|------|
| **시술명 → 스팟 매칭을 최단 경로로** | 유저가 "리쥬란"을 받고 싶다고 이미 알고 있으면, 리쥬란 가능한 스팟에 2클릭 이내로 도달 |
| **탐색 진입점을 복수로 제공** | 시술명으로 찾기(Wizard), 카테고리로 찾기(Tab), 트렌드로 찾기(인기 시술) — 여러 경로 |
| **스크롤 없이 첫 스팟 카드가 보여야** | Phase 2 진입 시점에서 스팟 카드가 above-the-fold에 위치 |

### 3-2. Phase 2 콘텐츠 배치 논의

#### [2-A] 실시간 인기 시술 섹션 (P2-8, P1 승격)

**Growth Manager**: 3/24에서 P1으로 승격한 섹션입니다. **Phase 2의 첫 번째 콘텐츠로 배치**를 제안합니다. 이유는:

1. 유저가 Phase 1(인지)을 지나 탐색에 진입하면, **가장 먼저 보고 싶은 건 "지금 뭐가 핫한가"**입니다
2. 목적형 방문자라도 **"다른 사람들이 뭘 예약하는지"**는 의사결정에 강한 영향을 줍니다

## 4. Phase 3: 검증 — "이거 괜찮은 거 맞지?"

### 4-1. Phase 3의 역할 재정의

**PM**: 3/22에서 Phase 3을 "리뷰 + 블로그로 검증"이라고 정의했는데, 리서치를 반영하면 **검증의 범위가 더 넓어져야** 합니다.

**Growth Manager**: 추가로, 리뷰에서 **시술명 태그**가 보여야 합니다. 지금은 스팟 이름만 보이는데, 유저가 "리쥬란 후기"를 찾고 있을 때 스팟 이름만으로는 시술 내용을 알 수 없어요. **시술명 태그 + 스팟명 병기**가 필요합니다.

**PM**: 리뷰 데이터에 시술명 필드가 있는지 확인이 필요합니다. 없으면 리뷰 작성 시 시술명 선택을 추가해야 하고, 이건 스팟 상세의 변경이라 K뷰티 페이지 scope 밖입니다.

**PO**: 리뷰에 시술명 태그 표시는 **데이터 구조 확인 후 결정**으로 기록합니다.

#### [3-B] 시술 가이드 블로그 (기존 블로그 섹션 리프레이밍)

**Marketer**: 기존 Popular Blogs + Latest Blogs를 합치되, **"블로그"라는 라벨을 "시술 가이드"로 리프레이밍**하면 검증 단계의 성격에 더 맞습니다.

```
┌─────────────────────────────────────────────┐
│ 📖 시술 가이드 — 시술 전에 알아두세요          │
│                                             │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│ │리쥬란     │ │울쎄라 vs  │ │한국 피부과│     │
│ │완벽 가이드│ │슈링크    │ │처음이라면│     │
│ │          │ │어떤 게    │ │          │     │
│ │          │ │나을까?   │ │          │     │
│ └──────────┘ └──────────┘ └──────────┘     │
│                                             │
│ [전체 가이드 보기 →]                          │
└─────────────────────────────────────────────┘
```

**PM**: 리서치에서 고객들이 **시술 간 비교(울쎄라 vs 슈링크)**와 **콤비네이션 가이드(울쎄라+리쥬란)**에 관심이 높다고 나왔습니다. 블로그 콘텐츠 기획 시 이 포맷을 우선 제작하면 검증 단계의 체류 시간과 스팟 전환에 기여할 수 있어요.

**Marketer**: 맞습니다. 특히 **"울쎄라 vs 슈링크" 같은 비교 콘텐츠**는 레딧에서 가장 많이 공유되는 포맷이에요. 이걸 크리에이트립 블로그로 만들면 외부 유입 경로까지 확보됩니다.

**Growth Manager**: 한 가지 더 — 리서치에서 **차세대 시술(쥬베룩, 엑소좀)**의 인지도가 급부상 중이라고 나왔는데, 이 시술들에 대한 가이드 콘텐츠를 **선제적으로 준비**하면 검색 유입에서 유리합니다. 아직 경쟁이 적은 키워드니까요.

**PO**: 블로그 콘텐츠 방향은 마케팅팀 소관이지만, **시술 비교 + 차세대 시술 가이드를 우선 제작 과제로 기록**합니다.

### 4-3. Phase 3 최종 배치

```
Phase 3: 검증 — "실제 결과와 전문성으로 신뢰 확인"
│
├─ [3-A] 통합 리뷰 섹션
│       ├─ 필터: 전체 / 포토 / 영상
│       ├─ 국가 플래그 아이콘 표시
│       └─ 시술명 태그 표시 (데이터 구조 확인 후)
│
└─ [3-B] 시술 가이드 (기존 블로그 리프레이밍)
        ├─ 시술 비교 콘텐츠 우선 배치 (울쎄라 vs 슈링크 등)
        ├─ 차세대 시술 가이드 선제 준비 (쥬베룩, 엑소좀)
        └─ 콤비네이션 가이드 (울쎄라+리쥬란 등)
```

## 5. Phase 4: 행동 — "예약하기"

### 5-1. Phase 4 배치 논의

**PM**: Phase 4는 3/22에서 "지도 + 문의 폼"으로 합의했었는데, 변경이 필요한 부분이 있을까요?

**Growth Manager**: 지도 섹션은 유지하되, 하나 추가를 제안합니다. 리서치에서 고객들이 **공장형 vs 부티크 병원을 전략적으로 구분**한다고 나왔는데, 지도에서 **클리닉 유형 필터**를 제공하면 이 행동을 지원할 수 있어요.

**Marketer**: 그건 민감한 이슈입니다. **파트너 클리닉에 "공장형"이라는 라벨**을 붙이면 관계가 어려워져요. 직접 라벨링 대신 **"1:1 맞춤 상담", "프리미엄", "퀵 시술"** 같은 서비스 특성 태그로 간접 분류하는 게 낫습니다.

**PM**: 동의합니다. 클리닉 유형 태그는 **어드민에서 관리하는 방식으로, 파트너 클리닉에 불리하지 않은 표현**을 써야 해요.

**PO**: 클리닉 유형 태그는 좋은 아이디어지만, **표현 설계가 민감하니 P3(중기)로** 분류합니다.

### 5-2. Phase 4 최종 배치

```
Phase 4: 행동 — "예약하기"
│
├─ [4-A] 지도 섹션 (기존 유지)
│       └─ 클리닉 서비스 특성 태그 필터 (P3, 표현 설계 후)
│
├─ [4-B] Inquiry Form (기존 유지)
│
└─ [Floating] 챗봇 + 이메일 상담 버튼 (기존 유지)
```
