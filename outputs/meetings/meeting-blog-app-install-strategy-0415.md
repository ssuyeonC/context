## 1. Q1 — 어디에 집중할 것인가

### 1-1. 기각된 안: 메인홈 → 지도(/map) 유도

**PM**: 원래 검토했던 안은, 메인홈에 지도 페이지(/map)로 이동할 수 있는 피드형 영역을 추가하는 것이었습니다. 지도의 특정 POI(올리브영, 아트박스 등)는 상세 정보를 보려면 앱 설치가 필요하니까, 그 버튼을 활용하자는 거였어요.

**Growth Manager**: 퍼널을 추정해봤습니다.

| 지표 | 수치 |
|------|------|
| 메인홈 모바일웹(한국) 활성 사용자 | 1,253명/월 |
| /map 활성 사용자 | 4,916명/월 (GA 기준) |
| 예상 퍼널 전환 | 월 2~6명 |

메인홈 → 피드 클릭(CTR 5~10%) → /map → POI 상세 클릭 → 앱 설치(CVR 5~10%). 각 단계에서 이탈이 발생하면 월 2~6명 수준밖에 안 됩니다. 현재 앱 일평균 96명 대비 임팩트가 없어요.

**PO**: 기각합니다. 모수가 너무 작아요.

### 1-2. 페이지별 활성 사용자 비교

**Growth Manager**: 그래서 모수가 큰 페이지를 찾았습니다.

| 순위 | 페이지 | 활성 사용자 |
|------|--------|------------|
| 1 | /blog/{id} | **538,587** |
| 2 | /spot/{id} (상세) | 228,046 |
| 3 | / (메인홈) | 68,987 |
| 4 | /landing-area | 11,623 |
| 5 | /stays/{id} | 11,190 |

> 수치는 `UserEventsAcquiredFromAll` 기반(유입 세션 중심)으로, GA Pages and Screens 수치와 차이가 있을 수 있음

**PO**: 블로그 상세가 538K로 압도적 1위군요. 블로그를 주요 채널로 선정합시다.

### 1-3. 블로그 내 타겟 유형 선정

**PM**: 블로그는 크게 두 성격으로 나뉩니다. 여행 정보 블로그와 여행 외 정보 블로그(한국 이름, 연예 정보 등)입니다. 여행 외 정보 블로그의 조회수가 압도적으로 많아요. 이런 블로그에 스팟이 연결되어있는 경우도 있는데, 이 전환율은 또 많이 낮은 편이고. 반대로 여행 상품과 관련된 블로그의 전환율은 꽤 높은데 조회수는 그렇게 높지 않은 편입니다.

**Growth Manager**: DB 데이터로 확인한 카테고리별 현황입니다.

| 카테고리 | 블로그 수 | 총 조회수 | 평균 조회수 | 스팟 연결률 |
|---------|----------|----------|-----------|-----------|
| 여행 리뷰 (MAIN_REVIEW) | 2,229 | 1.68억 | 48,872 | 60.8% |
| 예약 연결 (MIDDLE_RESERVATION) | 358 | 2,690만 | 48,554 | 99.7% |
| 문화/뉴스 (MAIN_CULTURE_AND_NEWS) | 916 | 2,194만 | 23,023 | 3.8% |

**Marketer**: 문화/뉴스 블로그는 유입 목적 자체가 여행이 아니라 정보 소비예요. 여기서 앱 설치를 유도해봤자 여행 앱을 깔 동기가 없습니다. 설령 깔아도 리텐션이 안 나올 거예요.

**PM**: 동의합니다. 예약 연결 블로그는 전환 의도가 가장 높지만 글 수가 358개로 모수가 너무 적고요.

**PO**: **여행 리뷰 블로그**로 합시다. 볼륨(2,229개, 1.68억 조회)과 스팟 연결률(60.8%)의 균형이 가장 좋습니다.

## 2. Q2 — 단기에 무엇을 할 것인가

### 2-1. 현재 블로그의 앱 설치 띠배너 현황

**PM**: 현재 블로그 상세에도 이미 앱 설치 유도 띠배너가 노출되고 있습니다. 3월 6일에 배포된 거예요.

| 항목 | 현황 |
|------|------|
| **배포일** | 2026-03-06 |
| **노출 위치** | 메인홈, 스팟홈, 스팟상세, 스팟리스트, 블로그 상세 |
| **현재 CTR** | 낮은 편 |
| **GA 이벤트** | AppInstallBannerImpression, AppInstallBannerClick |

### 2-2. 왜 CTR이 낮은가

**Marketer**: 블로그는 오가닉 유입이 많아서, 유저가 정보를 탐색하는 맥락에 있어요. 거기서 갑자기 "앱 깔아라"는 맥락과 안 맞습니다. 유저 입장에서 앱을 깔아야 하는 이유가 명확하지 않아요.

**PM**: 맞아요. '앱에서 더 많은 여행 정보 보기' 같은 메시지는 자연스럽긴 한데, 사실 앱에 그런 차별화된 기능이 있는 건 아닙니다. 지금 앱과 웹이 차별화된 포인트는 혜택(쿠폰)이에요. 블로그를 찾아오는 오가닉 유저의 맥락과는 좀 다르지만, 전환 동기로는 혜택이 가장 직접적입니다.

### 2-3. 단기 액션 확정

**PM**: 앱 전용 쿠폰이 배포되면 띠배너 메시지도 그에 맞게 바꿀 겁니다. 이건 적용이 될 예정이라고 봐도 돼요. 두 가지를 동시에 진행합니다.

| # | 액션 | 내용 | 시점 |
|---|------|------|------|
| 1 | **앱 전용 쿠폰 배포** | 앱에서만 사용 가능한 쿠폰 제공 | 5월 중 |
| 2 | **띠배너 메시지 전환** | 현재 일반 메시지 → 앱 전용 쿠폰 혜택 메시지로 변경 | 쿠폰 배포 시점 (5월 중) |

**Growth Manager**: 성과 측정은 어떻게 합니까?

**PM**: 기존 GA 이벤트(AppInstallBannerImpression/Click)로 충분합니다. 블로그 페이지에서의 Impression과 Click을 스팟 페이지와 비교하면 채널별 효과를 볼 수 있어요.

**PO**: 좋습니다. 1번과 2번을 동시에 진행합시다.

## 3. Q3 — 중기에 어떤 가치를 줄 것인가

### 3-1. 논의 방향 설정

**PM**: 띠배너 외에, 블로그 콘텐츠 안에서 앱 전용 기능으로 연결되는 포인트를 추가하고 싶습니다. 예를 들어 '앱에서 지도로 보기' 같은 버튼.

**PO**: 지도 페이지의 '앱에서 보기' 버튼을 블로그 내 스팟 영역에 붙이는 건?

**PM**: 그건 안 됩니다. 지도 페이지의 그 버튼은 스팟이 아니라 올리브영, 아트박스, 롯데마트 등의 POI에만 제한적으로 붙어있는 거라, 스팟이 연결된 블로그에 직접 노출하는 건 바람직하지 않습니다.

**PM**: 반드시 지금 있는 기능을 추가하는 것 외에, **어떤 가치를 줄 수 있는 기능을 새로 만들 수 있을지** 아이데이션 수준으로 논의해봅시다.

### 3-2. 블로그 유저의 맥락

**Growth Manager**: 아이데이션 전에, 타겟 유저의 맥락을 정리합시다. 블로그 유저(세그먼트 2)의 프로필입니다.

| 항목 | 내용 |
|------|------|
| **여행 단계** | 여행 전 |
| **주요 접점** | /blog/{id} |
| **유저 관계** | 신규 웹 유저 (대부분 오가닉) |
| **JTBD** | *"한국 여행 전체 판을 망하지 않는 구조로 세팅하고 싶다"* |

**Marketer**: 핵심은 이 유저에게 **앱을 설치해야만 얻을 수 있는 가치**가 무엇인지입니다. 웹에서도 똑같이 할 수 있으면 설치할 이유가 없어요.

### 3-3. 아이디어 후보

**PO**: 자유롭게 아이디어를 내봅시다.

| | 아이디어 | 개요 | 앱 전용 가치 |
|---|---------|------|------------|
| A | 블로그에서 읽은 장소를 한 탭으로 일정에 추가 | 웹에서는 불가능한 매끄러운 경험 | 편의 |
| B | 여행 중 푸시 알림 — "저장한 스팟 근처에요!" | 위치 기반 리마인더 | 실행 |
| C | 블로그 콘텐츠 오프라인 저장 | 비행기에서도 읽을 수 있는 여행 가이드 | 편의 |
| D | 블로그 기반 나만의 여행 체크리스트 자동 생성 | 앱 전용 | 정보 |

**Marketer**: A안은 편리하지만 "한 탭으로 일정에 추가"는 웹에서도 구현 가능해요. 앱만의 이유가 약합니다.

**Growth Manager**: C안(오프라인 저장)은 r/koreatravel 분석에서도 오프라인 정보 접근이 반복적 니즈로 나왔습니다. 실용적인 가치인데, 다만 오프라인 저장을 위해 앱을 까는 건 동기가 좀 약하지 않나요? 여행 출발 직전이 아니면 체감이 안 됩니다.

**PM**: D안(체크리스트 자동 생성)도 재밌지만, "자동 생성"이 얼마나 정확한지가 기능의 신뢰를 좌우해요. 초기에 품질이 낮으면 오히려 역효과입니다.

**PM**: **B안(위치 기반 푸시 알림)**이 가장 끌립니다.

### 3-4. B안 구체화 — 여행 중 위치 기반 리마인더

**PO**: B안이 동작하려면 전제 조건이 있지 않나요?

**PM**: 두 가지입니다. 유저가 블로그에서 스팟을 '저장'하는 행동, 그리고 앱에서 위치 기반 푸시 알림 시스템. **둘 다 현재 앱에 있습니다.** 스팟 좋아요/저장 기능도 있고, 푸시 알림 시스템도 있어요.

**Growth Manager**: 그러면 새로운 인프라 구축 없이 연결 로직만 추가하면 되는 건가요?

**PM**: 네, 기본 인프라는 있습니다. 지오펜싱 정밀도나 배터리 영향 같은 기술 검토는 필요하지만, 프로토타입 수준은 빠르게 만들 수 있어요.

**Marketer**: 퍼널 구조를 그려보면 이렇습니다.

```
블로그에서 여행 정보 탐색
  → 마음에 드는 스팟 저장/좋아요
    → "여행 중 이 장소 근처에서 알림 받으려면 앱 설치"
      → 앱 설치
        → 여행 중 위치 기반 리마인더 발동
```

**Marketer**: 이게 좋은 이유는, 기존 띠배너가 맥락 없이 "앱 깔아라"였다면, 이건 **유저가 이미 취한 행동(저장)에 이어지는 가치 제안**이라는 점이에요. 블로그 유저의 맥락("여행을 세팅하고 싶다")과 앱의 가치("세팅한 걸 여행 중에 실행하게 도와준다")가 자연스럽게 연결됩니다.

**PO**: 좋습니다. B안을 중기 아이데이션의 1순위로 가져갑시다.

## 4. 인사이트

### Insight 1. 모수가 작은 곳에서 전환을 쥐어짜는 것보다, 모수가 큰 곳에서 자연스러운 퍼널을 만드는 것이 낫다

메인홈(1,253명) → 지도 → POI → 앱 설치는 월 2~6명이었다. 블로그(538K) → 저장 → 앱 설치는 퍼널이 짧고 모수가 400배 이상이다. **같은 전환율이라도 모수에서 승부가 난다.**

### Insight 2. 띠배너의 한계는 "맥락 없는 제안"이다

현재 띠배너 CTR이 낮은 이유는 유저의 현재 행동과 앱 설치 제안 사이에 연결 고리가 없기 때문이다. **쿠폰 메시지(단기)는 외적 동기, 위치 기반 리마인더(중기)는 내적 동기다.** 둘 다 필요하지만, 장기적으로는 내적 동기가 리텐션에 유리하다.

### Insight 3. 블로그 유저의 JTBD는 "세팅"이지만, 앱의 가치는 "실행"이다

블로그에서 여행을 세팅하고 → 앱에서 여행을 실행한다. 이 **단계 전환**이 앱 설치의 자연스러운 이유가 된다. 앱에서도 같은 콘텐츠를 보여주는 것이 아니라, **"세팅한 것을 여행 중에 활성화해주는"** 기능이 핵심.

### Insight 4. B안의 인프라가 이미 존재한다는 것은 실현 가능성이 높다는 뜻이다

스팟 저장 기능 + 푸시 알림 시스템이 모두 앱에 있다. 새로운 인프라 구축 없이 **연결 로직만 추가**하면 되므로, 아이데이션 단계에서 가장 빠르게 프로토타입을 만들 수 있는 안이다.
