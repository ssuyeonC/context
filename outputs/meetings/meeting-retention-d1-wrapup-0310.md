## 3. 유저 행동 레이어 모델

```
Layer 0: 방문 — 오가닉/SNS/광고 유입
   ↓ 교차탐색 유도 (추천, 검색, 럭키드로우)
Layer 1: 탐색 — 상품 2개+ 비교  │  리텐션 1.63x · 액션율 11.64%
   ↓ 저장 동기 (매력도, 소셜프루프)
Layer 2: 저장 — 좋아요/찜        │  리텐션 2.17x · 액션율 1.00%
   ↓ "일정에 넣어볼까요?" ⚠️ 높은 마찰 (발산→수렴 전환)
Layer 3: 계획 — 플래너에 배치    │  리텐션 1.72x · 액션율 0.27%
   ↓
Layer 4: 구매 + 실행
   ↓
Layer 5: 공유 + 재방문
```

**구매(L4)는 어느 레이어에서든 발생 가능.** 플래너(L3)의 결정적 가치는 **"구매 후"에 나타남** — 플래너에 맥락이 있는 유저만 여행 중/후에 재방문.

## 4. 전략 포트폴리오 — 다채널 Open Loop

| Open Loop 유형 | 메커니즘 | 대상 | 플래너 의존 | 단기 임팩트 | Lock-in |
|---------------|---------|------|-----------|-----------|---------|
| 교차탐색 루프 | 추천 카드 + 비교하기 | 상품 탐색 유저 | 없음 | **최고** | 중간 |
| 콘텐츠 시리즈 | 블로그 파트별 + 상품 인라인 | 블로그 유입 | 없음 | 높음 | 낮음 |
| 럭키드로우×교차탐색 | 매일 응모 + 3회 조회 추가 응모권 | 전체 재방문 | 없음 | 중간→높음 | 중간 |
| 가격/관심 알림 | "관심 있어요" → D+1 CRM | 구매 고려 유저 | 없음 | 중간 | 중간 |
| 플래너 Open Loop | 일정 추가 → 빈 날짜 | 계획 단계 유저 | 100% | 낮음 (모수 0.27%) | **최고** |
| 좋아요 CRM | 찜 → 시의성 리마인더 | 좋아요 유저 | 없음 | 낮음 | 중간 |

> **"플래너는 왕관이지 유일한 무기가 아니다."**
> 단기 임팩트는 교차탐색·좋아요·럭키드로우에서, 장기 Lock-in은 플래너에서.

### A-1 "좋아요 스트릭" 시그널 품질 우려 (PO 추가 논의)

좋아요 데이터는 **퍼널 진입점**(COM-2320 바텀시트 트리거)과 **유저 취향 시그널**(개인화 추천, CRM)의 이중 역할을 수행. 보상으로 인위적 좋아요를 유도하면:

- 개인화 추천 정확도 하락
- CRM 메시지 효과 감소 (관심 없는 상품에 발송)
- 플래너 전환률 희석 (모수 ↑ but 전환률 ↓ → 절대 전환 수 정체)

**대안 방향**: 좋아요 행동 자체를 인위적으로 올리지 말고, **매력적인 콘텐츠/상품 노출 확대**로 자연스러운 좋아요 증가 유도. A-6(오늘의 발견), A-4(Hidden Combo), COM-2320 바텀시트의 후속 경험 품질 강화가 더 바람직.

## 9. 플래너 전략 — 단계적 투자 + 데이터 기반 확대

### 포지셔닝 전환

> **"일정 짜는 도구" → "일정 검증 도구"**
> Reddit 34.3%의 검증 니즈 흡수. 구글 스프레드시트/노션이 할 수 없는 유일한 가치.

### 투자 단계

```
[즉시] 카테고리 A — 진입 마찰 제거
  ├─ 진입점 확대 (검색/좋아요/블로그 "내 일정에 추가")
  ├─ "날짜 없이 일단 추가" 경량 온보딩
  ├─ "일정 검증" 라이트 (빡빡함 경고 + 이동 시간 추정)
```
