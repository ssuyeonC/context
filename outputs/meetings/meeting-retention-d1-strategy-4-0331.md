**PO**: 4/7 Go/No-Go까지 **정확히 1주**입니다. T1이 배포된 상태이고, 초기 수치가 나왔으니 오늘은 세 가지를 논의합니다.

1. **T1 초기 데이터 해석** — 배포 후 수치가 의미하는 것과 한계

## 1. T1 초기 데이터 리뷰

### 1-1. 배포 현황

**PM**: T1의 두 가지 핵심 기능이 모두 배포 완료되었습니다.

| 기능 | 상태 | 배포일 |
|------|------|--------|
| 좋아요 페이지 내 "플랜에 추가" 기능 | **배포 완료** | 3월 중순~하순 |
| 좋아요 클릭 후 바텀시트 내 '플랜 생성하러 가기' 개선 | **배포 완료** | 3월 중순~하순 |

### 1-2. 초기 수치

**Growth Manager**: 배포 후 수집된 클릭율 데이터입니다.

| 지표 | 수치 | 비고 |
|------|------|------|
| 좋아요 페이지 "플랜에 추가" 버튼 클릭율 | **5.21%** | 좋아요 페이지 방문자 대비 |
| 바텀시트 내 '플랜 생성하러 가기' 클릭율 | **4.59%** | 바텀시트 노출 대비 |

**PO**: 이 수치를 어떻게 해석하나요?

**Growth Manager**: 두 관점에서 봐야 합니다.

**첫째, T1 목표 대비 해석.** 3/10 Wrap-Up에서 T1의 핵심 지표를 **"좋아요→플래너 전환율 10%+"**로 설정했습니다. 현재 5.21%는 이 목표의 절반 수준입니다. 다만 주의할 점은, **"플랜에 추가" 버튼 클릭이 곧 플래너 전환 완료는 아닙니다.** 클릭 후 실제로 플랜에 스팟을 추가하고 플래너를 사용하는 비율은 별도로 추적해야 합니다.

**둘째, L2→L3 브릿지로서의 의미.** 3/5 플래너 당위성 회의에서 좋아요(L2)→플래너(L3)를 "발산→수렴 심리 전환"이라고 정의했고, 이 전환이 자연스럽게 일어나지 않는 것이 플래너 진입률 0.27%의 본질적 원인이라고 분석했습니다. **5.21%의 클릭율은 이 마찰 구간에 UI 수준의 브릿지를 놓았을 때 반응하는 유저가 존재한다는 것을 확인**해주지만, 아직 목표에는 미달입니다.

**PM**: 바텀시트의 4.59%도 비슷한 맥락입니다. 좋아요를 클릭하는 순간은 "발산" 심리가 가장 강한 타이밍인데, 그 순간에 4.59%가 "수렴"(플랜 생성)으로 전환 의사를 보인 겁니다. **순간적 모멘텀으로는 의미 있는 수치**이지만, 실제 플랜 생성까지 이어지는지가 관건이에요.

**Marketer**: 유저 관점에서 보면, 좋아요 페이지에서 5.21%, 바텀시트에서 4.59%면 **두 진입점의 효율이 비슷한 수준**이에요. 이건 "좋아요 맥락에 있는 유저의 약 5%가 플래너에 관심을 보인다"는 일관된 패턴으로 읽힙니다. 한쪽이 극단적으로 높거나 낮지 않으니, 특정 UI의 문제라기보다는 **현재 좋아요 유저풀에서 플래너로 전환 가능한 자연스러운 비율**에 가깝습니다.

**PO**: 정리하면:

| 관점 | 해석 |
|------|------|
| 목표 대비 | 10% 목표의 약 절반 — **미달이지만 0%는 아님** |
| 브릿지 효과 | L2→L3 마찰 구간에서 UI 개입이 반응을 만들어냄 — **방향은 유효** |
| 두 진입점 비교 | 5.21% vs 4.59%로 일관적 — **특정 UI 이슈가 아닌 구조적 전환율** |
| 미확인 | 클릭 후 실제 플랜 생성/스팟 추가 완료율, 이 유저들의 D+1 재방문 여부 |

### 1-3. 추가 확인 필요 사항

**Growth Manager**: 4/7 Go/No-Go에서 플래너 진입률 0.5%+ 달성 여부를 판단하려면, 다음 데이터가 필수입니다.

| 필요 데이터 | 현재 상태 | 4/7까지 확보 가능 여부 |
|------------|----------|---------------------|
| T1 경유 실제 플랜 생성 수 (일별) | 미확인 | **확인 필요** |
| T1 이후 전체 플래너 진입률 변화 | 미확인 | 트렌드 확인 가능 |
| T1 신규 플래너 유저의 D+1 재방문률 | 아직 코호트 부족 | 1주 데이터로 초기 시그널 가능 |

**PM**: T1 경유 플랜 생성 수는 이번 주 안에 대시보드에 추가하겠습니다.

**PO**: 좋습니다. **이번 주가 4/7 판단을 위한 데이터 수집의 마지막 주**입니다. 놓치면 안 됩니다.

## 2. T2/T3 착수 상태 점검

### 2-1. T2: 럭키드로우 × 교차탐색

**PM**: T2 현황입니다.

| 항목 | 상태 |
|------|------|
| 현재 | **진행 예정** — 스펙 확정, 개발 착수 준비 |
| 핵심 | 서로 다른 스팟 상세페이지 3회 조회 시 럭키드로우 추가 응모권 |
| 리스크 | 없음 — 스펙 변경 사항 없음 |

**Growth Manager**: T2는 3/10 회의에서 확정한 스펙 그대로 가는 건가요? 마켓 스캔에서 "3회 조건이 너무 쉬워 무의미한 클릭이 될 수 있다"는 리스크(#18)가 있었는데요.

**PO**: 맞습니다. 3회 vs 4회 A/B 테스트를 리스크 가드레일에 넣어뒀었죠. 지금 시점에서는 **먼저 3회로 출시하고, 2주 데이터로 "3회 달성률"과 "달성 후 실제 응모 전환율"을 봐서 판단**하는 게 맞습니다. A/B까지 하면 출시가 늦어져요.

**PM**: 동의합니다. 3회로 배포 후, 달성률이 80% 이상이면 조건이 너무 쉬운 거니까 4회로 올리는 판단을 하면 됩니다.

### 2-2. T3: 앱 전용 쿠폰

**PM**: T3 현황입니다.

| 항목 | 상태 |
|------|------|
| 현재 | **진행 예정** — 백엔드 설계 진행 중 |
| Layer 1 (Welcome 10%) | 설계 완료 |
| Layer 2 (주간 카테고리 쿠폰) | 설계 완료 |
| Layer 3 (조건부 쿠폰) | Phase 1 후반 추가 예정 |
| 리스크 | 쿠폰 시스템 인프라 구축이 선행되어야 함 |

**Marketer**: T3의 웹→앱 전환 배너 문구는 마케팅팀에서 A/B 시안을 준비 중입니다. "앱에서만 받을 수 있는 혜택"과 "앱으로 열면 10% 할인" 두 가지 방향으로요.

**PO**: 좋습니다. T3는 Layer 1+2를 먼저 출시하고, T1/T2 효과가 확인되면 Layer 3(조건부 쿠폰: 좋아요 3개+, 플래너 스팟 추가)를 붙이는 순서로 갑니다. **Layer 3가 삼각 파이프라인의 연결고리**이니까, T1/T2가 작동하는 걸 확인한 후에 연결해야 의미가 있습니다.

## 3. 4/7 Go/No-Go 준비

### 3-1. 판단 기준 재점검

**PO**: Wrap-Up(3/10)에서 합의한 Go/No-Go 기준을 재점검합시다.

| # | 기준 | 현재 상태 | 4/7 판단 가능성 |
|---|------|----------|---------------|
| 1 | 플래너 진입률 0.5%+ | T1 배포 완료, 클릭율 5.21%/4.59% 확인. 실제 진입률 변화는 미확인 | **이번 주 데이터 필수** |
| 2 | T2 교차탐색율 18%+ | T2 미배포 | **4/7 시점에 판단 불가** |
| 3 | T1 신규 플래너 유저 D+1 재방문률 ≈ 기존 유저 | 코호트 수집 중 | **초기 시그널 수준** |
| 4 | 삼각 파이프라인 작동 여부 | T2/T3 미배포 | **4/7 시점에 판단 불가** |

**Growth Manager**: 솔직히 말하면, **4개 기준 중 4/7에 판단 가능한 건 1번과 3번뿐**입니다. T2가 아직 배포 전이니 2번과 4번은 물리적으로 불가능해요.

**PO**: 맞습니다. 이건 3/10에 타임라인을 설정할 때 T2가 Week 2(3/18~3/24)에 배포될 것을 전제로 한 기준이었는데, 실제로는 일정이 밀렸습니다. 현실적으로 **4/7 Go/No-Go의 범위를 조정**해야 합니다.

### 3-2. 조정된 Go/No-Go 프레임

**PO**: 제안합니다. 4/7은 **"Full Go/No-Go"가 아니라 "T1 중간 판정 + Phase 1 전체 일정 재확인"**으로 범위를 조정합시다.

| 4/7 판단 항목 | 기준 | Go 조건 |
|--------------|------|---------|
| T1 플래너 진입률 변화 | T1 배포 전후 비교 | 상승 추세 확인 (0.5% 미달이어도 방향성이면 Go) |
| T1 신규 플래너 유저 품질 | D+1 재방문률 | 기존 유저 대비 50% 이상 |
| T2/T3 일정 | 배포 예정일 확정 여부 | 구체적 일정 커밋 |

**Growth Manager**: 동의합니다. 원래 기준을 **"4/7에 완전한 판단"에서 "4/7에 방향성 확인, 4/21에 Full 판정"**으로 2단계로 나누는 거죠.

**PM**: 그러면 4/21 판정에는 T2 초기 데이터(2주분)가 포함될 수 있으니 더 의미 있는 판단이 가능합니다.

**Marketer**: 한 가지 추가합니다. T1의 클릭율이 5% 수준이라는 건, **"클릭은 하지만 실제 플랜 생성까지 가는지"**가 핵심 질문이에요. 4/7까지 이 퍼널 드롭오프를 봐야 합니다. 클릭율 5%인데 실제 플랜 생성이 1%라면, 그건 UI 문구나 플래너 온보딩에 문제가 있다는 거고, 3~4%면 건강한 수준이에요.

**PO**: 정리하겠습니다.

| 일정 | 판정 | 범위 |
|------|------|------|
| **4/7** | T1 중간 판정 | 플래너 진입률 방향성 + 신규 유저 품질 초기 시그널 + T2/T3 일정 확정 |
| **4/21** | Phase 1 Full 판정 | T1 4주 데이터 + T2 초기 데이터 + 파이프라인 시너지 초기 신호 |

## 4. 삼각 파이프라인 현황 정리

**PO**: 현재 삼각 파이프라인의 전체 상태를 시각화합니다.

```
T3 (앱 전용 쿠폰: "앱으로 와") ──── 진행 예정 (설계 중)
     ↓ 주간 쿠폰으로 카테고리별 탐색 유도
T2 (럭키드로우×교차탐색: "3개 더 보면 보상") ──── 진행 예정 (스펙 확정)
     ↓ 탐색한 스팟이 좋아요/플래너 모수가 됨
T1 (좋아요→플랜 추가: "일정에 넣어볼까?") ──── ✅ 배포 완료
     ↓ 플래너 Open Loop                         클릭율 5.21% / 4.59%
RETURN_D1 (다음 날 재방문)
```

**핵심 인사이트**: 파이프라인의 **출구(T1)가 먼저 배포**되었고, 입구(T3)와 중간(T2)이 아직 열리지 않은 상태입니다. 현재 T1의 5% 클릭율은 **기존 오가닉 좋아요 유저(일 ~108명)만으로** 만들어진 수치입니다. T2/T3가 배포되면 좋아요 모수 자체가 커지면서 T1의 절대 전환 수도 증가할 것으로 기대합니다.
