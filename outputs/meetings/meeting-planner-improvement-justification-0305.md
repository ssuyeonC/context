# 플래너 기능 개선 당위성 검토 회의록

**일시**: 2026-03-05
**참석자**: PO (제품 방향/전략 총괄), PM (세부 기획), Growth Manager (데이터/전환 판단), Marketer (고객 접점/커뮤니케이션)
**목적**: 2차 RETURN_D1 전략 회의에서 "플래너는 왕관이지 유일한 무기가 아니다"로 합의했으나, 중장기적으로 플래너 개선에 투자하는 것이 정당한지 별도 검토
**참고 자료**: 2차 회의록, 유저 행동 레이어 모델, 좋아요 vs 플래너 비교 분석, 플랜 생성 CSV(2/2~3/3), r/koreatravel 10,000건 분석, DAU 모델 분석

## 1. 왜 이 회의가 필요한가

**PO**: 2차 회의에서 플래너의 임팩트 순위가 **7개 행동 중 꼴찌**(리프트×모수 = 0.5)라는 결론이 나왔습니다. 교차탐색(19.0), 앱 활동(17.4)과 비교하면 차이가 압도적이에요. 그래서 "다채널 Open Loop 포트폴리오"로 전략을 수정했고, 플래너를 유일한 무기로 보지 않기로 했습니다.

그런데 동시에 우리는 Phase 1에 **플래너 진입점 확대, 일정 검증 라이트, 멀티시티 템플릿, 좋아요→플래너 브릿지**를 포함시켰어요. 모순이 있습니다. **"임팩트 꼴찌인 기능에 Phase 1 리소스의 상당 부분을 투자하는 게 합리적인가?"** — 이걸 정면으로 검토하지 않으면 실행 단계에서 우선순위 충돌이 생깁니다.

**Growth Manager**: 맞습니다. 솔직히 2차 회의에서 플래너에 대한 두 가지 상반된 결론이 공존하고 있어요.

1. **"단기 임팩트 최하위, 플래너에 올인하면 안 된다"**
2. **"Lock-in 최고, 장기적으로 플래너는 가장 강력한 무기다"**

이 두 명제가 모순이 아니라 **시간축이 다른 같은 결론**인지, 아니면 **진짜 모순**인지 정리해야 합니다.

**Marketer**: 제가 한 가지 더 촉발점을 던지겠습니다. 유저의 여행 준비 과정을 레이어 모델로 정리해봤는데, **플래너가 차지하는 위치를 보면 "임팩트 꼴찌"라는 결론이 맞는 동시에 "가장 중요한 기능"이라는 결론도 성립합니다.** 이걸 먼저 공유하고 논의하죠.

## 2. 유저 행동 레이어 모델 — 플래너의 구조적 위치

**Marketer**: 유저가 크리에이트립에서 거치는 행동을 레이어로 분해했습니다.

```
Layer 0: 방문 — 오가닉/SNS 유입, 단일 콘텐츠 소비
   ↓ 교차탐색 유도 (추천 영역, 검색 최적화)
Layer 1: 탐색 — 상품 2개+ 비교, 검색, 카테고리 탐색
   │        리텐션 1.63x · 액션율 11.64%
   ↓ 저장 동기 발생 (소셜프루프, 매력도)
Layer 2: 저장 — 좋아요/찜으로 "내 리스트" 형성
   │        리텐션 2.17x · 액션율 1.00%
   ↓ "일정에 넣어볼까요?" ⚠️ 높은 마찰 구간
Layer 3: 계획 — 플래너에 일정 배치, 빈 날짜 Open Loop
   │        리텐션 1.72x · 액션율 0.27%
   ↓
Layer 4: 구매 + 실행 — 여행 중 플랜 참고, 추가 구매
   ↓
Layer 5: 공유 + 재방문 — 일정 공유, 다음 여행 시 재진입
```

핵심은 **구매(Layer 4)가 어느 레이어에서든 발생**할 수 있다는 점이에요. Layer 1에서 바로 구매하는 유저도 있고, Layer 2에서 찜 → 구매로 가는 유저도 있어요. 그래서 단기 구매 전환만 보면 플래너가 불필요해 보입니다.

**하지만 Layer 4 이후의 분기가 결정적입니다.**

```
Layer 4 (구매 + 실행)
  ├─ [플래너에 맥락이 있는 유저] → Layer 5 (공유 + 재방문) → 다음 여행 시 L0 재진입
  └─ [플랫폼에 남은 맥락이 없는 유저] → 이탈 → 다음 여행 시 Klook·KKday·검색으로 진입
```

**Layer 3(플래너)를 거친 유저와 거치지 않은 유저의 결정적 차이는 "구매 후"에 나타납니다.** 플래너에 일정이 있는 유저는 여행 중에 크리에이트립에 들어올 이유가 있어요 — "내 일정 확인", "다음 스팟 확인", "빈 시간에 뭐 하지?". 플래너가 없는 유저는 구매한 상품의 바우처만 확인하고 끝입니다.

**Growth Manager**: 이 레이어 모델의 가장 중요한 시사점이 뭔지 짚어보겠습니다. **"구매 후 이탈" vs "구매 후 재방문"을 가르는 결정적 행동이 Layer 3(플래너)**라는 거예요. Layer 1(탐색)이나 Layer 2(좋아요)에서 구매한 유저는 트랜잭션 관계만 형성합니다. "좋은 상품 샀다, 끝." 다음 여행 때 또 크리에이트립을 쓸 이유가 없어요.

하지만 Layer 3(플래너)를 거친 유저는 **"내 여행 맥락"이 크리에이트립에 축적**돼 있습니다. 이건 트랜잭션이 아니라 **관계**예요. 다음 여행 때 "아, 저번에 크리에이트립에서 일정 짰었지"로 돌아올 확률이 높아집니다.

**PM**: 잠깐, 그런데 이 "관계"의 강도가 정말 유의미한가요? 현실적으로 한국 여행을 반복하는 외국인이 전체의 몇 %인지가 중요합니다. 대부분의 유저가 한국 여행을 한 번만 한다면, Layer 5(다음 여행 재진입)의 모수 자체가 작아서 플래너의 Lock-in 가치가 과대평가될 수 있어요.

**Growth Manager**: 핵심적인 챌린지입니다. DAU 모델 데이터를 보면, 전체 방문자 중 **RESURRECTED(30일 이상 미방문 후 복귀) 버킷이 존재한다**는 것 자체가 재방문이 일어나고 있다는 증거예요. 하지만 **연간 데이터에서 RETURN_D1 + RETURN_D7 + RETURN_D30의 합산이 전체의 약 40%**를 차지합니다. 이 유저들은 "한 번만 왔다 가는" 유저가 아니에요. 그리고 이 재방문 유저의 구매 전환율이 NEW USER보다 2~3배 높다는 것도 이미 확인했고요.

**PO**: 좋습니다. 그러면 플래너의 가치를 두 가지 시간축으로 분리해서 보겠습니다.

| | 단기 (이번 여행 내) | 장기 (다음 여행) |
|---|---|---|
| **플래너 없이 구매** | 트랜잭션 완료, 이탈 | 다른 플랫폼으로 진입 가능성 높음 |
| **플래너 있이 구매** | 여행 중 추가 구매 발생 가능 | 다음 여행 시 크리에이트립 재진입 확률 상승 |

**Marketer**: PO님이 정리한 표에서 한 가지 더 추가해야 할 셀이 있습니다. **"플래너 있이 구매하지 않는 유저"**예요. 플래너에 스팟을 추가했지만 아직 구매하지 않은 유저 — 이 유저에게 플래너의 빈 날짜, 미예약 스팟이 **구매를 유도하는 Open Loop**로 작동합니다. "아직 이 스팟 예약 안 했는데"라는 미완결 상태가 구매 트리거가 되는 거죠.

**PM**: 정리하면, 플래너의 가치는 **세 가지 차원**에 걸쳐 있습니다.

1. **여행 전**: 빈 날짜 + 미예약 스팟 → 재방문 + 구매 유도 (Open Loop)
2. **여행 중**: "내 일정" 참조 → 서비스 이용 지속 + 추가 구매
3. **여행 후**: 맥락이 축적된 플랫폼 → 다음 여행 재진입 (Lock-in)

리프트×모수 순위표에서는 1번만 측정됩니다. 2번과 3번은 현재 우리 측정 프레임에 포함되어 있지 않아요.

## 3. 핵심 행동 분석 — Layer별 게이트

**PO**: 레이어 모델을 기반으로, 각 전환 구간의 "게이트(관문)"를 정리해봅시다. 이것이 우리가 어디에 투자해야 하는지 결정하는 프레임입니다.

### 게이트 1: L0→L1 — "방문자"를 "탐색자"로 (상품 2개+ 탐색)

**Growth Manager**: 이건 2차 회의에서 **임팩트 1위**로 확인된 구간입니다.

- **액션율 11.64%** — 모수가 가장 큼
- 리텐션 리프트 1.63x, 구매 전환 리프트 3.83x
- 추천 탭 클릭, 상품카드 클릭, 카테고리 탐색 → 모두 이 게이트를 열기 위한 수단
- **Phase 0~1에서 이미 교차탐색 루프 강화를 최우선으로 잡았고, 럭키드로우 연동까지 설계 완료**

현재 전략에서 가장 잘 커버되고 있는 구간이에요. 추가 논의 불필요합니다.

### 게이트 2: L1→L2 — "탐색자"를 "저장자"로 (좋아요 클릭)

**Growth Manager**: 리텐션 리프트 최고(2.17x), 구매 전환 리프트 최고(6.62x)인 구간입니다.

- **액션율 1.00%** — 탐색자 대비 91% 이상이 저장하지 않고 이탈
- "탐색"을 "축적"으로 바꾸는 순간 — **유저의 데이터가 플랫폼에 남기 시작하는 첫 지점**
- 이 행동이 일어나야 "지나가는 방문자"가 "자기 데이터가 있는 유저"가 됨

**Marketer**: 좋아요의 심리적 기제를 정확히 짚어야 합니다. 좋아요는 **"발산"** 행동이에요. 선택지를 넓히는 중입니다. "이것도 괜찮나?"라는 질문에 대한 답이죠. 아이템 간에 관계가 없고, 각각이 독립적인 후보입니다.

**PM**: 좋아요 리스트 재조회(#11)도 중요한 숨은 지표입니다. 좋아요를 눌렀다고 끝이 아니라, **다시 돌아와서 리스트를 보는 행동**이 실제 리텐션이에요. CRM("찜한 상품 중 남산타워가 인기에요")의 직접적 랜딩 포인트이기도 하고, Layer 2에서의 Open Loop가 실제로 작동하는지 확인하는 지표입니다.

### 게이트 3: L2→L3 — "저장자"를 "계획자"로 (플래너에 스팟 추가) ⚠️

**PO**: 이 회의의 핵심 구간입니다.

**Growth Manager**: 숫자부터 보겠습니다.

| 지표 | 값 |
|------|-----|
| 리텐션 리프트 | 1.72x |
| 구매 전환 리프트 | 3.49x |
| 액션율 | **0.27%** |
| 일 플랜 생성 | **33~53건** (2/2~3/3 데이터, 평균 ~34건) |
| 리프트×모수 순위 | **7위 (최하위)** |

**그리고 여기가 가장 높은 마찰 구간입니다.** 좋아요(L2)에서 플래너(L3)로 넘어가려면 유저에게 근본적으로 다른 것을 요구해요.

**Marketer**: 이 마찰의 본질을 설명하겠습니다.

| | **좋아요 리스트** (L2) | **플래너** (L3) |
|---|---|---|
| 심리 기제 | **발산** — 선택지를 넓히는 중 | **수렴** — 선택지를 좁히는 중 |
| 유저의 질문 | "이것도 괜찮나?" | "이걸 언제 하지?" |
| 담고 있는 것 | 후보들 (비교 대상) | 결정된 것들 (실행 계획) |
| 아이템 간 관계 | 없음 (각각 독립) | 있음 (순서, 동선, 시간) |
| 여행 시점 | 여행 전 초기 (몇 주 전) | 여행 전 후기 ~ 여행 중 |
| Open Loop | "뭐가 더 있을까?" | "빈 날짜에 뭘 넣지?" |
| 재방문 트리거 | "후보 더 찾아야지" | "계획 완성해야지" |

**PO**: 이 비교표에서 핵심 인사이트가 보입니다. **좋아요와 플래너는 "같은 기능의 강도 차이"가 아니라 "완전히 다른 심리 단계"**입니다. 발산에서 수렴으로의 전환은 유저에게 인지적 비용이 큽니다.

"이것도 좋아, 저것도 좋아" (발산) → "자, 이제 골라야 해. 언제 갈 거야?" (수렴)

이 전환이 자연스럽게 일어나지 않는 이유가 **액션율 0.27%**로 나타나는 거예요.

**PM**: 더 구체적으로 말하면, 플래너 진입의 전제 조건이 세 가지입니다.

1. **여행 날짜가 확정되어야 함** — "언제 가는지 모르겠는데 일정 짜기"는 불가능
2. **최소한의 선택이 완료되어야 함** — "아직 뭘 할지 모르겠는데 일정에 넣기"도 어색
3. **플래너라는 기능의 존재를 알아야 함** — 크리에이트립에 일정 기능이 있다는 인지

세 가지 중 하나라도 충족되지 않으면 L2→L3 전환이 일어나지 않습니다. 그리고 **우리의 주요 유저 대부분은 3번이 충족되지 않는 상태**예요. 플래너 인지도 자체가 낮습니다.

## 4. 플래너를 "개선해야 하는가"에 대한 정면 토론

**PO**: 이제 핵심 논점을 다루겠습니다. **"플래너 개선에 투자하는 것이 정당한가?"** 찬반 양쪽 모두 진지하게 검토합시다.

### 반대 논거: "플래너 개선은 당장 불필요하다"

**Growth Manager**: 제가 먼저 반대 논거를 정리하겠습니다. 솔직하게요.

**논거 1: 임팩트 대비 리소스 비효율**

리프트×모수 순위에서 꼴찌입니다. 같은 개발 리소스를 교차탐색 루프(임팩트 1위), 좋아요 활용(4위), 비교하기(신규)에 투자하는 게 **RETURN_D1에 대한 단기 ROI가 압도적으로 높아요.** 플래너 관련 Phase 1 액션들(진입점 확대, 일정 검증 라이트, 템플릿)의 개발 규모를 합산하면 **최소 2~3주**입니다. 이 시간에 교차탐색 + 비교하기 + CRM 자동화를 완료하는 게 RETURN_D1 숫자에 더 빠르게 기여합니다.

**논거 2: 인과관계 불확실**

2차 회의에서 마케터가 정확히 지적했습니다. **"플래너를 써서 리텐션이 높은 건지, 이미 여행 확정+구매 의도 높은 유저가 플래너까지 도달한 건지"** 구분할 수 없어요. 만약 후자라면, 플래너 진입점을 아무리 넓혀도 "원래 플래너를 안 쓸 유저"를 플래너에 넣는 건 리텐션에 효과가 없습니다. 수치만 올라가고 실질은 변하지 않아요.

**논거 3: 대안 도구 존재**

2차 회의에서도 언급했지만, 구글 스프레드시트, 노션, TripAdvisor, 종이 노트로 일정을 짜는 유저가 많아요. r/koreatravel에서도 여행 계획을 올리는 유저 대부분이 구글 독스나 이미지로 일정을 정리합니다. **"크리에이트립 플래너가 아니면 안 되는 이유"가 현재 명확하지 않아요.**

**논거 4: 여행 플랫폼의 본질적 한계**

듀오링고는 매일 학습이라는 반복 행동이 있으니까 스트릭이 작동해요. 하지만 여행은 1년에 1~2번이에요. 플래너에 아무리 좋은 Open Loop를 만들어도, **여행 전 2~4주 + 여행 중 3~7일**이 전부입니다. 그 이후는 플래너의 가치가 0에 수렴해요. 장기 Lock-in이라고 했지만, **다음 여행까지 6~12개월의 공백 동안 플래너의 맥락이 유지되는지**가 불확실합니다.

### 찬성 논거: "플래너 개선은 반드시 필요하다"

**Marketer**: 반대 논거가 강력하지만, 제가 하나씩 반박하겠습니다.

**반박 1: "단기 임팩트 최하위"는 "현재 상태의 플래너"에 대한 평가**

임팩트 순위가 꼴찌인 건 **모수가 0.27%이기 때문**이에요. 리프트 자체는 1.72x로 중간 이상입니다. 모수가 낮은 이유는 **기능이 나쁘기 때문이 아니라 진입점이 부족하고 인지도가 낮기 때문**일 수 있어요. 좋아요도 액션율이 1%이지만 리텐션 리프트 최고(2.17x)잖아요. "액션율이 낮다 = 기능이 불필요하다"는 등치가 아닙니다.

만약 플래너 진입률을 0.27%에서 1%로 올리면 임팩트 순위가 어떻게 바뀌나요?

**Growth Manager**: 계산해보겠습니다.

| 행동 | 리프트 | 액션율 (현재) | 임팩트 (현재) | 액션율 (목표) | 임팩트 (목표) |
|------|--------|-------------|------------|-------------|------------|
| 상품 2개+ 탐색 | 1.63 | 11.64% | 19.0 | — | 19.0 |
| 앱 활동 | 1.81 | 9.60% | 17.4 | — | 17.4 |
| 검색 클릭 | 1.72 | 4.67% | 8.0 | — | 8.0 |
| 상품카드 클릭 | 1.76 | 1.67% | 2.9 | — | 2.9 |
| 좋아요 | 2.17 | 1.00% | 2.2 | — | 2.2 |
| 럭키드로우 | 2.13 | 0.41% | 0.9 | — | 0.9 |
| **플래너** | **1.72** | **0.27%** | **0.5** | **1.0%** | **1.7** |

... 1%로 올리면 임팩트 0.5 → 1.7로 3.4배 상승. 좋아요(2.2)에 근접합니다. **5위에서 전체 균형이 맞아져요.** 하지만 여전히 교차탐색(19.0)이나 앱 활동(17.4)과는 격차가 큽니다.

**Marketer**: 하지만 이 임팩트 표에는 **Layer 4~5의 가치가 포함되어 있지 않아요.** 구매 후 재방문, 다음 여행 재진입은 측정하지 않고 있습니다. 교차탐색의 임팩트 19.0은 "이번 여행에서의 D+1 재방문"까지만 측정한 거예요. 플래너의 진짜 가치는 이 표에 반영되지 않습니다.

**반박 2: 인과관계 불확실성은 모든 행동에 동일하게 적용**

플래너만의 문제가 아닙니다. **좋아요도 마찬가지예요.** "좋아요를 눌러서 리텐션이 높은 건지, 관심이 높은 유저가 좋아요까지 누르는 건지" — 이건 로지스틱 회귀에서 다른 변수를 통제했을 때 플래너의 승산비가 유의미했다는 것으로 부분적으로 해소됩니다. **완벽한 인과 추론은 A/B 테스트로만 가능하고, 그건 기능을 만들어야 할 수 있어요.** 인과관계 불확실성을 이유로 기능 개선을 안 하면, 검증할 기회 자체가 사라집니다.

**반박 3: "크리에이트립 플래너가 아니면 안 되는 이유"를 만들면 됨**

구글 스프레드시트나 노션에서 일정을 짜는 유저가 많은 건 사실이에요. 하지만 이 도구들은 **"일정을 적는 것"만 해주지, "일정을 검증해주지는 않습니다."** Reddit 데이터가 이걸 명확히 보여줬어요.

- "내 계획 괜찮아?" (검증 요청): 19.6%
- "이게 현실적이야?" (가능성 확인): 14.7%
- **합산 34.3%가 "검증"을 원함 — "추천"(24.6%)보다 10pp 높음**

구글 스프레드시트는 "3일차에 5개 스팟이 빡빡한지" 알려주지 않아요. Reddit에 올려야 아는 거예요. **크리에이트립 플래너가 이동 시간 계산 + 빡빡함 경고를 해주면, 이건 구글 스프레드시트가 할 수 없는 일**입니다. "크리에이트립이 아니면 안 되는 이유"가 생기는 거죠.

**PM**: 반박 4도 추가하겠습니다. **"여행 전 2~4주가 전부"라는 한계는 사실이지만, 그 2~4주가 구매의 90%가 일어나는 구간이에요.** 리텐션이 6개월 지속될 필요는 없습니다. 여행 전 2~4주 동안 매일 또는 2~3일에 한 번 돌아오게 만드는 게 핵심이에요. 그 기간 동안 플래너의 "빈 날짜 Open Loop"가 작동하면, RETURN_D1 모수에 직접 기여합니다.

## 5. 레이어 모델에서 "플래너가 없으면 생기는 구멍"

**PO**: 양쪽 논거를 들었습니다. 이제 좀 더 구조적으로 생각해보겠습니다. **"만약 플래너 개선을 하지 않으면 어떻게 되는가?"**를 레이어 모델에서 추적해봅시다.

**Growth Manager**: 교차탐색 루프, 좋아요, 비교하기, 럭키드로우, CRM — 이것들이 모두 잘 작동한다고 가정합시다. 유저 여정은 이렇게 됩니다.

```
L0 (방문) → L1 (탐색: 2개+ 상품 비교) → L2 (좋아요/비교함에 저장)
  ↓
구매 (L1 또는 L2에서 바로)
  ↓
여행 중: 바우처 확인 → 이 상품만 사용 → 끝
  ↓
여행 후: 크리에이트립에 남은 맥락 없음 → 다음 여행 시 Klook/KKday/구글 검색으로 진입
```

**핵심 문제: L3(계획)이 비어있으면, 유저에게 "크리에이트립에서만 볼 수 있는 것"이 없어요.** 좋아요 리스트? 솔직히 다음 여행 때 좋아요 리스트를 보러 오는 유저가 얼마나 될까요. 비교함? 비교 결정은 이미 끝났을 거예요.

**Marketer**: 더 구체적으로 말하면, **경쟁사 대비 차별화의 관점**에서 문제가 생깁니다.

| 기능 | 크리에이트립 | Klook | KKday |
|------|------------|-------|-------|
| 상품 탐색 | O | O | O |
| 좋아요/찜 | O | O | O |
| 검색 | O | O | O |
| 리뷰 | O | O | O |
| **일정 계획 + 검증** | **O (유일)** | X | X |
| **동선 지도** | **O (유일)** | X | X |

교차탐색, 좋아요, 검색 최적화 — 이것들은 모두 **Klook과 KKday도 할 수 있는 것**이에요. 우리가 아무리 잘 해도 "크리에이트립만의 경험"이 아닙니다. 하지만 **플래너(일정 계획 + 검증 + 동선 지도)는 Klook/KKday에 없는 기능**이에요. 이것이 "크리에이트립에서만 닫을 수 있는 Open Loop"의 가장 명확한 구현입니다.

**PM**: 정리하면 이렇습니다.

> **교차탐색·좋아요·럭키드로우는 "유저를 크리에이트립에 머물게 하는" 장치.**
> **플래너는 "유저를 크리에이트립에 가두는" 장치.**

전자는 단기 리텐션(이번 방문, 내일 재방문)에 기여하고, 후자는 장기 Lock-in(여행 전 전체 기간, 여행 중, 다음 여행)에 기여합니다. 둘 다 필요해요. 하나가 다른 하나를 대체하지 못합니다.

**PO**: 동의합니다. 그런데 **"둘 다 필요하다"는 걸 인정한다고 해서 "둘 다 지금 해야 한다"는 결론이 되지는 않아요.** 우선순위의 문제입니다.

**Growth Manager**: 구체적 데이터를 보겠습니다. 2/2~3/3 기간 플랜 생성 추이입니다.

**Marketer**: 이건 봄 여행 시즌 진입 시그널과 정확히 맞물립니다. 3월에 여행 방문자가 14,159명으로 급증한 것과 같은 시점이에요. **시즌 트래픽이 올라올 때 플래너 사용도 같이 올라간다**는 건, 플래너 수요 자체는 존재한다는 증거입니다.

**Growth Manager**: 하지만 비율로 보면 달라요. 3/3 기준 여행 방문자 14,159명 중 플랜 생성 53건이면 **0.37%**입니다. 2월 평균 0.27%에서 약간 올랐지만, 여전히 1% 미만이에요. **절대값은 올랐지만 비율은 거의 변화 없음** — 시즌 효과로 모수가 커진 것이지, 플래너 자체의 매력이 올라간 건 아닙니다.

**PM**: 이게 바로 **진입점 부족 + 인지도 부족 + 마찰**의 문제라는 거예요. 시즌에 여행 의도가 높아지면 플래너를 쓸 유저도 늘어나지만, **플래너의 존재를 알고 있는 유저 풀이 제한적**이라 비율은 변하지 않는 겁니다.

**PO**: 역으로 말하면, **진입점을 확대하고 인지도를 높이면 비율 자체가 올라갈 수 있다**는 뜻이기도 합니다. 시즌 타이밍에 맞춰 진입점 확대를 하면 절대값과 비율 모두 올릴 수 있어요.

## 7. "개선 방향" 합의 — 무엇을 어떤 순서로

**PO**: 여기까지의 논의를 종합하면, **"플래너 개선은 필요하되, 접근 방식이 중요하다"**로 합의가 모여가고 있습니다. 구체적으로 어떤 개선이 정당하고, 어떤 개선은 시기상조인지 분리해봅시다.

**Growth Manager**: 데이터 관점에서 플래너 개선을 세 가지 카테고리로 나누겠습니다.

### 카테고리 A: "지금 해야 하는 것" — 진입 마찰 제거

**이건 플래너 기능 자체를 개선하는 게 아니라, 기존 플래너에 도달하는 경로를 넓히는 것입니다.**

| 액션 | 근거 | 개발 규모 |
|------|------|----------|
| 검색 결과 "내 일정에 추가" 인라인 버튼 | 검색 클릭 일 950명, 임팩트 3위 모수에 진입점 배치 | Phase 0에 이미 포함 |
| 좋아요 리스트 → "찜한 X개로 플랜 만들기" | L2→L3 브릿지, Cold Start 해소 | 프론트 1~2일 |
| 블로그/가이드 본문 내 "일정에 추가" | 세그먼트 2(블로그 유입) 유저에게 진입점 제공 | 프론트 1일 |
| 플랜 없는 유저 "날짜 없이 일단 추가" 경량 온보딩 | L2→L3 마찰의 전제 조건 1(날짜 확정) 제거 | 프론트 2일 |

**이 카테고리의 핵심은 "플래너를 바꾸는 게 아니라 플래너까지의 거리를 줄이는 것"**입니다. 기존 모수(검색 950명/일, 좋아요 108명/일, 블로그 유입 다수)에 진입점만 추가하는 거라 리스크가 낮고, 효과를 빠르게 측정할 수 있어요.

**Marketer**: 동의합니다. 이 카테고리는 **Phase 1 초반에 즉시 실행**이 맞아요. 플래너 자체를 건드리는 게 아니니까 개발 비용도 낮고, 진입률 변화를 바로 측정할 수 있습니다. 0.27%가 0.5%만 돼도 성공입니다.

### 카테고리 B: "검증 후 해야 하는 것" — 플래너 내 경험 개선

**PM**: 이건 플래너 안에 들어온 유저의 경험을 개선하는 것입니다.

| 액션 | 근거 | 개발 규모 | 전제 조건 |
|------|------|----------|----------|
| 스팟 추가 후 "근처 추천" 표시 | 교차탐색 트리거, L3 내에서 L1 행동 유도 | 프론트 1~2일 | 카테고리 A로 진입률 상승 확인 후 |
| 빈 날짜 시각적 표시 | Open Loop 강화 | 프론트 1일 | 카테고리 A로 진입률 상승 확인 후 |
| 2개+ 스팟 추가 시 지도 핀 자동 표시 | 경쟁사 대비 차별 가치, 동선 시각화 | 프론트 2~3일 | 카테고리 A로 진입률 상승 확인 후 |
| 멀티시티 템플릿 (서울+부산 3~4일) | Reddit 최대 수요 패턴, Cold Start 해소 | 콘텐츠 제작 + 프론트 2일 | 봄 시즌 타이밍 |
| "일정 검증" 라이트 (이동 시간 + 빡빡함 경고) | Reddit 검증 니즈 34.3%, 경쟁사 불가 기능 | 프론트+백엔드 3~5일 | 카테고리 A로 진입률 상승 확인 후 |

**Growth Manager**: 여기서 제가 한 가지 중요한 순서 제안을 하겠습니다. **"일정 검증" 라이트와 멀티시티 템플릿은 카테고리 A와 동시 실행이 가능합니다.** 이유를 설명하면:

"일정 검증"은 **플래너의 포지셔닝을 바꾸는 기능**이에요. 현재 플래너는 "일정 짜는 도구"인데, 일정 검증이 추가되면 **"일정 검증 도구"**로 리프레이밍됩니다. 이 리프레이밍이 중요한 이유는, 카테고리 A의 진입점 확대와 함께 **"왜 크리에이트립 플래너를 써야 하는지"에 대한 답**을 제공하기 때문입니다.

진입점만 넓히고 플래너 안이 비어있으면, 유저가 들어와도 "이게 구글 스프레드시트와 뭐가 다르지?"로 끝나요. 하지만 **이동 시간 자동 계산 + "3일차에 5개는 빡빡합니다" 피드백**이 있으면, "아, 이건 여기서만 해주는 거네"가 됩니다.

**Marketer**: 동의합니다. 그리고 멀티시티 템플릿은 **봄 시즌에 맞춰야 하는 시간 제약**이 있어요. 4월 중순이 지나면 벚꽃 시즌이 끝나기 때문에, 3월 중에 "서울+부산 3~4일" 템플릿이 준비되어야 합니다. 이건 카테고리 A 결과를 기다릴 시간이 없어요.

**PM**: 그러면 수정합시다. **"일정 검증" 라이트와 멀티시티 템플릿은 카테고리 A와 병행 실행**, 나머지 카테고리 B(근처 추천, 빈 날짜 표시, 지도 핀)는 **카테고리 A 효과 검증 후 실행**.

### 카테고리 C: "확실한 데이터가 쌓인 후" — 중장기 투자

| 액션 | 근거 | 전제 조건 |
|------|------|----------|
| "여행 준비도 스코어" | 프로그레스 바로 Open Loop 시각화 | 플래너 일 사용자 100명+ 달성 후 |
| "일정 검증" 풀 스펙 (휴무일, 영업시간, 맥락 피드백) | Reddit "itinerary check" 완전 대체 | 라이트 버전 효과 검증 후 |
| 플래너 오프라인 캐싱 (앱) | 여행 중 사용성 | 앱 전환 전략 효과 확인 후 |
| 동선 최적화 추천 (거리→맥락 기반) | Lock-in 극대화 | 지도 핀 기능 사용 데이터 확인 후 |

**PO**: 이 카테고리는 **플래너가 실제로 성장하고 있다는 데이터가 있을 때만 투자**합니다. 카테고리 A+B로 플래너 진입률이 1%를 넘지 못하면, 카테고리 C는 보류하고 교차탐색·좋아요 루프에 리소스를 집중합니다.

## 8. "투자 정당성" 최종 판단 기준

**Growth Manager**: 플래너 개선의 당위를 판단하는 **정량 기준**을 설정합시다. 감에 의한 판단을 막으려면 숫자가 필요합니다.

### Go/No-Go 기준

**카테고리 A (진입점 확대) 실행 후 4주 시점에 평가:**

| 지표 | Go 기준 | No-Go 시 대응 |
|------|---------|-------------|
| 플래너 진입률 | 0.27% → **0.5%+** | 진입점 추가 확대 또는 UX 개선 검토 |
| 일 플랜 생성 수 | ~34건 → **50건+** (시즌 보정) | 시즌 효과 제거 후 순수 진입점 효과 재분석 |
| 플래너 유저 D+1 재방문률 | 베이스라인 대비 | **+10% 이상이면 Go** |
| "내 일정에 추가" 버튼 CTR | 측정 시작 | 0.5% 미만이면 버튼 위치/문구 재설계 |

**PM**: 추가 기준을 제안합니다. **"플래너 진입 경로별 D+1 재방문률"을 분리 트래킹**해야 합니다.

- 검색 결과 → 플래너 진입 유저
- 좋아요 리스트 → 플래너 진입 유저
- 블로그 → 플래너 진입 유저
- 직접 플래너 진입 유저

경로별로 재방문률이 다를 거예요. **어떤 경로에서 온 유저가 실제 리텐션 효과가 있는지**를 알면, 카테고리 B에서 어디에 집중해야 하는지 결정할 수 있습니다.

**Growth Manager**: 그리고 가장 중요한 인과관계 검증. **카테고리 A로 진입점을 넓혀서 유입된 "신규 플래너 유저"의 D+1 재방문률이 "기존 플래너 유저"와 비슷한 수준인지를 봐야 합니다.**

- 만약 **비슷하면**: 플래너 사용 자체가 리텐션에 기여하는 증거 → 카테고리 B 투자 정당화
- 만약 **낮으면**: 기존 플래너 유저는 원래 의도가 높은 유저였을 뿐, 플래너 때문이 아님 → 카테고리 B 축소, 교차탐색에 집중

**PO**: 이건 인과 추론의 자연 실험(Natural Experiment) 설계예요. 진입점 확대가 **"기존에 플래너를 몰랐던 유저"를 플래너에 넣는 도구**로 작동하니까, 이 유저의 리텐션 변화를 보면 플래너의 인과 효과를 추정할 수 있습니다. 좋습니다.

## 9. "일정 검증" — 플래너의 게임 체인저 가능성

**Marketer**: 카테고리 B에서 한 가지만 깊게 논의하고 싶습니다. **"일정 검증" 기능이 플래너의 포지셔닝을 완전히 바꿀 수 있는 기능**이라고 생각해요.

2차 회의에서 Reddit 데이터가 이걸 보여줬습니다.

| 질문 유형 | 건수 | 비중 |
|----------|------|------|
| "추천해줘" | 2,052 | 24.6% |
| **"내 계획 괜찮아?" (검증 요청)** | **1,640** | **19.6%** |
| **"이게 현실적이야?" (가능성 확인)** | **1,228** | **14.7%** |
| "A vs B 어떤 게 나아?" (비교) | 953 | 11.4% |

**검증 + 가능성 확인 합산 34.3%** — 이건 유저의 1/3이 "내 계획을 검증받고 싶다"는 거예요. 현재 이 니즈를 해소하는 방법은 **Reddit에 올리기**밖에 없습니다. 만약 크리에이트립 플래너가 이걸 자동으로 해주면?

**Growth Manager**: 구체적으로 어떤 수준의 검증이 가능한지가 중요합니다.

| 검증 유형 | 라이트 (Phase 1) | 풀 스펙 (Phase 2) |
|----------|-----------------|------------------|
| 일정 빡빡함 경고 | "3일차 5개 스팟 → 과밀" | 스팟 간 이동 시간 고려 세밀 판단 |
| 이동 시간 | 스팟 간 직선거리 기반 추정 | 지도 API 기반 정확한 대중교통 소요 시간 |
| 휴무일 경고 | X | "이 스팟은 월요일 휴무" |
| 영업시간 체크 | X | "이 스팟은 18시에 마감, 17시 도착 추천" |
| 맥락 피드백 | X | "경복궁→남산타워 이동 시 명동 경유 추천" |

**PM**: 라이트 버전(Phase 1)만으로도 **"빡빡함 경고 + 이동 시간 추정"은 가능**합니다. 이것만으로도 Reddit에서 가장 많이 묻는 질문 — **"Is my 3-day Seoul+Busan itinerary too packed?"** — 에 대한 답을 플래너가 해줄 수 있어요.

**Marketer**: 이게 되면 유저 플로우가 완전히 바뀝니다.

```
[현재]
유저: 일정 작성 → Reddit에 "itinerary check please" 게시 → 댓글 대기 (6~24시간)
    → 피드백 반영 → 재게시 → 반복

[일정 검증 도입 후]
유저: 크리에이트립 플래너에 일정 입력
    → 즉시 피드백: "3일차 과밀, 1개 줄이세요" / "경복궁→남산 이동 40분"
    → 수정 → 재검증 → 즉시 피드백
    → 이 수정-검증 루프 자체가 "다시 와야 하는 이유" = Open Loop
```

**PO**: "수정 → 재검증 → 수정" 루프가 반복되는 동안 유저는 계속 크리에이트립에 머물거나 돌아옵니다. **이건 플래너만이 만들 수 있는, 교차탐색이나 좋아요로는 불가능한 종류의 Open Loop**예요.

**Growth Manager**: 그리고 이 기능이 Reddit에서의 오가닉 인지도 구축에도 기여합니다. r/koreatravel에서 "itinerary check" 게시글에 **"크리에이트립 플래너에서 자동으로 검증해줘요"**라고 답변할 수 있게 되면, 2차 회의에서 확인한 **크리에이트립 인지도 위기(0.1%)**를 해소하는 오가닉 채널이 됩니다.

**PM**: 맞습니다. 2차 회의에서 "크리에이트립은 Klook=뭘 살까, Creatrip=어떻게 돌까로 포지셔닝하자"고 합의했잖아요. **"일정 검증"은 "어떻게 돌까"의 가장 구체적인 구현**입니다.

## 10. 최종 합의

**PO**: 논의를 정리합니다.

### 합의 1: 플래너 개선은 정당하되, "단계적 투자 + 데이터 기반 확대"로 접근

플래너는 **단기 임팩트는 최하위이지만, 구조적으로 대체 불가능한 기능**입니다.

- **경쟁사가 복제할 수 없는 유일한 기능** (Klook/KKday에 일정 계획 없음)
- **Layer 3 없이는 "구매 후 이탈 → 다른 플랫폼으로 유출"을 막을 수 없음**
- **"일정 검증"은 Reddit 34.3% 수요가 확인된, 명확한 시장 니즈**
- **좋아요(발산)와 플래너(수렴)는 심리적으로 완전히 다른 단계 — 하나가 다른 하나를 대체 불가**

**그러나** 현재 0.27%의 액션율에서 대규모 투자는 정당화되지 않습니다. **"진입점 확대 → 효과 검증 → 내부 경험 개선 → 효과 재검증 → 고도화"** 순서로 투자합니다.

### 합의 2: 실행 순서

```
[즉시 실행] 카테고리 A + 일정 검증 라이트 + 멀티시티 템플릿
  ├─ 진입점 확대 (검색/좋아요/블로그 "내 일정에 추가")
  ├─ "날짜 없이 일단 추가" 경량 온보딩
  ├─ "일정 검증" 라이트 (빡빡함 경고 + 이동 시간 추정)
  └─ "서울+부산 3~4일" 멀티시티 템플릿

[4주 후 Go/No-Go 판단] ← 핵심 게이트
  ├─ 플래너 진입률 0.5%+ → Go → 카테고리 B 실행
  ├─ 신규 플래너 유저 D+1 재방문률이 기존 유저와 유사 → 인과 효과 확인
  └─ 진입률 0.5% 미달 or 재방문률 차이 없음 → No-Go → 리소스 재배분

[Go 시] 카테고리 B
  ├─ 스팟 추가 후 "근처 추천" 표시
  ├─ 빈 날짜 시각적 표시
  └─ 2개+ 스팟 추가 시 지도 핀 자동 표시

[플래너 일 100명+ 달성 시] 카테고리 C
  ├─ "여행 준비도 스코어"
  ├─ "일정 검증" 풀 스펙
  ├─ 플래너 오프라인 캐싱
  └─ 동선 최적화 추천
```

> **"단기 임팩트는 교차탐색·좋아요·럭키드로우에서, 장기 Lock-in은 플래너에서. 둘 다 필요하지만 시간축이 다르다."**

> **"플래너 투자는 데이터가 허락할 때만 확대한다. 4주 후 Go/No-Go가 첫 번째 게이트."**
