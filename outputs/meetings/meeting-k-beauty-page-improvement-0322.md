# K-Beauty 페이지 개선 방향 검토 회의록

**일시**: 2026-03-22
**참석자**: PO (제품 방향/전략 총괄), PM (세부 기획), Growth Manager (데이터/전환 판단), Marketer (고객 접점/커뮤니케이션)
**목적**: K-beauty 페이지(`/k-beauty`)의 현재 퍼포먼스를 진단하고, 페이지 개선이 올바른 방향인지 판단 + 개선한다면 어떤 방향으로 진행할지 합의
**참고 자료**: K-beauty 페이지 코드베이스 분석, GA 퍼포먼스 데이터, 현재 페이지 구조

---

## 0. 오늘 회의의 초점

**PO**: 회사가 뷰티 카테고리를 전략적으로 밀고 있는 상황에서, K-beauty 페이지가 그 허브 역할을 제대로 하고 있는지 점검해야 합니다. 오늘은 세 가지 질문에 답을 내려야 해요.

| # | 질문 | 판단 기준 |
|---|------|---------|
| Q1 | K-beauty 페이지를 개선하는 게 맞는가? | 퍼포먼스 데이터, 전략적 위치 |
| Q2 | 이 페이지로의 진입을 어떻게 늘릴 것인가? | 현재 진입점 분석, 새로운 진입 경로 |
| Q3 | 페이지 내부를 어떻게 개선할 것인가? | 현재 구조의 문제점, 콘텐츠 추가/디자인 방향 |

이 세 질문이 서로 종속적입니다. Q1이 "No"라면 Q2, Q3는 무의미하고, Q2를 먼저 해결하지 않으면 Q3의 ROI가 낮아요. 순서대로 가겠습니다.

## 1. Q1 — K-beauty 페이지 개선이 맞는가?

### 1-1. 현재 퍼포먼스 진단

**Growth Manager**: 먼저 숫자를 정리합니다.

| 지표 | 값 | 해석 |
|------|-----|------|
| 메인 → K-beauty 클릭율 | **~10%** (970 중 94) | 메인에서의 진입 전환은 낮은 편 |
| K-beauty 내 스팟 클릭율 | **30~33%** | 페이지에 들어온 유저의 1/3이 스팟을 탐색 — **나쁘지 않음** |
| K-beauty 내 시술 추천 버튼 CTR | **1.4%** | 추천 기능의 발견성이 매우 낮음 |

두 가지 시사점이 보입니다.

첫째, **"페이지 자체의 전환력은 나쁘지 않다."** 들어온 유저의 30~33%가 스팟을 클릭한다는 건, 페이지 콘텐츠가 어느 정도는 유저의 관심을 이끌고 있다는 뜻이에요. 문제는 **들어오는 유저 수 자체가 적다**는 것.

둘째, **"시술 추천(Preference) 기능이 거의 활용되지 않는다."** 1.4%는 사실상 이 기능이 없는 것이나 마찬가지입니다. 8단계 모달(연령, 예산, 관심 시술, 지역, 우선순위, 언어 테마, 피부 고민, 시술 부위)이 진입 장벽 자체인 거죠.

**PM**: 저도 코드를 분석해봤는데, 시술 추천이 8단계 모달이라는 건 유저에게 **"이 페이지에서 뭘 해야 하는지 모르겠는데 8개나 물어본다"**는 경험이에요. 이건 개선 여지가 명확합니다.

### 1-2. 전략적 위치 판단

**Marketer**: 뷰티가 회사의 전략 카테고리인 상황에서, K-beauty 페이지의 역할을 정리해보겠습니다.

```
유저 여정에서 K-beauty 페이지의 위치:

[메인 페이지] ──(10%)──→ [K-beauty 페이지] ──(30~33%)──→ [스팟 상세]
     │                         │
     │                         ├── 블로그 콘텐츠 소비
     │                         ├── 리뷰 탐색
     │                         └── 문의 폼 작성
     │
     └── [GNB > Travel > K-beauty]
```

K-beauty 페이지는 **뷰티 카테고리의 허브**입니다. 개별 스팟 상세로의 진입점이면서, 동시에 "한국에서 뷰티 시술을 받고 싶다"는 유저의 **탐색 시작점**이에요. 이 페이지가 없으면 유저는 검색이나 카테고리 리스트에서 개별 스팟을 하나씩 찾아야 합니다.

**PO**: 결론적으로, **K-beauty 페이지 개선은 진행해야 합니다.** 이유는:

1. **전략적 필요성**: 뷰티를 밀고 있는데 그 허브 페이지를 방치할 수 없음
2. **개선 여지가 명확**: 진입율 10%, 추천 CTR 1.4% — 상향 여지가 큼
3. **페이지 내 전환력(30~33%)은 기반이 있음**: 완전히 새로 만들 필요 없이, 진입과 콘텐츠를 보강하면 됨

단, **우선순위 배분이 중요합니다.** 진입(Q2)이 막혀 있는 상태에서 내부 개선(Q3)만 하면 효과가 제한적이에요. **Q2(진입 확대) → Q3(내부 개선)** 순서로 가되, 기술적으로 병렬 가능한 건 병렬로 진행합시다.

**Growth Manager**: 동의합니다. 추가로, 현재 코드베이스 분석에서 확인한 기술적 현실도 Q1 판단에 영향을 줍니다.

| 현재 상태 | 의미 |
|-----------|------|
| 상단 블로그 배너가 **언어별로 하드코딩** (14개 언어 × 10개 블로그 코드) | 콘텐츠 교체가 개발 배포 의존적 — 마케팅팀이 자체 운영 불가 |
| 카테고리 아이콘이 **코드에 정적 매핑** | 새 카테고리 추가 시 개발 필요 |
| 시술 추천 8단계가 **프론트엔드 로직에 고정** | A/B 테스트나 단계 조정이 불가 |
| Trust Metrics 숫자(100+, 50K, 98%, 100)가 **하드코딩** | 실제 데이터 반영 불가 |

**PM**: 이 기술 부채는 "어드민에서 관리할 수 있도록 개선" 계획과 맞물립니다. 어드민화는 Q3의 전제 조건이에요. 하드코딩된 콘텐츠를 어드민으로 옮기지 않으면 디자인 개선을 하더라도 운영이 안 됩니다.

**PO**: Q1 결론을 공식화합니다.

> **합의: K-beauty 페이지 개선은 진행한다.** 어드민화(기술 부채 해소)를 기반으로, 진입 확대(Q2)와 내부 개선(Q3)을 병렬 추진한다.

## 2. Q2 — 페이지 진입을 어떻게 늘릴 것인가?

### 2-1. 현재 진입점 분석

**PM**: 현재 K-beauty 페이지로의 진입점을 코드 기준으로 정리합니다.

| 진입점 | 위치 | 형태 | 문제점 |
|--------|------|------|--------|
| 메인 페이지 카드 | `HomeKBeautyMenu` — 메인 페이지 내 카드형 위젯 | 카테고리 탭 칩 나열 + 화살표 링크 | 메인 페이지 내 다른 컨텐츠에 묻힘, 10% CTR |
| GNB | Travel 하위 | 텍스트 링크 | 뷰티에 관심 있는 유저가 "Travel" 밑을 찾을 이유가 약함 |

**두 개뿐입니다.** 그리고 둘 다 유저의 **능동적 탐색**을 전제로 합니다. "K-beauty가 뭔지 알고 있고, 찾아서 들어오는" 유저만 도달해요.

**Marketer**: 이게 핵심 문제입니다. 뷰티 시술에 관심 있는 유저가 크리에이트립에 처음 왔을 때, **"아, 여기 뷰티 전문 페이지가 있구나"를 발견하는 경로가 너무 제한적**이에요.

경쟁 플랫폼을 보면, 뷰티/웰니스를 밀고 있는 서비스들은 **메인 내비게이션 레벨에서 독립 카테고리**로 분리합니다. 우리는 Travel 하위에 숨겨져 있어요.

### 2-2. 진입 확대 방안

**Growth Manager**: 진입 확대를 **노출 레벨**로 분류해서 논의합시다.

#### 방안 A: GNB 레벨 승격

```
현재: Home | Travel > K-beauty | Stays | ...
개선: Home | K-beauty | Travel | Stays | ...
```

**Marketer**: GNB에 독립 탭으로 올리는 건 가장 강력한 진입 확대입니다. **"이 플랫폼이 뷰티를 중요하게 다루고 있구나"**라는 인식을 즉각적으로 줘요.

**PM**: 다만 GNB 구조 변경은 **모든 유저에게 영향**을 미칩니다. 뷰티에 관심 없는 유저에게도 GNB 공간을 할애하게 돼요. 이건 데이터로 판단해야 합니다 — 전체 유저 중 뷰티 관련 행동을 보이는 비율이 어느 정도인지.

**PO**: GNB 승격은 임팩트가 크지만 리스크도 있어요. **A/B 테스트 가능한 형태**로 접근합시다. 먼저 특정 언어군(뷰티 수요가 높은 언어)에서만 노출하고, 데이터를 본 후 전체 적용 여부를 결정하는 게 안전합니다.

#### 방안 B: 메인 페이지 진입 강화

**Growth Manager**: 현재 `HomeKBeautyMenu`는 카테고리 칩을 나열하는 형태인데, 이것만으로는 **"왜 K-beauty 페이지에 가야 하는지"**가 전달되지 않아요.

```
현재:
┌─────────────────────────────────────────────┐
│ K-Beauty를 찾아보세요 →                       │
│ [피부관리] [의료시술] [네일/메이크업] [헤어] ... │
└─────────────────────────────────────────────┘

개선안:
┌─────────────────────────────────────────────┐
│ [히어로 비주얼: Before/After or 시술 사진]    │
│                                             │
│ "한국에서만 가능한 뷰티 시술"                  │
│ 100+ 클리닉 · 50K+ 방문자 · 98% 만족도       │
│                                             │
│ [K-Beauty 전체 보기]  or  [내게 맞는 시술 찾기] │
│ [피부관리] [의료시술] [네일/메이크업] [헤어] ... │
└─────────────────────────────────────────────┘
```

**Marketer**: 핵심은 **"왜"를 먼저 전달**하는 거예요. 지금은 카테고리 나열만 있어서 "뭐가 있는지"만 보여주는데, "왜 여기서 해야 하는지"를 소셜 프루프(100+ 클리닉, 98% 만족도)와 비주얼로 먼저 보여주면 클릭 동기가 생깁니다.

**PM**: 이 방안은 `HomeKBeautyMenu` 컴포넌트 수정으로 구현 가능하고, 어드민화와도 연결됩니다. Trust Metrics 숫자를 어드민에서 관리하게 되면 메인 페이지 카드에도 동일한 데이터를 쓸 수 있어요.

#### 방안 C: 맥락적 진입점 추가

**Growth Manager**: 유저가 이미 뷰티 관련 행동을 하고 있을 때 자연스럽게 K-beauty 페이지로 유도하는 방법입니다.

| 맥락 | 진입점 | 기대 효과 |
|------|--------|----------|
| 뷰티 스팟 상세 → 하단 | "다른 뷰티 시술도 보기" CTA → K-beauty 페이지 | 교차탐색 유도 |
| 뷰티 블로그 읽기 중 | "이 시술 예약하기" 인라인 CTA → K-beauty 페이지 | 콘텐츠→커머스 전환 |
| 검색 결과 (뷰티 키워드) | K-beauty 페이지 배너/카드 노출 | 검색 유저 캡처 |
| 메인 페이지 팝업/배너 | 시즌 프로모션 연동 | 이벤트 기반 유입 |

**Marketer**: 방안 C의 핵심은 **"유저가 이미 뷰티에 관심을 보인 시점"에서 진입을 제안**하는 거예요. 콜드 스타트(메인 페이지에서 갑자기 K-beauty 클릭)보다 전환율이 높을 수밖에 없습니다.

### 2-3. Q2 결론

**PO**: 진입 확대 방안을 정리합니다.

| 방안 | 난이도 | 기대 임팩트 | 우선순위 |
|------|--------|-----------|---------|
| A: GNB 승격 (A/B 테스트) | 중 | 높음 | P1 — 단, A/B 테스트 설계 먼저 |
| B: 메인 카드 리디자인 | 낮음 | 중간 | P1 — 어드민화와 병렬 |
| C: 맥락적 진입점 (스팟 상세, 블로그) | 중 | 중~높음 | P2 — 스팟 상세부터 시작 |

**Growth Manager**: 한 가지 주의할 점이 있습니다. **진입을 늘려도 페이지 품질이 낮으면 이탈만 늘어납니다.** 방안 B(메인 카드 리디자인)로 기대치를 높여놓고 실제 페이지가 기대에 못 미치면 역효과예요. 그래서 **Q2(진입)와 Q3(내부 개선)는 동시에 가야 합니다.**

**PO**: 동의합니다. Q3로 넘어갑시다.

## 3. Q3 — 페이지 내부를 어떻게 개선할 것인가?

### 3-1. 현재 페이지 구조 분석

**PM**: 코드 분석 기반으로 현재 페이지 구조를 정리합니다.

```
현재 K-beauty 페이지 구조 (위→아래):

1. [Travel Nav] 브레드크럼
2. [Top Banner] 언어별 하드코딩 블로그 캐러셀 (10개)
3. [Category Tab] 7개 카테고리 + All 탭
4. [Categorized Spots] 카테고리별 스팟 그리드
5. [My Preference] (All 탭에서만) 8단계 모달 → 추천 ← CTR 1.4%
6. [Trust Highlights] "Why Creatrip" 4개 카드
7. [Tab Contents] 탭별 동적 콘텐츠 (블로그/스팟 혼합)
8. [Review Choice Groups] 리뷰 기반 추천
9. [Recent Reviews] 최근 리뷰 캐러셀
10. [Photo Reviews] 포토 리뷰 캐러셀
11. ─── 구분선 ───
12. [Popular Blogs] 인기 블로그 캐러셀
13. [Latest Blogs] 최신 블로그 캐러셀
14. [Map Section] 지도
15. [Trust Metrics] 숫자 통계 (All 탭에서만)
16. [Inquiry Form] 이메일 상담 신청
17. [Floating] 챗봇 + 이메일 상담 버튼
18. [Popup Banner] 팝업
```

**Marketer**: **18개 섹션**입니다. 이건 너무 많아요. 유저가 스크롤하면서 **정보 과부하**를 경험할 수 있습니다. 그리고 섹션 간의 **명확한 스토리라인**이 없어요. "블로그 → 스팟 → 리뷰 → 블로그 → 지도 → 통계 → 폼"으로 콘텐츠 유형이 반복적으로 왔다갔다합니다.

**Growth Manager**: 특히 **블로그 관련 섹션이 3개**(Top Banner, Popular Blogs, Latest Blogs)로 분산되어 있고, **리뷰 관련도 3개**(Review Choice Groups, Recent Reviews, Photo Reviews)로 분산되어 있어요. 이걸 유저가 어떻게 구분하는지 의문입니다.

### 3-2. 핵심 문제점

**PM**: 문제를 구조화합니다.

| 문제 | 현재 상태 | 영향 |
|------|---------|------|
| **스토리라인 부재** | 18개 섹션이 타입별로 산발적 배치 | 유저가 "다음에 뭐가 나오지?"를 예측 불가, 이탈 유발 |
| **시술 추천 진입 장벽** | 8단계 모달, All 탭에서만 노출 | CTR 1.4%, 개인화 경험 사실상 미제공 |
| **콘텐츠 중복/분산** | 블로그 3곳, 리뷰 3곳 | 같은 타입 콘텐츠가 반복 — 차이를 모르겠음 |
| **Trust 메시지 분리** | Highlights(섹션 6)와 Metrics(섹션 15)가 떨어져 있음 | 한 번에 신뢰를 형성하지 못함 |
| **CTA 불명확** | 페이지 목적이 "탐색"인지 "상담"인지 "예약"인지 | 유저가 다음 행동을 모름 |

### 3-3. 개선 방향 — 스토리라인 기반 재구성

**Marketer**: 페이지를 **유저의 의사결정 여정**에 맞춰 재구성해야 합니다.

```
유저의 뷰티 시술 의사결정 흐름:

"한국에서 뷰티 시술 받을 수 있나?" (인지)
  → "어떤 시술이 있지?" (탐색)
    → "이 시술 괜찮나? 후기는?" (검증)
      → "어떻게 예약하지?" (행동)
```

이 흐름에 맞게 페이지를 구성하면:

```
개선된 K-beauty 페이지 구조:

Phase 1: 인지 — "여기서 뷰티 시술을 받을 수 있다"
├─ [Hero] 임팩트 있는 비주얼 + 핵심 가치 제안
├─ [Trust] Why Creatrip (Highlights + Metrics 통합)
└─ [Social Proof] 핵심 통계 한 줄 (100+ 클리닉 / 50K 방문자 / 98% 만족도)

Phase 2: 탐색 — "나에게 맞는 시술 찾기"
├─ [Quick Match] 간소화된 추천 (2~3 질문, 8단계 → 축소)
├─ [Category Tab + Spots] 카테고리별 스팟 탐색
└─ [Curated Content] 탭별 블로그+스팟 혼합 콘텐츠 (기존 Tab Contents)

Phase 3: 검증 — "이거 괜찮은 거 맞지?"
├─ [Reviews] 통합 리뷰 섹션 (텍스트 + 포토 통합, 필터로 구분)
└─ [Blog Stories] 실제 경험담 블로그 (Popular + Latest 통합)

Phase 4: 행동 — "예약/문의하기"
├─ [Map] 클리닉 위치
└─ [Inquiry Form] 상담 신청 (기존 유지)
```

**PM**: 이 구조에서 **가장 큰 변화는 Phase 1의 Trust 통합과 Phase 2의 Quick Match**입니다.

Trust 통합은 현재 분리된 `KBeautyMainTrustHighlightsSection`(섹션 6)과 `KBeautyMainTrustMetricsSection`(섹션 15)을 **페이지 최상단으로 올려서 합치는 것**이에요. 유저가 페이지에 진입하자마자 "이 플랫폼을 믿을 수 있다"는 인식을 형성해야 합니다.

Quick Match는 현재 8단계 모달을 **2~3개 핵심 질문으로 축소**하는 것입니다. 예를 들면:

```
[Quick Match: 나에게 맞는 시술 찾기]
┌─────────────────────────────────────────────┐
│ Q1. 어떤 시술에 관심 있나요?                   │
│ [피부관리] [의료시술] [네일] [헤어] [기타]      │
│                                             │
│ Q2. 예산은 어느 정도인가요?                    │
│ [~₩50K] [₩50K~100K] [₩100K~300K] [₩300K+]  │
│                                             │
│ [추천 결과 보기 →]                            │
└─────────────────────────────────────────────┘
```

**Growth Manager**: Quick Match의 핵심은 **인라인으로 페이지 내에 배치**하는 것입니다. 현재는 모달이라 "열어야 보이는" 구조인데, 이걸 **스크롤하면 자연스럽게 만나는** 형태로 바꿔야 해요. 모달은 유저가 "이걸 열어야겠다"는 의도가 있어야 하지만, 인라인은 "아, 이런 것도 있네"로 자연스럽게 시작됩니다.

### 3-4. 콘텐츠 추가 검토

**PO**: 기존 콘텐츠 재구성 외에, 새로 추가하면 좋을 콘텐츠도 논의합시다.

**Marketer**: 뷰티 카테고리의 특성상, 유저가 가장 필요로 하는 콘텐츠는 다음과 같습니다.

| 콘텐츠 유형 | 목적 | 현재 유무 |
|------------|------|---------|
| **Before/After 갤러리** | 시술 결과를 시각적으로 확인 | 없음 |
| **가격 비교 가이드** | "한국 vs 자국" 가격 차이 인식 | 없음 |
| **시술 과정 안내** | 불안감 해소 (뭘 하는 건지 모름) | 블로그에 일부 존재 |
| **FAQ / 안전 정보** | "외국인인데 괜찮을까?" 해소 | 없음 |
| **실시간 인기 시술** | 트렌드 인식 + FOMO 유도 | 없음 (하드코딩 블로그만) |

**Growth Manager**: 이 중 **"실시간 인기 시술"은 어드민화 이후 바로 구현 가능**합니다. 현재 스팟 예약 데이터를 기반으로 "이번 주 가장 많이 예약된 시술" 같은 섹션을 자동으로 만들 수 있어요.

**Before/After 갤러리**는 임팩트가 가장 크지만 **콘텐츠 수급이 전제 조건**입니다. 클리닉과의 협업이 필요하고, 개인정보 이슈도 있어요. 이건 중기 과제로 분류하는 게 맞습니다.

**PM**: **가격 비교 가이드**는 유저별 국가 감지(`useLanguage`)를 이미 하고 있으니, 언어/국가별로 "한국에서의 이 시술 가격은 당신 나라 대비 X% 저렴합니다" 같은 메시지를 넣을 수 있어요. 이건 콘텐츠 팀에서 데이터만 정리해주면 구현 난이도가 낮습니다.

### 3-5. 디자인 개선 방향

**Marketer**: 디자인 관점에서 세 가지를 제안합니다.

**1. 히어로 섹션 강화**

현재 Top Banner는 블로그 캐러셀인데, 히어로 비주얼로서의 임팩트가 약합니다. 뷰티 페이지의 첫인상을 결정하는 영역이므로 **풀 블리드(full-width) 비주얼 + 핵심 카피 + CTA** 형태로 바꿔야 해요.

**2. 섹션 간 시각적 계층 구조**

현재는 18개 섹션이 거의 동일한 비중으로 나열되어 있어요. **Phase별로 배경색/간격/타이포그래피를 달리해서** "지금 어디에 있는지"를 유저가 직관적으로 알 수 있게 해야 합니다.

**3. 모바일 우선 레이아웃 재설계**

뷰티 시술을 알아보는 유저의 대부분은 모바일일 가능성이 높습니다. 현재 페이지가 데스크탑 레이아웃 기준으로 설계되어 있고 모바일은 반응형으로 축소되는 구조인데, **모바일에서의 스크롤 깊이와 섹션 노출 순서를 최적화**해야 합니다.

```
│ ✅ 피부관리                                   │
│                                             │
│ [~₩50K] [₩50K~100K] [₩100K~300K] [₩300K+]  │
│                                             │
│         ← 클릭하면 바로 결과 표시 →             │
└─────────────────────────────────────────────┘
         ↓ (결과가 같은 영역에 렌더링)
┌─────────────────────────────────────────────┐
│ 🔍 나에게 맞는 시술 찾기                       │
│                                             │
│ ✅ 피부관리 · ₩50K~100K                      │
│ [다시 선택하기]                               │
│                                             │
│ 추천 스팟 3개:                                │
│ ┌──────┐ ┌──────┐ ┌──────┐                  │
│ │ 스팟1 │ │ 스팟2 │ │ 스팟3 │                  │
│ └──────┘ └──────┘ └──────┘                  │
│                                             │
│ [더 많은 추천 보기 →]                          │
└─────────────────────────────────────────────┘
```

**Growth Manager**: 이 방식의 장점이 두 가지 있습니다.

첫째, **"결과 보기" 버튼 없이 마지막 선택이 곧 트리거**입니다. 폼은 "다 채우고 → 제출"이라는 2단계 인지 부하가 있는데, 스텝 위저드는 마지막 선택을 클릭하는 순간 결과가 나옵니다. 버튼 하나가 줄어드는 게 아니라, **"제출해야 한다"는 심리적 부담 자체가 사라집니다.**

둘째, **추천 결과가 같은 영역에 렌더링**되기 때문에 페이지 이동이 없습니다. 유저가 스크롤 위치를 잃지 않아요. 만약 결과가 마음에 안 들면 "다시 선택하기"로 즉시 돌아갈 수 있고, 마음에 들면 바로 스팟 상세로 진입합니다.

**Marketer**: 그리고 이전 단계 선택이 **요약 태그로 남아 있어서** ("✅ 피부관리 · ₩50K~100K") 유저가 자기가 뭘 선택했는지를 항상 인지합니다. 모달은 닫히면 맥락이 사라지지만, 임베디드는 맥락이 페이지에 남아요.

**PM**: 단계 수에 대해서는 — 현재 8단계를 **3단계(관심 시술 → 예산 → 지역 or 피부 고민)** 정도로 축소하는 게 적절해 보입니다. 나머지 세부 옵션(연령, 우선순위, 언어 테마 등)은 "더 정밀한 추천 받기"로 기존 모달을 연결하는 심화 경로로 유지하면 됩니다.

**PO**: 합의합니다. 기존 합의의 "인라인 Quick Match"를 **"임베디드 스텝 위저드"**로 구체화합니다. 기존 §3-3의 Quick Match 와이어프레임을 이 방식으로 대체합니다.

#### 2. 뷰티캐시백 → Phase 1(인지) 독립 섹션으로 승격

**Marketer**: 뷰티캐시백은 우리 플랫폼의 **핵심 차별화 포인트**입니다. "예약하고, 시술 받고, 영수증 인증하면 포인트로 돌려준다" — 이건 경쟁사에서 제공하지 않는 독자적인 가치예요.

현재 이 내용이 Trust Highlights의 4개 카드 중 하나(`beauty-cashback.description`)로 묻혀 있는데, **독립적인 가치 제안 섹션으로 분리해서 Phase 1(인지)에 배치**해야 합니다.

```
개선된 Phase 1: 인지 — "여기서 뷰티 시술을 받을 수 있다"

├─ [Hero] 임팩트 있는 비주얼 + 핵심 가치 제안
├─ [Beauty Cashback] 캐시백 제도 소개 + CTA
│   ┌─────────────────────────────────────────────┐
│   │ 💰 뷰티캐시백                                │
│   │                                             │
│   │ 크리에이트립에서 예약 → 시술 → 영수증 인증      │
│   │ = 시술비의 일부를 포인트로 돌려드려요            │
│   │                                             │
│   │ [캐시백 자세히 보기 →]                         │
│   └─────────────────────────────────────────────┘
├─ [Trust] Why Creatrip (Highlights + Metrics 통합, 캐시백 카드는 제거)
└─ [Social Proof] 핵심 통계 한 줄
```

**Growth Manager**: 캐시백을 인지 단계에 넣는 이유가 명확합니다. 유저가 페이지에 진입했을 때 **"여기서 예약하면 돈을 돌려받는다"**는 정보는 탐색 전에 알아야 하는 정보예요. 탐색 중에 알게 되면 "아 그거 아까 봤나?" 하고 위로 스크롤해야 합니다. **의사결정의 전제 조건**이니까 가장 먼저 보여줘야 합니다.

**PM**: 기술적으로도 자연스럽습니다. 현재 `KBeautyMainTrustHighlightsSection`의 캐시백 카드를 **독립 컴포넌트로 분리**하고, Trust Highlights에서는 나머지 3개(플랫폼 소개, 가격 경쟁력, 24시간 지원)만 남기면 됩니다. Trust 카드가 4개에서 3개로 줄면 그리드 레이아웃도 더 깔끔해지고요.

**PO**: 동의합니다. 캐시백은 **Phase 1(인지)에 Hero 바로 아래, Trust 위에 독립 섹션으로 배치**합니다.
