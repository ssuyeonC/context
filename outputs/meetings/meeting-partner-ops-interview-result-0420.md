## 🔴 공통 페인포인트

### 1. 카톡방 의존성이 너무 크다
- 3명 모두 카톡방을 주 소통 채널로 사용
- 파트너들도 카톡을 가장 선호 (가장 즉각적)
- **지윤님**: "단톡방은 없앨 수 없는 구조... 카톡이 거의 대부분"
- **혜원님**: 네이버처럼 업체↔고객 메신저 창구 요청 (유사 요청 3건)

### 2. 파트너의 어드민 인지도/활용도 낮음
- **혜원님**: 알림톡 버튼 누르고서야 어드민 존재를 알게 됨
- **지윤님**: 매뉴얼 전달하면 조금씩 늘어나는 추세, **파트너 어드민은 한 분이 관리하는 게 효과적**
- **성민님**: 블로그/스팟 수정은 성민님+인턴만 관여 (파트너가 직접 못함)

### 3. 스팟 컨텐츠 수정 리소스 과다
- 3명 모두 "파트너가 직접 컨텐츠 수정"을 니즈로 언급
- 자잘한 것(가격/주의사항/이벤트)이 많이 들어오는 게 문제

## 🟡 인터뷰이별 고유 인사이트

### 혜원님 (뷰티 시술)
- ⚠️ **알림톡 템플릿 구분 문제**: 확정/취소/변경 템플릿이 유사해 동일 건인 줄 알고 재확인
- **오버부킹 케이스 많음** → 날짜별 예약 건수 조회 니즈
- CX팀이 예약 접수를 주로 담당

### 지윤님 (메디컬)
- 🌟 **성공 사례: 메디큐브**
  - 승인 지연 컴플레인 → 어드민 제안 → 전환 성공
  - **"파트너 담당자가 한 분일 때" 전환이 쉬움**
- 🌟 **성공 사례: 뷰티블라썸**
  - 파트너 어드민 선제안 + 단톡방 없이 운영
  - **"파트너 담당자 연령대"도 전환 성공의 주요 변수**
- 자동확정 기준: 피부과는 상담실장·통역·원장 필요 → 수동 불가피
- 날짜 변경 요청은 대부분 유저 단순 변심
- 제안: **여러 파트너 대상 서면 설문 인터뷰**도 고려

### 성민님 (체험/식당/의상)
- 💣 **가장 복잡한 업무 구조**: 블로그-스팟 연결 관리
  - 스팟 가격 변경 시 블로그도 수동 수정 필요
  - 크트 특성상 블로그를 직접 써주며 받는 돈이 있음
  - 파트너 연령대 높음 → 더 어려움
- 💣 **수수료 입력 UX 문제**
  - 입력란은 존재하나, **작성하지 않는 케이스가 있음**
  - **정산타입이 "정산가 정산"인 경우 입력하지 않아야 하는 경우도 있음**
  - → 수수료율 필수값 여부/정산타입별 분기 로직 재검토 필요
  - 세금계산서 미발행 업체 1건이 현재 유일한 예외 케이스
- 🎯 **구체적 개선 제안**
  - 파트너 블로그 양식 정형화 → 양식 내에서만 파트너가 수정 가능
  - 파트너 수정 요청 + 관리자 승인 워크플로우
    - 어드민 내 알림 / 슬랙 알림
    - → 관리자 확인 → AI 번역 → 바로 적용 or 어권 확인 요청
  - 알림톡 내에서 예약 변경 요청 (날짜/시간, 최대 5개)

## 🧭 통합 개선 우선순위

### 🥇 최우선: 예약 처리 자동화 & 알림톡 개선
- **날짜별 예약 건수 조회** (혜원: 오버부킹 방지)
- **알림톡 템플릿 명확화** (혜원: 확정/취소/변경/거절 구분)
- **알림톡 내 예약 변경 요청** 기능 (성민: 날짜/시간 5개 옵션)

### 🥈 차우선: 파트너 직접 컨텐츠 수정
- **스팟 내부 컨텐츠 파트너 수정** (이미지 순서, 텍스트, 가격)
- **수정 요청 → 관리자 승인 워크플로우** (성민 제안)
  - 어드민/슬랙 알림 → 승인 → AI 번역 → 반영
- **수수료 입력 UX 개선** (성민: 정산타입별 조건부 필수/비활성화 처리)
- **블로그 양식 정형화** → 양식 내 파트너 수정 (성민)

### 🥉 장기: 파트너 어드민 전환 확대
- **온보딩 강화**: 어드민 존재 자체를 모름 (혜원)
- **타겟 세분화**: "담당자 1명 체제 + 비교적 젊은 연령대" 파트너부터 우선 전환 (지윤 인사이트)
- **카톡 → 어드민 내 메신저 전환** (본질은 카톡 의존도 자체를 낮추는 것)
- **스팟 구성 컨설팅 대체 기능** (혜원)
