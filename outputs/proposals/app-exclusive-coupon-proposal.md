# 앱 전용 쿠폰 도입 Proposal

### 왜 이게 문제라고 생각했는가?

- 현재 크리에이트립 트래픽의 대부분이 **웹(모바일 웹 포함)에서 발생**하고 있어, 앱 유저 비중이 낮음
  - 웹 유저는 세션이 끝나면 재방문을 유도할 수단이 제한적 (브라우저 알림은 허용률 극히 낮음)
  - 반면 앱 유저는 **푸시 메시지, 인앱 배너, 뱃지 알림** 등 다양한 CRM 채널 활용 가능
- **푸시 메시지 도달 가능 유저 풀이 작아** CRM 캠페인의 절대적 임팩트가 제한됨
  - 재방문 유도, 장바구니 리마인더, 가격 변동 알림, 시즌 프로모션 등 검증된 CRM 시나리오가 있어도 대상 모수가 작으면 효과가 미미
  - 앱 유저 확대 = CRM 도달 가능 모수 확대 = 전체 리텐션/구매 전환의 레버리지 확대
- **웹과 앱의 구매 경험 차이가 없어**, 유저 입장에서 앱을 설치할 명확한 동기가 부족
  - "앱에서만 받을 수 있는 혜택"이 없으면 유저는 굳이 앱을 설치하지 않음
  - 앱 전용 쿠폰은 **가장 직접적이고 이해하기 쉬운 앱 전환 동기**

### 어떤 사용자에게 영향이 있을까?

- **모바일 웹에서 구매 의향이 있는 유저**
  - 이미 상품을 탐색하고 결제 직전까지 왔지만 아직 앱 미설치 상태인 유저
  - "앱에서 결제하면 할인"이라는 메시지에 가장 민감하게 반응할 그룹
  - 전환 비용(앱 설치)을 감수할 만큼 구매 의도가 높은 상태
- **과거 구매 이력이 있는 웹 유저 (재구매 잠재 고객)**
  - 이미 크리에이트립에서 결제 경험이 있어 신뢰도가 확보된 상태
  - 앱 설치 후 푸시 알림을 통해 재구매 유도가 가능한 고가치 세그먼트
  - 앱 전환 시 LTV 상승 잠재력이 가장 큰 그룹
- **여행 준비 단계의 첫 방문 유저**
  - 여러 상품을 비교하는 초기 단계에서 앱 쿠폰을 인지하면, 최종 결제를 앱에서 할 동기 확보
  - 앱 설치 → 여행 중 푸시 알림 → 현지에서 추가 구매의 여정으로 이어질 수 있음
- **기존 앱 유저 (이탈 방지)**
  - 앱 전용 혜택이 지속적으로 존재하면, 앱 삭제를 억제하고 DAU 유지에 기여

### 가설적 임팩트?

- **노출 대상**: 모바일 웹 일 방문자 **~3,700명/일** (상품 상세 ~5,300명 × 모바일 비중 70%)
- **앱 전환 가정** (기본 시나리오: 전환율 5%)
  - 신규 앱 유저: **~185명/일, 월 ~5,550명**
  - 푸시 허용률 60% 가정 시, 월 **~3,330명** 푸시 도달 가능 유저 순증
  - 6개월 누적 **~20,000명** 추가 푸시 도달 모수 확보
- **쿠폰 사용**: 앱 전환 유저의 70% 사용 가정 → **~130건/일**, 비용 ~390,000원/일
- **질적 임팩트**
  - 앱 유저 모수 확대 → 푸시 기반 CRM(재방문 유도, 장바구니 리마인더, D-day 알림 등)을 공격적으로 운영할 수 있는 기반 마련
  - 장기적으로 마케팅 비용 효율 개선 (paid → owned 채널 전환)

### 해결 방법 논의 및 실행

1. **앱 전용 쿠폰 설계**
   - **최초 설치 쿠폰**: 첫 주문 시 할인 (예: 3,000원, 유효기간 7일)
   - **정기 쿠폰**: 매주/매월 앱에서만 받을 수 있는 쿠폰 자동 발급 (앱 유지 동기)

2. **웹 → 앱 전환 유도 터치포인트**
   - 상세페이지 하단 배너 + 결제 페이지 바텀시트 + 장바구니 배너
   - 딥링크로 앱 설치 → 해당 상품 페이지 바로 랜딩

3. **CRM 시나리오 연계**
   - D+0: 환영 쿠폰 푸시 → D+3: 만료 리마인더 → D+7: 정기 쿠폰 발급
   - 주간 정기 + 여행 D-day 기반 쿠폰 발송

4. **측정 지표**
   - 배너 CTR, 앱 설치 전환율, 쿠폰 사용률, 앱 DAU/MAU 변화, 푸시 도달 모수 추이, 쿠폰 ROI

5. **미해결 사항**
   - 쿠폰 금액 최적화 (A/B 테스트), 웹→앱 cannibalization 측정, 재설치 남용 방지, 기존 쿠폰과 중복 사용 정책, 푸시 빈도 조율
