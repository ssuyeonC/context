# K-Beauty 페이지 구조 개편 제안서

> **작성일**: 2026-03-25
> **작성**: Product팀
> **상태**: 내부 리뷰 대기
> **관련 문서**: 3/22 개선 방향 회의록 · 3/24 데이터 리뷰 · 3/25 아이데이션 회의록 · 외국인 피부과 인지도 리서치

---

## TL;DR

K-Beauty 페이지는 월 873세션, 스팟 클릭율 18.9%로 뷰티 전략 카테고리의 허브 역할을 하기엔 부족한 상태입니다. 외국인 고객 리서치 결과, 우리 타겟은 "시술을 받기 위해 한국에 오는" 고관여 목적형 방문자임에도, 현재 페이지는 K뷰티를 소개하는 영역과 특정 기준으로 나열된 스팟을 병렬적으로 노출하는 구조로 설계되어 있어 이들의 의사결정 흐름과 불일치합니다.

**핵심 제안**: 이미 시술을 정하고 온 고관여 고객이 **"왜 크리에이트립인가"를 빠르게 인지하고 → 원하는 시술/클리닉을 최단 경로로 탐색하고 → 후기와 가이드로 검증하고 → 바로 예약할 수 있는** 의사결정 흐름 기반의 4-Phase 10개 섹션 구조로 재설계합니다. 이를 통해 스팟 클릭율을 18.9% → 25~30%로 끌어올리는 것이 목표입니다.

## 1. 배경: 왜 지금 이걸 해야 하는가

### 전략적 맥락

회사가 뷰티를 전략 카테고리로 밀고 있는 상황에서, K-Beauty 페이지는 뷰티의 **유일한 허브 페이지**입니다. 개별 스팟 상세로의 진입점이자, "한국에서 뷰티 시술을 받고 싶다"는 유저의 탐색 시작점입니다. 이 페이지가 제 역할을 하지 못하면, 뷰티 전략 자체의 전환 퍼널이 막힙니다.

### 현재 상태 (3/24 데이터 기준)

```
K뷰티 페이지 조회      873 세션/월 (일 약 30)
       │
       │  18.9% 스팟 클릭
       ▼
스팟 카드 클릭          165 세션
       │
       │  7.9% 구매
       ▼
스팟 구매 완료           13 세션 (CVR 1.49%)
```

- **월 873세션**은 허브 페이지로서 절대적으로 부족
- **81%의 유저(708명)가 스팟을 클릭하지 않고 이탈** — 가장 큰 손실 구간
- 같은 세션 CVR 1.49%는 과소 측정 가능성 (크로스 세션 시 32.9%가 스팟 상세 방문)
- 시술 추천(8단계 모달) CTR은 **1.4%** — 사실상 사용되지 않는 기능

### 구조적 문제

| 문제 | 현재 상태 | 영향 |
|------|---------|------|
| 정보 과부하 | **18개 섹션**이 타입별로 산발 배치 | 유저가 "다음에 뭐가 나올지" 예측 불가, 이탈 유발 |
| 스토리라인 부재 | 블로그 3곳, 리뷰 3곳 분산 | 같은 유형 콘텐츠가 반복 — 유저가 차이를 인지 못함 |
| 추천 진입 장벽 | 8단계 모달, All 탭에서만 노출 | CTR 1.4%, 개인화 경험 사실상 미제공 |
| 기술 부채 | 블로그 배너 14개 언어 하드코딩, Trust 숫자 하드코딩 | 콘텐츠 교체가 개발 배포에 의존 — 마케팅팀 자체 운영 불가 |

## 2. 고객 인사이트: 우리가 간과하고 있던 것

> 출처: 한국 피부과 시술명 외국인 인지도 리서치 (2026-03)

### 우리 고객은 초보자가 아니라 고관여 소비자다

현재 페이지는 "한국에서 뷰티 시술을 받을 수 있어요"를 설명하는 데 지면의 상당 부분을 쓰고 있습니다. 하지만 리서치가 보여주는 현실은 다릅니다.

**전제 1 — 피부 미용이 여행의 '목적'이다**

피부미용을 위해 한국을 찾는 외국인 관광객은 **"한국 여행 온 김에 피부 미용을 받는다"가 아니라 "피부 미용을 받기 위해 한국에 방문한다"**에 가깝습니다.

- 피부과 진료가 외국인 환자 방문의 **56.6%**로 압도적 1위 (2024년 117만 명)
- 리쥬란 엠보싱 회복 기간을 고려해 여행 일정 자체를 조정하는 수준
- "한국에서의 울쎄라는 비행기 값을 뽑는 시술"이라는 인식

→ **"여기서 뷰티 시술이 가능해요"는 이미 알고 있는 정보. 인지 단계를 길게 가져갈 필요 없음.**

**전제 2 — 고객은 이미 '무엇을 받아야 하는지' 안다**

- PN 성분, HIFU 원리 등 성분/기술 수준까지 사전 학습
- 리쥬란 종류별 차이(힐러/HB Plus/S/I)를 구분하고 목적에 맞게 선택
- 콤비네이션 테라피(울쎄라+리쥬란, 포텐자+엑소좀)를 **고객이 먼저 제안**

→ **"피부과란 무엇인가"를 설명하지 말고, "이 시술을 어디서, 얼마에, 어떤 조합으로 받을 수 있는가"에 빠르게 도달하는 구조가 필요.**

**전제 3 — 국가별로 고객 여정이 완전히 다르다**

| 국적 | 핵심 동기 | 의사결정 기준 | 선호 시술 |
|------|----------|-------------|----------|
| 일본 | 가성비 + 빠른 시술 | 가격, 소요 시간 | 보톡스, 레이저, 점 빼기 |
| 중국 | 정품 신뢰 + 프리미엄 | 정품 인증, 의료진 경력 | 울쎄라, 써마지 |
| 미국/유럽 | 과학적 근거 | 성분 원리, 임상 결과 | 리쥬란, 쥬베룩 |
| 동남아 | K-뷰티 동경 | 비주얼 결과, 트렌드 | 리쥬란, 엑소좀 |

→ **단일 구조로 모든 국가를 커버하되, 콘텐츠 레이어에서 국가별 차별화가 가능한 구조를 설계해야 함.**

### 추가 인사이트

| 인사이트 | 제품 시사점 |
|---------|-----------|
| 가격 투명성이 신뢰의 전제 조건 (레딧으로 시세 파악, 서면 견적서 요구) | 가격 정보를 숨기면 역효과. 시술별 시세 범위, 정품 인증 정보가 핵심 |
| 공장형 vs 부티크 병원 구분이 이미 상식 | 병원 유형/특화 분야 태깅이 차별화 포인트 |
| 콤비네이션 시술이 객단가 상승의 열쇠 | 단일 시술 나열 → 목적별 조합 패키지 추천 |
| 차세대 시술(쥬베룩, 엑소좀) 키워드 급부상 | 선제적 콘텐츠 확보로 얼리어답터 선점 기회 |

## 3. 해결 방향: 4-Phase 스토리라인 기반 재설계

### 설계 원칙

1. **인지는 짧고 강하게, 탐색으로 빠르게 전환** — 목적형 방문자에게 "왜 크리에이트립인가"만 전달
2. **시술명 → 스팟 매칭을 최단 경로로** — 유저가 "리쥬란"을 알고 왔으면 2클릭 이내로 해당 스팟에 도달
3. **탐색 진입점을 복수로 제공** — 인기 시술로, Wizard로, 카테고리로 — 여러 경로
4. **18개 → 10개 섹션으로 축소** — 정보 과부하 해소, 스토리라인 명확화

### 전체 페이지 구조


### Phase 2: 탐색 — "나에게 맞는 시술과 클리닉 찾기"

> **이 구간이 스팟 클릭율 18.9% → 25~30%를 달성해야 하는 핵심 구간**
> 873명 중 708명이 스팟을 클릭하지 않고 이탈하는 곳

**[2-A] 실시간 인기 시술 — Phase 2 최상단 배치**

- 3/24 데이터 리뷰에서 P2 → **P1으로 승격**된 섹션
- 목적형 방문자가 탐색에 진입하면 가장 먼저 보고 싶은 건 "지금 뭐가 핫한가"
- **순위 기반 표시로 시작**, 예약 수가 임계치를 넘으면 숫자 노출 (조건부 로직)
  - 이유: 월 873세션 기준 예약 수가 적을 수 있어 숫자가 작으면 오히려 역효과
- **콤비네이션 태그** 지원 — "리쥬란 + 리프팅" 같은 조합 패키지 자연 노출로 객단가 상승

**[2-B] Embedded Step Wizard (3단계 인터랙티브 탐색)**

```
Step 1/3: 어떤 시술에 관심 있나요?

[💇 헤어]
[✨ 기타]

      ↓ 클릭 시 자동 전환

Step 2/3: 예산은 어느 정도인가요?

      ↓ 클릭 시 자동 전환

Step 3/3: 선호 지역은?
[강남/신사]  [홍대/마포]  [명동/을지로]  [전체]

      ↓ 마지막 선택이 곧 결과 트리거

추천 스팟 3개가 같은 영역에 인라인 렌더링
[더 정밀한 추천 받기 →] → 기존 심화 모달 연결
```

기존 8단계 모달(CTR 1.4%)에서 완전히 전환한 설계:

| 기존 (8단계 모달) | 개선 (임베디드 스텝 위저드) |
|---|---|
| 버튼 클릭 → 모달 열림 → 8단계 입력 → 결과 | 스크롤 중 자연스럽게 도달 → 3단계 클릭 → 즉시 결과 |
| "폼을 채워야 한다" | "클릭하니까 반응한다" — 게이미피케이션에 가까운 경험 |
| 결과 확인 후 모달 닫으면 맥락 소실 | 이전 선택이 요약 태그로 남아 맥락 유지 |

**Step 1 설계 포인트**: 대분류(카테고리) + 인기 시술명 병기

- 리서치 전제 2에 따라 고객은 "피부관리"보다 "리쥬란"으로 사고
- 대분류는 고정, 괄호 안의 시술명은 **어드민에서 동적 관리** (트렌드 반영)

**Step 3을 "지역"으로 확정한 근거**:

- 리서치에서 고객이 이미 시술을 정해서 오므로 "피부 고민"보다 "지역"이 더 실용적
- 강남, 홍대, 명동이 구매 스팟 데이터의 주요 지역
- 여행 일정과 직결되는 정보

**[2-C] Category Tab + Spots**

- 기존 카테고리 탭과 스팟 그리드를 유지 — Wizard 미사용 유저를 위한 직접 탐색 경로
- 언어별 탭 순서 분기(일본: 피부관리 먼저, 중국: 리프팅 먼저 등)는 데이터 확보 후 P2

**[2-D] 가격 비교 가이드 (P2)**

```
┌─────────────────────────────────────────────┐
│ 💡 한국에서는 이 가격에 가능합니다              │
│                                             │
│  시술        한국        Your Country        │
│  리쥬란      ₩150,000    $350~500           │
│  울쎄라      ₩500,000    $2,000~4,000       │
│  보톡스      ₩50,000     $200~400           │
│                                             │
│  * 클리닉별 가격 차이가 있습니다               │
└─────────────────────────────────────────────┘
```

- `useLanguage`로 유저 국가 감지 → "Your Country" 열의 통화/가격을 동적 변경
- 리서치에서 "비행기 값을 뽑는 시술"이라는 인식을 **숫자로 시각화**
- 어드민 관리 정적 콘텐츠로 시작, 마케터가 분기별 업데이트

### Phase 3: 검증 — "실제 결과와 전문성으로 신뢰 확인"

> 3/22 원안: "리뷰와 블로그로 검증"
> **수정**: "실제 결과와 전문성으로 신뢰 확인"
> 근거: 리서치에서 검증 행동이 후기 읽기를 넘어 정품 인증, 장비 확인, 생물학적 원리 질문까지 포함

**[3-A] 통합 리뷰 섹션**

- 기존 3개(Review Choice Groups, Recent Reviews, Photo Reviews) → **하나로 통합**
- 필터(전체/포토/영상) + **국가 플래그 아이콘** + **시술명 태그**
- 국가 플래그: 리서치에서 고객이 "같은 국가 사람의 후기"를 특히 신뢰
- 시술명 태그: "리쥬란 후기"를 찾는 유저가 스팟 이름만으로는 시술 내용을 알 수 없음 (데이터 구조 확인 후 적용)

**[3-B] 시술 가이드 (블로그 리프레이밍)**

- "블로그" → "시술 가이드"로 명칭 변경 — 검증 단계 성격에 부합
- 우선 제작 콘텐츠:
  - **시술 비교**: "울쎄라 vs 슈링크, 어떤 게 나을까?" — 레딧에서 가장 많이 공유되는 포맷
  - **콤비네이션 가이드**: "울쎄라 + 리쥬란, 왜 같이 받을까?"
  - **차세대 시술**: 쥬베룩, 엑소좀 — 아직 경쟁이 적은 키워드로 검색 유입 선점
