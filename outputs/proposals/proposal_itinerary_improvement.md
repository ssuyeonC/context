# 여행 플래너(/itinerary) 사용성 개선 제안

### 왜 이게 문제라고 생각했는가?

- 여행 플래너는 지난 6개월 간 월 평균 약 **655명**이 플랜을 생성했지만, 그 중 **약 20%의 유저만 플랜을 수정**함
    - 플랜을 수정했다는 것은 실제로 활용하거나 활용할 의지가 있다고 추정할 수 있는 행동
    - 즉, **80%의 유저가 플랜을 만들기만 하고 이탈**하고 있는 상황
- 데이터 분석 결과, 플랜 생성 자체가 **리텐션 확률과 구매 전환율을 높이는 행동**으로 확인됨
    - 그럼에도 생성 이후 플랜을 지속 관리하도록 유도하는 장치가 부족하여, 이 잠재력을 충분히 활용하지 못하고 있다고 판단

### 어떤 사용자에게 영향이 있을까?

- **플랜을 생성했지만 수정하지 않는 유저 (~80%)**
    - 플랜 생성 후 추가 편집 없이 이탈하는 대다수 유저. 편집 동기나 접근성이 부족한 상태
- **장기 체류 관광객**
    - 한국 방문 외국인 관광객의 평균 체재 기간이 7.8일(한국문화관광연구원, 외래관광객조사)인데, 현재 플랜은 최대 5일까지만 생성 가능하여 실제 여행 일정을 온전히 담지 못하는 유저
- **복수 플랜을 생성하는 유저**
    - 하나의 유저가 여러 플랜을 만드는 경우가 많은데, 어떤 플랜이 '진짜 여행 플랜'인지 구분이 안 되어 관리가 분산되는 상태
- **장소 추가 기능을 활용하고 싶은 유저**
    - Google Maps API 기반 장소 추가 기능이 이미 존재하지만, 매우 제한적인 환경에서만 사용 가능하여 사실상 활용하는 유저가 없음

### 가설적 임팩트?

- **플랜 일 수 확대 (5일 → 10일)**
    - 평균 체재 기간 7.8일을 커버하는 플랜을 생성할 수 있게 됨
    - 플랜이 실제 여행 전체를 담을 수 있으므로, 생성 후 수정·관리할 동기가 강화될 것으로 기대
    - 현재 월 655명 생성 기준, 장기 체류 유저의 플랜 수정률(현 20%)이 개선될 여지가 있음
- **Google Maps API 장소 추가 접근성 개선**
    - 기존에 사실상 사용되지 않던 기능이 활성화되면, 플랜 편집 행동 자체가 늘어날 것으로 기대
    - Google Maps API 무료 Cap 범위 내에서 현재 유저 사용량을 충분히 커버할 수 있어, 추가 비용 부담 없이 도입 가능
- **메인 플랜 설정 기능**
    - 복수 플랜 중 하나를 '메인'으로 지정함으로써, 유저의 관심과 편집이 특정 플랜에 집중됨
    - 플랜 수정률(현 20%) 개선에 직접적으로 기여할 수 있을 것으로 기대
- **종합 기대 효과**
    - 플랜 생성 → 수정 전환율 20% → **30~40%** 개선 목표 (가설)
    - 플랜 활용도가 높아지면 크리에이트립 서비스 전반의 리텐션과 구매 전환율에도 긍정적 영향

### 해결 방법 논의 및 실행

1. **플랜 최대 일 수 확대 (5일 → 10일)**
    - 플랜 생성/편집 시 최대 일 수 제한을 10일로 변경
    - 기존 5일 이하 플랜은 영향 없이 유지
    - 근거: 외국인 관광객 평균 체재 기간 7.8일 (한국문화관광연구원, 외래관광객조사)

2. **Google Maps API 장소 추가 기능 접근성 개선**
    - 현재 제한적으로만 노출되는 장소 추가 기능을 플랜 편집 화면에서 쉽게 접근할 수 있도록 UI 개선
    - Google Maps API 무료 Cap 범위 내 운영 가능 여부 확인 후 적용
    - 장소 검색 → 플랜에 추가하는 플로우를 직관적으로 개선

3. **메인 플랜 설정 기능**
    - 유저가 보유한 복수 플랜 중 하나를 '메인 플랜'으로 지정하는 기능 추가
    - 메인 플랜은 플래너 진입 시 기본 노출, 상단 고정 등으로 접근성 우대
    - 메인 플랜에 대한 편집·관심 집중도를 높여 플랜 수정률 개선 유도
