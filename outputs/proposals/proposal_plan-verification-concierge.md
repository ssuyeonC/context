# 기획안: 플랜 검증하기 (Concierge MVP)

### 왜 이게 문제라고 생각했는가?

- 한국 여행을 준비하는 외국인의 **가장 큰 불안은 "내 일정이 현실적인지 모르겠다"**는 것
    - r/koreatravel 10,000건 분석 결과, **검증 요청(19.6%) + 가능성 확인(14.7%) = 34.3%**로 추천 요청(24.6%)보다 10pp 높음
    - "Is my 3-day Seoul+Busan itinerary too packed?", "Am I packing too much into my days?" 등의 질문이 반복적으로 등장
    - 특히 3~4일에 서울+부산을 시도하는 여행자가 매우 많으며(Reddit 내 서울만 여행자와 동수), 이들의 "빡빡한 일정" 불안이 핵심 마찰
- 현재 이 검증 니즈를 해소하는 유일한 방법은 **Reddit에 글을 올리고 6~24시간 댓글을 기다리는 것**
    - 피드백 반영 후 재게시 → 재대기의 비효율적 루프
    - 피드백의 품질도 답변자에 따라 편차가 큼
- **Klook, KKday에는 일정 계획/검증 기능이 아예 없고**, 구글 스프레드시트/노션은 일정을 "적는" 도구일 뿐 "괜찮은지" 알려주지 않음
    - 크리에이트립이 이 영역을 선점하면 **경쟁사가 단기간에 복제할 수 없는 차별화 포인트**가 됨
- 크리에이트립은 한국 여행만을 전문적으로 다루는 플랫폼 → **"한국 여행 전문가가 내 일정을 검증해준다"**는 포지셔닝은 해외 고객(특히 대만)에게 신뢰감과 전문성을 강하게 어필할 수 있음

### 어떤 사용자에게 영향이 있을까?

- **"일정 짜는 중" 유저 — 여행 날짜가 확정되고 스팟을 배치하는 단계의 유저**
    - 플래너에서 일정을 구성했지만 "이게 현실적인지" 확신이 없는 유저
    - Reddit에서 "itinerary check please"를 올리는 유저와 동일한 니즈
    - 일 플랜 생성 수: 2월 평균 ~34건, 3월 초 ~50건 (봄 시즌 진입으로 증가 추세)
- **서울+부산 멀티시티 여행자**
    - 3~4일에 2개 도시를 시도하는 여행자가 Reddit 내에서 가장 많은 검증 질문을 올리는 세그먼트
    - "too packed", "too ambitious" 불안이 집중되는 구간
- **초행자 (r/koreatravel 전체의 10.6%)**
    - 기본 운영(공항, 교통, 동선)에 대한 불확실성이 높아 일정 전체에 대한 검증 니즈가 가장 강함
    - "놓치는 것에 대한 불안"(24.5%), "과잉 일정 걱정"(16.5%)이 두드러짐

### 가설적 임팩트?

- **검증 요청 규모 추정**
    - 현재 일 플랜 생성 ~34~50건
    - 전체 플랜 생성자 중 검증을 요청하는 비율을 30~50%로 가정하면 → **일 10~25건** 수준
    - 이 규모는 1~2명이 수동으로 처리 가능한 범위
- **리텐션 효과 가설**
    - 플래너 사용 유저의 리텐션 리프트는 1.72x, 구매 전환 리프트는 3.49x로 확인됨
    - 검증 피드백을 받은 유저는 **"수정 → 재검증"** 루프를 반복하며 재방문할 가능성이 높음
    - 이 수정-재검증 루프 자체가 "다시 와야 하는 이유" = Open Loop로 작동
- **구매 전환 가설**
    - 검증 과정에서 "이 스팟은 예약이 필요해요" 등의 가이드를 제공하면 자연스럽게 구매로 연결 가능
    - Reddit에서는 불가능한 "검증 → 즉시 예약" 원스톱 플로우를 크리에이트립만 제공할 수 있음
- **브랜드 효과 가설**
    - "한국 여행 전문 플랫폼이 내 일정을 직접 봐준다"는 경험 → 크리에이트립의 전문성 인지도 강화
    - 만족한 유저가 Reddit 등에서 "크리에이트립에서 일정 검증해줘요"라고 자발적 언급 → 오가닉 인지도 구축 (현재 Reddit 내 인지도 0.1%)
- **AI/알고리즘 설계를 위한 데이터 축적**
    - 사람이 실제로 검증 요청을 처리하면서, 유저가 어떤 종류의 피드백에 만족하는지 패턴이 축적됨
    - 반복되는 검증 패턴(빡빡함, 이동 시간, 동선 비효율)을 식별 → 향후 자동화 규칙의 직접적 입력

### 해결 방법 논의 및 실행

**접근 방식: Concierge MVP — 사람이 먼저, AI/알고리즘은 나중에**

당장 AI나 알고리즘을 도입하지 않고, 유저의 검증 요청이 들어오면 **사람이 직접 판단하여 검증 결과를 전달**하는 방식으로 시작한다. 수요와 효과가 확인되면 반복 패턴을 규칙화 → 라이트 자동 검증 → AI 도입의 3단계로 확장한다.

1. **유저 플로우**
    - 유저가 플래너에서 일정을 구성한 후 **"내 플랜 검증하기"** 버튼을 클릭
    - 검증 요청이 운영팀(또는 담당자)에게 전달됨
    - 담당자가 일정을 검토하고 피드백을 작성 (목표 응답 시간: 24시간 이내)
    - 유저에게 검증 결과가 전달됨 (앱 푸시 / 이메일 / 인앱 알림)
    - 유저는 피드백을 반영해 일정을 수정하고, 필요 시 재검증 요청

2. **검증 피드백 항목 (예시)**
    - 일정 과밀 여부: "3일차에 5개 스팟은 빡빡할 수 있어요. 4개로 줄이는 걸 추천해요"
    - 동선 효율: "경복궁 → 남산타워는 이동 40분, 사이에 명동을 넣으면 동선이 효율적이에요"
    - 시간 배분: "이 일정이면 오전에 여유가 있어요. 북촌한옥마을을 추가해보세요"
    - 현실성 체크: "3일에 서울+부산은 KTX 이동 포함 시 타이트해요. 서울 2일 + 부산 1일 배분을 추천해요"
    - 예약/팁: "이 스팟은 사전 예약이 필요해요" / "월요일 휴무예요"

3. **언어 대응**
    - 크리에이트립 서비스 자체의 자동번역 기능을 활용하여 기본 커버 가능
    - 타 팀 협조를 통한 대만 원어민 활용 가능성 있음 (추후 확정)

4. **운영 부담 관리**
    - 초기 일 10~25건 수준 → 1~2명으로 운영 가능
    - Reddit 데이터 기준 반복되는 검증 패턴이 명확하므로(서울+부산 3~4일, 일정 과밀, 이동 시간) **검증 템플릿화**로 처리 속도를 높일 수 있음
    - 자주 나오는 피드백을 템플릿으로 관리 → 운영 부담이 시간이 갈수록 감소

5. **확장 로드맵 (3단계)**

    | 단계 | 방식 | 트리거 |
    |------|------|--------|
    | 1단계 (현재) | 사람이 직접 검증 | 즉시 시작 |
    | 2단계 | 반복 패턴을 규칙 기반으로 자동화 (빡빡함 경고, 이동 시간 자동 계산 등) | 검증 요청 패턴 데이터 축적 후 |
    | 3단계 | AI/알고리즘 기반 자동 검증 + 사람 보완 | 2단계 효과 검증 후 |

6. **성공 지표**

    | 지표 | 측정 기준 |
    |------|----------|
    | 검증 요청률 | 플래너 유저 중 검증 요청 비율 |
    | 검증 후 D+1 재방문률 | 검증 받은 유저가 다음 날 돌아오는 비율 |
    | 검증 후 일정 수정률 | 피드백을 반영해 일정을 수정하는 비율 |
    | 검증 후 구매 전환 | 검증 받은 유저의 구매 전환율 |
    | 유저 만족도 | 검증 피드백에 대한 만족도 (리액션 또는 설문) |
    | 응답 시간 | 요청 → 피드백 전달까지 소요 시간 (목표: 24시간 이내) |

7. **주의사항**
    - **응답 시간이 핵심 가치**: Reddit 대비 차별화 포인트는 "빠른 전문가 피드백". 응답이 24시간을 넘기면 Reddit과 차별화가 약해짐
    - **피드백 품질 일관성**: 담당자별 피드백 품질 편차를 줄이기 위해 검증 가이드라인/체크리스트 사전 제작 필요
    - **확장성 한계 인지**: 사람 기반 운영은 일 50건 이상에서 병목 발생. 수요가 확인되면 2단계(규칙 기반 자동화)로 빠르게 전환 필요
