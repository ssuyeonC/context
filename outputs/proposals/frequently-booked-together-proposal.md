# Frequently Booked Together 영역 개선 Proposal

### 왜 이게 문제라고 생각했는가?

- "Frequently Booked Together"가 유저에게 의미 있으려면, **같은 여행 맥락에서의 교차구매**가 반영되어야 함
  - "이 상품을 산 사람이 같은 여행에서 함께 산 상품" → 여행 준비에 실질적 도움
- 그러나 현재 로직은 **시간 제한 없이 전체 구매 이력을 조회**
  - 다른 여행에서 산 상품도 섞임 → 맥락 없는 추천이 되어 "함께 구매" 의미가 희석
- 추가로, 카테고리 구분 없이 최신 예약순 정렬 → 특정 인기 상품에 편중
  - 유사 상품(대체재)이 노출되어 "발견 가치"가 낮음

### 어떤 사용자에게 영향이 있을까?

- **New User (스팟 상세 페이지 방문자)**
  - 스팟 상세 페이지에 New User 트래픽이 압도적으로 집중
  - 개인화 데이터가 없는 상태에서 추천 품질 자체가 중요
- **여행 준비 중인 유저**
  - "이 상품 외에 뭘 더 준비해야 하지?"라는 질문에 답을 주는 영역
  - 보완재 중심 추천으로 "여행 준비가 덜 됐다"는 인식 → 재방문 동기 부여
- **D1 Return User로 전환 가능한 유저**
  - New User → D1 Return User 전환이 핵심 지표
  - 앱 비율이 웹 대비 6%로, 푸시보다 페이지 내 경험 개선이 레버리지가 큼

### 가설적 임팩트?

- **핵심 지표**: D1 Retention (New User → D1 Return User)
- **기대 효과**
  - 보완재 중심 추천으로 "여행 준비가 더 필요하다"는 인식 유도
  - 추가 상품 탐색 → 재방문 동기 증가
- **측정 가능한 지표**
  - Frequently Booked Together 영역 CTR
  - 추가 상품 조회율
  - D1 Retention 변화 (A/B 테스트 필요)
- **질적 임팩트**
  - 테스트 결과, 예스한복 → 증명사진 연계처럼 실제 소비 패턴이 자연스럽게 반영됨
  - 카테고리 간 겹침 없이 "발견 가치"가 있는 보완재 구성 확인

### 해결 방법 논의 및 실행

1. **추천 로직 변경**
   - 제안 로직
     - 상품 A를 구매한 모든 유저의 구매내역 분석 → **상위 3개 카테고리** 추출 → 카테고리당 **구매수 TOP 1, 2** → 총 **최대 6개** 상품 노출
   - 상세 조건
     - 카테고리 레벨: `MIDDLE_RESERVATION`으로 한정
     - 자기 카테고리 제외: 상품 A의 카테고리는 추천에서 제외
     - 지역/지하철 type 제외: `CITY`, `DETAIL_LOCATION`, `LEGAL_LOCATION`, `MAIN_SUBWAY`, `MIDDLE_SUBWAY`
     - 활성 상품만: `is_reservable = 1 AND is_in_business = 1`
     - 선정 기준: 구매수 단일 기준 (TOP 1, 2)
   - 블랙리스트 카테고리 (19개)
     - 교통: 공항 교통(746), 기차&버스(747), 교통예약(366), 페리(748), 자동차 대여(876), 콘서트 교통(3052)
     - 통신: 와이파이(904), 유심(903), 이심(905)
     - 배달/배송/수령: 배달서비스(365), 배송(750), 직접수령(749)
     - 공항/여행편의: 공항 서비스(751), 여행 필수(752), 여행편리(367), 여행필수 패키지(915)
     - 보험/카드: 보험(890), 선불카드(3061)
     - 상품권: 상품권(875)

2. **UI 변경 제안**
   - 레이아웃: 가로 캐러셀 → 방사형 마인드맵 구조 (상품 A 중심, 카테고리 3개 방사, 각 2개 상품)
   - 배치: 하단 → 상단 이동 (New User 스크롤 깊이가 깊지 않음)

3. **테스트 결과**
   - AREX(11229): 포토&의상대여(예스한복, 인코리아한복), 어트랙션(N서울타워, 롯데월드), 식당/맛집(크트패스, 진미식당)
   - 예스한복(11006): 어트랙션(N서울타워, 롯데월드), 식당/맛집(진미식당, 크트패스), 증명사진(포크스튜디오, 타임온미 스튜디오)
   - 결과: 카테고리 간 겹침 없음, 상품 중복 없음, "발견 가치"가 있는 보완재 구성

4. **미해결 사항**
   - 다양한 카테고리의 스팟에서 추가 테스트
   - 구매 이력이 적은 스팟의 fallback 정책 정의
   - 마인드맵 UI 기술적 구현 가능성 검토 및 디자인 확정
   - A/B 테스트 설계
   - 성과 측정 지표 최종 확정
