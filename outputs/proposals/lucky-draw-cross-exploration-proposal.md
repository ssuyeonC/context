# 럭키드로우 × 교차탐색 연동 Proposal

### 왜 이게 문제라고 생각했는가?

- 럭키드로우는 **리텐션 리프트 2.13x, 구매 전환 리프트 5.11x**로 과소평가된 기능
  - 좋아요(리텐션 2.17x)와 거의 동등한 리텐션 효과이면서, 구매 전환도 5배 이상
  - 환율 페이지(구매 전환 0.11x)와 달리, 뽑기 유저가 실제 구매까지 이어지고 있음
- 그러나 **액션율이 0.41%로 극도로 낮음** — 진입점 부족이 원인
  - 현재 럭키드로우 페이지의 진입 경로가 제한적이어서, 기능 자체를 모르는 유저가 대부분
- 현재 럭키드로우 추가 응모권 획득 경로는 **"공유"뿐** — 내부 탐색 행동과 연결되지 않음
  - 공유는 외부 확산(인지도) 기여는 하지만, 교차탐색(리텐션 임팩트 1위)과 무관
  - **"상품 2개 이상 탐색"이 리프트×모수 기준 RETURN_D1 임팩트 1위(리프트 1.63x, 모수 11.64%)**인데, 럭키드로우와 연결되어 있지 않음
- 상세페이지 조회 시 우측 하단에 뜨는 **"Recently Viewed" 플로팅 버튼의 클릭률이 낮음**
  - 유저가 이 버튼의 존재를 인지하지 못하거나, 클릭할 동기가 없음

### 어떤 사용자에게 영향이 있을까?

- **상품 1개만 보고 이탈하는 유저 (전체 방문자의 ~70%)**
  - 가장 큰 이탈 세그먼트. "1개 더 보면 추가 응모권"이라는 구체적 보상이 2개째 탐색으로의 임계점을 낮춤
  - 이 유저가 2개 이상 탐색으로 넘어가면 리텐션 리프트 1.63x, 구매 전환 리프트 3.83x 적용
- **이미 럭키드로우에 참여하고 있는 유저 (액션율 0.41%)**
  - 기존 "매일 1회 + 공유 추가 응모" 외에 **"상품 탐색 기반 추가 응모"라는 새로운 경로**가 열림
  - 응모 기회 확대 → 럭키드로우 참여 빈도 증가 → D+1 재방문 강화
- **여행 준비 중인 RETURN_D1 타겟 유저**
  - 여러 상품을 비교·탐색하는 행동 자체가 RETURN_D1 전환의 핵심 레버
  - 럭키드로우 보상이 탐색 행동의 추가 동기로 작동
- **Recently Viewed 플로팅 버튼을 인지하지 못하는 유저**
  - 툴팁("앞으로 N개만 더!") 노출을 통해 플로팅 버튼의 존재 자체를 인지시킴
  - 플로팅 버튼 클릭 → Recently Viewed 모달 → 과거 탐색 상품 재방문의 흐름이 활성화

### 가설적 임팩트?

- **핵심 지표**: RETURN_D1 모수 증가 + "상품 2개 이상 탐색" 비율 향상
- **노출 대상**
  - 상품 상세페이지 일 평균 방문자: **~5,300명**
  - 이 중 **~80%가 상품 1개만 보고 이탈** (~4,275명/일) — 이 유저가 직접 대상
  - 재방문(RETURN_D1) 유저는 비재방문 유저 대비 교차탐색 비율이 **2배** → 교차탐색을 유도하면 재방문으로 이어질 가능성이 높음
  > 📎 산출 근거: 일 여행 방문자 8,800명 중 2개+ 탐색 비율 11.64%(=1,025명) → 상세페이지 5,300명 중 19.3%만 2개+ 탐색, 나머지 80.7%가 1개 이탈 (데이터 출처: 방문자_수_전체.csv, RETURN_D1 버킷 분석)
- **교차탐색 전환 가정**
  - 1개 탐색 이탈 유저 모수: **~4,275명/일** (상세페이지 방문자의 80.7%)
  - 럭키드로우 추가 응모권 동기로 **이 유저의 5~15%가 N회 달성(추가 탐색)**으로 전환 시:

  | 시나리오 | 전환율 | 추가 교차탐색 유저 | 비고 |
  |---------|--------|-----------------|------|
  | 보수적 | 5% | 4,275 × 5% = **~214명/일** | N=5 또는 동기 약할 때 |
  | 기본 | 10% | 4,275 × 10% = **~428명/일** | N=3, 툴팁 주목도 효과 |
  | 낙관적 | 15% | 4,275 × 15% = **~641명/일** | N=3 + CRM 연계 시 |

  - 기본 시나리오(428명/일) 기준, "상품 2개 이상 탐색" 유저가 기존 ~1,025명 → **~1,453명으로 +42% 증가**
  - 1개 이탈 모수가 70% 가정(3,710명)보다 실제(4,275명)가 더 크므로, **임팩트도 이전 추정보다 15% 이상 상향**
- **RETURN_D1 임팩트 추정**
  - 추가 교차탐색 428명 × RETURN_D1 리프트 1.63x 적용
  - 현재 상세페이지 방문자의 평균 D+1 재방문률을 R이라 하면, 추가 교차탐색 유저의 재방문률은 R × 1.63
  - **추가 RETURN_D1 유저 ≒ 428 × R × 0.63** (리프트 순증분)
  - R = 8%(여행 방문자 평균) 가정 시: 428 × 0.08 × 0.63 ≒ **~22명/일 순 RETURN_D1 증가**
  - 현재 RETURN_D1 여행 방문자 ~720명 대비 **+3.1%** — 단일 기능으로서 유의미한 기여
- **구매 전환 임팩트 추정**
  - 추가 교차탐색 428명 × 구매 전환 리프트 3.83x 적용
    - 428명이 기존에 1개만 보던 유저 → 이제 2개+ 탐색하는 유저로 전환됨
      - 2개 이상 탐색 유저는 평균 대비 구매 전환율이 3.83배 높다는 데이터가 있음
    - 428명의 "행동 변화"로 인한 구매 전환 순증분
      - 428명 × 9.2%(2개+ 탐색 유저 전환율) ≒ ~39건/일
        - 여행 방문자 평균 예약 전환율: ~2.4%
        - 2개+ 탐색 유저 예상 전환율: 2.4% × 3.83 = ~9.2%
    - 현재 일 평균 예약 ~291건 대비 **+13.4%**
    - 일 거래액 ~30M원 기준, 추가 매출 **~4.0M원/일** 추정
- **럭키드로우 참여율 가정**
  - 현재 액션율 0.41% (상세페이지 방문자 5,300명 기준 ~22명/일)
  - 진입점 확대(Recently Viewed 모달 배너) + 교차탐색 연동으로 **1~2%**까지 확대 가능
  - 1% 시 ~53명/일, 2% 시 ~106명/일 → 현재 대비 **2.4~4.8배**
  - 럭키드로우 참여자의 구매 전환 리프트 5.11x가 유지된다면, 참여자 모수 확대만으로 구매 전환 절대값 증가
- **Recently Viewed 플로팅 버튼 CTR**
  - 현재 베이스라인 측정 필요
  - "앞으로 N개만 더!" 툴팁 추가로 **CTR +50~100% 기대** (주목도 상승 효과)
- **질적 임팩트**
  - "앞으로 N개만 더"라는 카운트다운 자체가 **Open Loop**(미완결 상태) — 듀오링고 데일리 퀘스트와 동일 메커니즘
  - 추가 응모권의 전제 조건이 상세페이지 조회이므로, **허수 리텐션(뽑기만 하고 이탈) 방지** — 교차탐색이 선행 행동으로 보장됨
  - 기존 추가 응모권 경로(공유)와 병렬 운영: 공유=외부 확산(인지도), 탐색=내부 교차탐색(리텐션). 두 경로 모두 전략적 가치가 있음
- **비용 구조**
  - 추가 응모권 경품 비용 → 교차탐색 유저(구매 전환 리프트 3.83x)의 추가 매출로 상쇄
  - 기본 시나리오 추가 매출 ~3.5M원/일 대비 경품 비용(저비용 혜택 중심)은 미미
  - 경품은 저비용 혜택(5% 할인 쿠폰, 포인트 적립 등)으로도 뽑기 재미 유지 가능

- **임팩트 요약 (기본 시나리오: 전환율 10% 기준)**

  | 지표 | 현재 | 예상 변화 | 증감 |
  |------|------|----------|------|
  | 1개 탐색 이탈 유저 (대상 모수) | ~4,275명/일 (80.7%) | — | — |
  | 상품 2개+ 탐색 유저 | ~1,025명/일 | ~1,453명/일 | **+42%** |
  | RETURN_D1 순증 | — | ~22명/일 | **+3.1%** |
  | 추가 예약 | — | ~39건/일 | **+13.4%** |
  | 추가 일 매출 | — | ~4.0M원/일 | — |
  | 럭키드로우 참여자 | ~22명/일 | ~53~106명/일 | **+2.4~4.8x** |

### 해결 방법 논의 및 실행

1. **"상세페이지 N회 조회 → 추가 응모권" 정책 추가**
   - 상품 상세페이지를 N회(3~5회) 조회하면 추가 럭키드로우 응모권 1회 지급
   - N값은 A/B 테스트로 최적값 결정 (3회 vs 5회)
   - 카운트 기준: 세션 내 고유 상세페이지 조회 수 (같은 페이지 중복 조회 제외)
   - 일일 추가 응모권 획득 상한: 1회 (남용 방지)

2. **Recently Viewed 플로팅 버튼 + 툴팁 UI**
   - 위치: 상세페이지 우측 하단 (기존 Recently Viewed 플로팅 버튼 재활용)
   - 툴팁 노출 조건: 상세페이지 1회 이상 조회 + 아직 N회 미달성
   - 툴팁 문구:
     - "앞으로 N개 상세페이지만 더 보면 추가 응모권을 받을 수 있어요!"
     - 조회 진행에 따라 카운트다운: "앞으로 2개만 더!" → "앞으로 1개만 더!"
   - N회 달성 시: 축하 UI ("추가 응모권 획득!") + 럭키드로우 페이지 이동 유도
   - 툴팁 피로도 관리: 일 최초 상세페이지 진입 시에만 노출, 또는 3~5초 후 자동 닫힘

3. **Recently Viewed 모달 내 럭키드로우 배너 추가**
   - 위치: Recently Viewed 플로팅 버튼 클릭 시 열리는 모달 상단
   - 배너 내용:
     - 미달성 시: "상세페이지 N개 더 보면 추가 응모권!" + 프로그레스 바
     - 달성 시: "추가 응모권 사용 가능! → 럭키드로우 바로가기"
   - 신규 UI 최소화: 기존 모달에 배너 컴포넌트 1개 추가

4. **CRM 연계 (선택)**
   - D+0 저녁: "오늘 상품 2개 봤어요. 1개만 더 보면 추가 응모권! → 상품 보러 가기"
   - D+1 아침: "오늘의 럭키드로우가 기다리고 있어요. 상품 구경하면서 추가 응모권도 받아보세요 →"
   - 대상: 앱 유저 + 로그인 유저 (푸시/이메일 식별 가능 유저)

5. **개발 규모**
   - 상세페이지 조회 카운트 로직 (프론트 세션 기반) — 1일
   - Recently Viewed 플로팅 버튼 툴팁 UI — 1일
   - Recently Viewed 모달 내 럭키드로우 배너 추가 — 0.5일
   - 응모권 지급 API 연동 — 1일
   - **총 3~4일** (기존 UI 재활용으로 신규 UI 개발 최소화)

6. **측정 지표**
   - 추가 응모권 획득율 (상세 N회 달성 유저 / 상세 1회 이상 조회 유저)
   - Recently Viewed 플로팅 버튼 CTR (before/after)
   - 럭키드로우 일 참여자 수 변화
   - "상품 2개 이상 탐색" 비율 변화
   - 교차탐색 유저의 RETURN_D1 재방문률
   - 럭키드로우 경유 구매 전환율

7. **미해결 사항**
   - **상세페이지 방문자의 고유 상품 조회 수 분포 실측** — 기존 데이터 역산으로 ~80.7% 추정했으나, GA4에서 정확한 분포(1개/2개/3개/4개/5개+) 확인 시 N값 최적화에 활용 가능
   - N값 최적화 (3회 vs 5회) — A/B 테스트 필요
   - 추가 응모권 일일 상한(1회) vs 탐색량 비례 지급 — 비용 시뮬레이션 필요
   - "같은 상품 반복 조회" 필터링 로직 세부 정의
   - 앱/웹 동시 적용 vs 앱 우선 적용 판단
   - CRM 연계 시 기존 CRM 시나리오(시나리오 1~5)와의 발송 빈도 조율
