# 파트너 어드민 대시보드

## 배경
### 문제 정의

현재 파트너 어드민에 대시보드가 없어 파트너가 로그인하면 스팟 수정 요청 목록으로 바로 진입한다. 예약 현황, 승인 대기, 정산 등 핵심 정보를 한눈에 파악할 수 없으며, 각 메뉴를 직접 들어가야 한다.

또한 좌측 네비게이션 메뉴가 크리에이트립 어드민 카테고리("여행", "모바일 전용") 안에 파트너 항목이 섞여 있어, 파트너 입장에서 자신에게 필요한 메뉴를 직관적으로 찾기 어렵다.

### 가설적 임팩트

대시보드를 통해 승인 대기 예약을 즉시 노출하고 일괄 승인 기능을 제공하면, 파트너의 예약 직접 처리율(현재 19.2%)을 개선할 수 있다.

최근 3개월 데이터 기준 운영팀이 수동 처리하는 예약은 전체의 52.4%(11,833건)이며, 미사용 파트너(Type D)의 1,689건이 100% 운영 의존 상태이다. 대시보드가 파트너의 어드민 접속 동기와 처리 편의성을 높이면, 이 중 일부가 파트너 직접 처리로 전환될 수 있다.

## 구현
### 유저 플로우

**대시보드 진입**
- 파트너가 로그인하면 대시보드(`/travel-partner/dashboard`)로 랜딩한다.
- 좌측 네비게이션에서 "대시보드" 메뉴를 클릭하여 언제든 돌아올 수 있다.

**예약 현황 파악**
- 파트너는 대시보드 상단에서 인사 메시지, 오늘 날짜, 이번 달 정산 예정액을 확인한다.
- 승인 대기, 오늘 방문 예정, 이번 달 확정, 이번 달 사용완료 카드를 통해 예약 현황을 한눈에 파악한다.

**상품별 예약 분포 확인**
- 기간과 스팟을 선택하여 상품별 예약 분포를 도넛 차트로 확인한다.
- 차트의 상품 영역을 클릭하면 해당 상품의 옵션별 분포를 드릴다운으로 확인한다.

**승인 대기 예약 처리**
- 대시보드 하단의 승인 대기 테이블에서 스팟별 탭으로 필터링한다.
- 개별 "승인" 버튼으로 건별 승인하거나, 체크박스로 여러 건을 선택하여 "선택 승인"으로 일괄 처리한다.
- "전체 보기 →" 링크를 클릭하면 예약내역 페이지로 이동한다.

**오늘 방문 예정 확인**
- 오늘 날짜 기준 방문 예정 예약 목록을 확인한다.
- 승인 대기 상태인 건은 승인 버튼이 노출되어 바로 처리할 수 있다.

**네비게이션 메뉴**
- 파트너 전용 메뉴 구조: 대시보드 / 예약내역 / 파트너쉽 정산 / 스팟 관리(수정 요청, 등록 요청, 리뷰 관리)
- 기존 "모바일 전용" 카테고리의 파트너 항목은 제거하고, 해당 페이지(예약내역, 스팟 수정 요청, 정산) 상단에 "모바일 뷰" 토글 버튼을 배치하여 모바일 UI로 전환한다.

### 프론트엔드 요구사항

**FR-1. 대시보드 페이지 신규 생성**
- 경로: `/travel-partner/dashboard`
- 권한: `TRAVEL_PARTNER`, `TRAVEL_PARTNER_SUPERIOR`
- 파트너 로그인 시 기본 랜딩 페이지로 설정

**FR-2. 네비게이션 메뉴 재구성**
- 파트너 전용 메뉴 트리를 기존 "여행" 카테고리에서 분리
- 새 구조:
  - 대시보드
  - 예약내역
  - 파트너쉽 정산
  - 스팟 관리 (서브메뉴: 스팟 수정 요청 / 스팟 등록 요청 / 스팟 리뷰 관리)
- "모바일 전용" 카테고리에서 파트너용 항목(스팟 수정 요청 모바일, 파트너쉽 정산 모바일) 제거

**FR-3. 모바일 뷰 토글**
- 예약내역, 스팟 수정 요청, 파트너쉽 정산 페이지 상단에 "모바일 뷰" 토글 버튼 배치
- 토글 시 기존 모바일 전용 컴포넌트(`*ForMobile`)로 인라인 전환 (별도 라우트 이동 없음)

**FR-4. 인사 + 정산 요약 섹션**
- 파트너명 + 오늘 날짜 표시
- 이번 달 정산 예정액 집계 노출

**FR-5. 상품별 예약 분포 차트**
- 기간 선택: date range picker
- 스팟 선택: 파트너 소유 스팟 목록 셀렉트
- 도넛 차트: 상품(SpotItem)별 예약 건수 및 비율
- 클릭 드릴다운: 선택한 상품의 옵션(SpotOption)별 분포 (progress bar 형태)
- 가이드 텍스트: "상품 영역을 클릭하면 옵션별 분포를 확인할 수 있습니다"

**FR-6. 예약 현황 요약 카드 (4개)**
- 승인 대기: 건수 + "지금 바로 처리할 수 있어요" + "처리하러 가기 →" 링크
- 오늘 방문 예정: 건수 + 승인 필요 건수 알림 + "처리하러 가기 →" 링크
- 이번 달 확정: 건수 + "직접 승인 N건" 부가 정보
- 이번 달 사용완료: 건수

**FR-7. 승인 대기 예약 테이블**
- 스팟별 탭 필터 (전체 + 각 스팟, 건수 뱃지 포함)
- 테이블 컬럼: 체크박스, 예약코드, 스팟, 예약자, 방문일, 방문시간, 인원, 상태, 승인 버튼
- 체크박스 선택 후 "선택 승인" 버튼으로 일괄 승인
- "전체 보기 →" 링크 → 예약내역 페이지 이동

**FR-8. 오늘 방문 예정 테이블**
- 오늘 날짜 기준 방문 예정 예약 목록
- 테이블 컬럼: 예약코드, 스팟, 예약자, 방문시간, 인원, 상태, (승인 대기면 승인 버튼)

### 백엔드 요구사항

**BR-1. 대시보드 집계 쿼리**
- 파트너의 이번 달 정산 예정액 집계
- 예약 현황 카드용 데이터: 승인 대기 건수, 오늘 방문 예정 건수(승인 필요 건수 포함), 이번 달 확정 건수(직접 승인 건수 포함), 이번 달 사용완료 건수

**BR-2. 상품별 예약 분포 쿼리**
- 입력: 기간(시작일~종료일), 스팟 코드
- 출력: 상품(SpotItem)별 예약 건수 및 비율, 옵션(SpotOption)별 예약 건수 및 비율

**BR-3. 승인 대기 예약 목록 쿼리**
- 파트너 소유 스팟의 승인 대기 예약 목록 반환
- 스팟별 필터링 지원
- 기존 예약 목록 쿼리 재활용 가능 여부 검토

**BR-4. 오늘 방문 예정 예약 목록 쿼리**
- 오늘 날짜 기준 방문 예정 예약 목록 반환
- 기존 예약 목록 쿼리에 날짜 필터 추가로 대응 가능 여부 검토

**BR-5. 일괄 승인 뮤테이션**
- 여러 예약 ID를 입력받아 일괄 승인 처리
- 기존 개별 승인 뮤테이션 재활용 또는 배치 뮤테이션 신규 생성 검토
