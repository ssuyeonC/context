# 세부 기획안: 스팟 수가표 업로드 기능

## 배경

### 문제 정의

- 피부과의 경우, 시술의 조합으로 구성된 상품이 정리된 '수가표'가 유저의 상품 구매의사 결정에 크게 영향을 행사함
- 현재 '상점소개' 영역의 텍스트 에디터에 수가표를 이미지 형태로 업로드하고 있지만, 해당 영역은 기본적으로 닫혀있어 수가표에 대한 유저의 접근성이 낮은 상태
- 수가표가 유저에게 디폴트로 노출되지 않아 구매 전환에 부정적 영향을 미칠 수 있음

### 가설적 임팩트

- 수가표를 디폴트 노출 영역에 배치함으로써 피부과 카테고리 스팟의 상품 구매 전환율 개선 기대
- 유저가 시술 가격 정보에 쉽게 접근할 수 있어 구매의사 결정 시간 단축 기대

## 구현

### 유저 플로우

- **어드민**: 카테고리 코드 404가 연결된 스팟 편집 시, '기본 선택 입력 정보' 영역에서 수가표 이미지를 업로드할 수 있다
- **유저페이지**: 스팟 상세 페이지 접근 시, 업로드된 수가표가 'AI Review Summary' 섹션에 디폴트로 노출된다

### 프론트엔드 요구사항

**어드민**

- 카테고리 코드가 404인 카테고리가 연결된 스팟의 경우, '기본 선택 입력 정보' 영역 내 '수가표 업로드' 섹션 추가
  - 이미지 업로드 기능 제공

**유저페이지**

- 업로드된 수가표는 'AI Review Summary' 섹션에 노출
- 수가표 이미지는 가로길이 기준 자동 맞춤 (다른 컨텐츠와 동일한 방식)
- 수가표 이미지 Expand/Collapse 처리
  - 디폴트 상태: 이미지 상단 200~240px만 노출, 하단에 그라데이션 오버레이(white fade-out) 적용
  - 하단에 'READ MORE' 버튼 배치, 클릭 시 이미지 전체 노출 (height transition 애니메이션 적용)
  - 이미지 실제 높이가 디폴트 높이 이하인 경우 'READ MORE' 버튼 및 그라데이션 미노출

### 백엔드 요구사항

- 수가표 이미지 저장/조회를 위한 데이터 모델 및 API 대응
  - 스팟별 수가표 이미지 업로드 (최소 0개)
  - 스팟 상세 조회 시 수가표 이미지 정보 반환
- 카테고리 코드 404 연결 여부에 따른 수가표 필드 노출 제어
