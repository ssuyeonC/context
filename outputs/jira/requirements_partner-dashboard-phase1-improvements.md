# 파트너 어드민 대시보드 1차 개선 요구사항

## 배경

### 문제 정의

현재 파트너 어드민 대시보드는 다음과 같은 한계가 있다.

- 진입 직후 첫 화면을 4개 KPI 카드 그리드와 파이 차트가 차지하고 있어, "오늘의 할 일" 테이블이 fold 아래로 밀려 있다. 파트너가 처리해야 할 예약이 진입 직후 시야에 들어오지 않는다.
- 상단 스팟 필터가 파이 차트에만 적용되며, '이번 달 확정 / 사용완료' 등 KPI 카드는 항상 보유 전체 스팟 합산으로만 노출된다. 다중 스팟을 운영하는 파트너가 스팟 단위 현황을 단독으로 파악할 수 없다.
- 파이 차트의 표시 단위가 '상품' 고정이라, 다중 스팟 파트너가 "어느 스팟의 비중이 가장 큰가"부터 보고 싶을 때 두 번 이상의 인터랙션이 필요하다.
- '오늘 방문 예약' 카드의 버튼이 승인 필요 여부와 무관하게 '처리하러 가기' 한 가지 워딩으로만 노출된다. 파트너가 실제로 처리할 일이 있는지 카드만 봐서는 알 수 없다.
- 회고용 영역(파이 차트 + 월간 KPI)에 섹션 헤더가 없어 '오늘의 할 일' 섹션과 시각적 위계가 약하고, 어떤 영역이 필터 적용을 받는지 안내가 부족하다.

근거 데이터: 대시보드 391세션의 첫 클릭 분포 — 사이드바 "예약내역"으로 이탈 61%, 테이블 행 첫 클릭 0.8%, 인라인 승인 0.3%. 액션 영역이 발견되지 못한 채 사용자가 떠나는 구조적 문제.

### 가설적 임팩트

- 페이지 진입 직후 알림 배너와 두 테이블이 시야에 들어와, 사이드바로 즉시 이탈하던 사용자 중 일부가 대시보드에서 처리를 완료한다.
- 하단 회고 영역의 스팟 필터 한 번으로 월간 KPI와 차트가 동시에 해당 스팟 기준으로 전환되어, 다중 스팟 파트너의 의사결정 클릭 수가 감소한다.
- '처리하러 가기' / '확인하러 가기' 워딩 분기로 배너만 봐도 처리 필요 여부를 식별할 수 있어, 불필요한 스크롤·클릭이 감소한다.
- 섹션 헤더와 디스크립션으로 "어떤 영역이 필터 적용을 받고, 어떤 영역이 항상 전체 기준인지"가 명확해져 학습 비용이 감소한다.

## 구현

### 유저 플로우

**대시보드 진입**

- 파트너가 대시보드에 진입하면 화면 첫 영역에 인사말 + '승인 대기 / 오늘 방문 예약' 알림 배너 2개가 노출된다.
- 그 아래로 '오늘의 할 일' 섹션이 이어진다. 오늘 방문 예약 테이블, 승인 대기 예약 테이블 두 개가 순서대로 노출된다.
- 페이지 하단으로 스크롤하면 '이번 달 운영 현황' 회고 섹션(필터, 차트, 월간 KPI 카드)이 나타난다.

**알림 배너의 데이터 범위**

- 상단 '승인 대기' / '오늘 방문 예약' 알림 배너는 **항상 파트너 보유 전체 스팟 기준**으로 집계된다.
- 하단 '이번 달 운영 현황'의 필터 변경에 영향을 받지 않는다.

**오늘 방문 예약 배너의 버튼 분기**

- 오늘 방문 예약이 1건 이상 + 그중 승인 대기가 1건 이상 → 버튼 '처리하러 가기 →' 노출.
- 오늘 방문 예약이 1건 이상 + 승인 대기 0건 → 버튼 '확인하러 가기 →' 노출.
- 오늘 방문 예약 0건 → 버튼 미노출.
- 버튼 클릭 시 '오늘의 할 일' 섹션의 오늘 방문 예약 테이블 위치로 페이지 내 스크롤.

**오늘의 할 일 섹션**

- 두 테이블 모두 행 단위로 '승인하기' 버튼을 노출한다. 승인 대기 상태인 행에만 버튼이 활성화된다.
- 섹션 헤더 하단에 "모든 스팟의 예약 건 정보가 조회됩니다." 디스크립션을 표시하여 항상 전체 기준임을 명시한다.

**이번 달 운영 현황 섹션**

- 스팟 Select 기본값은 '전체'.
- '전체' 선택 상태
  - 파이 차트: 보유한 모든 스팟의 '스팟별 예약 분포' 표시.
  - 월간 KPI 카드(이번 달 확정 / 사용완료): 모든 스팟 합산 수치 표시.
- 특정 스팟 선택 상태
  - 파이 차트: 해당 스팟의 '상품별 예약 분포' 표시.
  - 월간 KPI 카드: 해당 스팟만 집계한 수치 표시.
- 이 섹션의 필터는 회고 섹션 내부에만 적용되며, 상단 알림 배너 및 '오늘의 할 일' 테이블에는 영향을 주지 않는다.

### 프론트엔드 요구사항

**FR-1. 페이지 레이아웃 재구성**

- 위에서 아래 순서로 배치: 헤더 → 알림 배너 2개 → '오늘의 할 일' 섹션 → '이번 달 운영 현황' 섹션.
- 기존 상단을 점유하던 4-카드 그리드와 파이 차트를 회고 섹션 내부로 이동.

**FR-2. 상단 알림 배너 영역**

- 풀-너비 가로 배너 2개 (승인 대기 / 오늘 방문 예약). 카드 그리드 형태 아님.
- 각 배너는 카운트 + 한 줄 설명 + 우측 액션 버튼 구성.
- 항상 보유 전체 스팟 기준으로 집계.
- '오늘 방문 예약' 배너의 버튼 워딩 분기는 유저 플로우 참조.
- '승인 대기' 배너의 버튼은 0건일 때 비노출, 1건 이상일 때 '처리하러 가기 →' 노출. 클릭 시 '오늘의 할 일' 섹션의 승인 대기 예약 테이블 위치로 스크롤.

**FR-3. '오늘의 할 일' 섹션 헤더 및 디스크립션**

- 섹션 헤더 워딩: "오늘의 할 일".
- 헤더 하단에 디스크립션 한 줄 추가: "모든 스팟의 예약 건 정보가 조회됩니다.".

**FR-4. '오늘의 할 일' 테이블 — 인라인 승인 버튼**

- 오늘 방문 예약 테이블, 승인 대기 예약 테이블 모두 마지막 컬럼으로 '승인하기' 버튼을 노출.
- 행 상태가 '승인 대기'일 때만 버튼 활성화. 그 외 상태에서는 버튼 비활성화 또는 비노출.
- 두 테이블 모두 데이터 집계 범위는 보유 전체 스팟 기준 (필터 미적용).

**FR-5. '이번 달 운영 현황' 섹션 헤더 신규 추가**

- 섹션 헤더 워딩: "이번 달 운영 현황".
- 헤더 UI는 '오늘의 할 일' 섹션 헤더와 동일한 스타일 유지.

**FR-6. '이번 달 운영 현황' 필터**

- 스팟 Select에 '전체' 옵션 추가.
- 기본 선택값을 '전체'로 변경.
- 필터 적용 범위는 이 섹션 내부의 파이 차트 + 월간 KPI 카드로 한정.

**FR-7. 파이 차트 — 스팟 단위 표시 모드 추가**

- '전체' 선택 시 — 스팟별 예약 분포 표시
  - 차트 타이틀: '스팟별 예약 분포'.
  - 범례: 스팟명 기준.
  - 슬라이스 클릭 시 팝업 타이틀: '{스팟명} - 상품별 분포', 해당 스팟의 상품 단위 breakdown 표시.
  - 도움말 문구: "스팟 영역을 클릭하면 상품별 분포를 확인할 수 있습니다".
- 특정 스팟 선택 시 — 상품별 예약 분포 표시
  - 기존 동작 유지. 타이틀 '상품별 예약 분포', 범례 상품명 기준.

**FR-8. 월간 KPI 카드**

- 카드 2개: 이번 달 확정 / 이번 달 사용완료.
- '이번 달 확정' 카드는 기존의 '직접 승인율' 보조 표시 유지.
- 필터 선택 스팟에 따라 집계 범위 전환.

### 백엔드 요구사항

**BR-1. 파트너 대시보드 요약 집계에 스팟 단위 조회 모드 추가**

- 기존 요약 집계 API에 '스팟 코드' 입력값을 선택적으로 추가.
- 입력값이 없으면 기존 동작(보유 전체 스팟 합산) 유지.
- 입력값이 있으면 해당 스팟만 집계.
- 인가 검증: 요청한 파트너가 해당 스팟을 관리하지 않는 경우 권한 오류 반환.
- 응답 필드 자체는 기존과 동일. 집계 범위만 입력값에 따라 분기.

**BR-2. 알림 배너용·회고 카드용 동일 집계 API의 이중 호출 허용**

- 동일 요약 API가 한 페이지에서 두 번 호출되는 패턴을 허용.
  - 알림 배너: 스팟 입력값 없이 호출 (항상 전체 기준).
  - 회고 KPI 카드: 필터 선택 스팟 입력값과 함께 호출.
- 응답 캐시는 입력값별로 분리되어야 한다.

**BR-3. 스팟별 예약 분포 집계 API**

- '전체' 모드 차트용 신규 데이터 — 파트너 보유 스팟 단위로 그룹핑된 예약 분포(스팟 코드, 스팟명, 예약 건수, 비율).
- 정렬 기준: 예약 건수 내림차순.
- 인가 검증: 요청 파트너가 보유하지 않은 스팟은 응답에 포함되지 않음.

**BR-4. 상품별 예약 분포 집계 API — 기존 유지**

- 특정 스팟 모드에서 사용되는 기존 상품별 분포 API는 동작 변경 없음.
- 파이 차트가 '전체' 모드의 슬라이스 클릭 → 특정 스팟 모드로 전환될 때 그대로 재사용.

## 비범위(Out of Scope)

- '오늘의 할 일' 섹션 두 테이블에 대한 필터 연동은 1차 범위 외. 디스크립션으로 "모든 스팟 기준"임을 안내하는 것으로 갈음.
- 기간(DateRange) 필터를 KPI 카드/테이블로 확장 적용하는 작업도 1차 범위 외.
- 알림 배너 디자인의 색상/아이콘/문구 디테일 변경은 디자인 단계에서 별도 정의.

## 검증 포인트

- 페이지 진입 시 위에서 아래 순서가 헤더 → 알림 배너 → 오늘의 할 일 → 이번 달 운영 현황 순으로 렌더되는지.
- 상단 알림 배너 2개가 항상 보유 전체 스팟 기준 수치를 보여주며, 하단 필터를 바꿔도 변하지 않는지.
- '이번 달 운영 현황' 필터 기본값이 '전체'로 진입되는지.
- 필터를 특정 스팟으로 바꿨을 때 월간 KPI 카드 2개와 파이 차트가 해당 스팟 기준 수치로 전환되는지. (해당 스팟만 직접 카운트한 값과 일치하는지 교차 검증).
- '오늘 방문 예약' 배너 버튼이 오늘 예약 0건 / 오늘 예약 있고 승인 대기 0건 / 오늘 예약 있고 승인 대기 1건 이상 세 케이스에서 각각 '비노출 / 확인하러 가기 / 처리하러 가기'로 정확히 표시되는지.
- '전체' 모드에서 파이 차트의 범례·팝업·도움말이 스팟 기준, 특정 스팟 모드에서는 상품 기준으로 표시되는지.
- 파트너가 관리하지 않는 스팟 코드를 직접 입력하여 호출 시 권한 오류가 반환되는지.
- '오늘의 할 일' 두 테이블의 행 단위 '승인하기' 버튼이 승인 대기 상태에서만 활성화되는지.
