# K뷰티 시술관리 페이지 개선

## 배경
### 문제 정의

현재 시술관리(`/beauty-treatment`) 기능은 중분류 피부과(404) 카테고리에만 한정되어 있으나, 시력교정 등 다른 도메인에서도 시술 정보 관리 수요가 발생하고 있다.

피부과의 경우 수가표 이미지 업로드 기능과 시술관리 데이터가 중복되는 케이스가 존재하여, 시술 가격 정보를 유저 페이지에서 선택적으로 숨김 처리할 수 있는 기능이 필요하다. 숨김 처리된 시술 정보는 상세 페이지에서는 비노출되지만, 검색 결과나 리스트 페이지에서 필터 조건으로는 여전히 활용된다.

도메인 확장에 따라 시술 연결 관리 모달의 UX/UI도 함께 개선이 필요하다.

또한 유저 페이지 검색 결과 카드 하단에서 K뷰티 시술 정보가 매칭될 때, 시술에 연결된 부가정보의 수에 따라 노출 포맷이 일관되지 않아 정리가 필요하다.

### 가설적 임팩트

- 시력교정 등 신규 도메인 확장으로 시술관리 기능의 활용 범위 확대
- 수가표 이미지와의 중복 문제 해결로 피부과 스팟 상세 페이지의 정보 정합성 개선
- 대분류 카테고리 기준 필터 추가로 운영 효율성 향상
- 검색 결과 카드의 K뷰티 시술 정보 노출 포맷 정리로 유저 탐색 경험 개선

## 구현
### 유저 플로우

- 어드민은 시술관리 페이지에서 대분류 카테고리(피부과, 시력교정) 기준 필터 탭을 통해 해당 카테고리에 속한 스팟만 조회한다.
- 어드민은 시술 연결 관리 상세 모달에서 '시술 가격 정보 비공개' 토글을 활성화하여, 해당 스팟에 연결된 시술 및 부가정보를 유저 페이지 상세에서 숨길 수 있다.
- 숨김 처리된 시술 정보는 유저 페이지의 검색/리스트에서 필터 조건으로는 계속 동작한다.
- 어드민은 시술 연결 관리 상세 모달에서 각 시술 정보를 collapse 형태로 확인하며, 디폴트는 접힌 상태이다.
- 유저가 검색창에 시술명을 입력하면, 매칭된 스팟 카드 하단에 K뷰티 시술 정보가 연결되어 있을 경우 해당 시술명이 우선 노출되며, 부가정보 개수에 따라 포맷이 달라진다.

### 프론트엔드 요구사항

**FR-0. 탭 이름 변경**
- K뷰티 시술관리 → 시술 정보 관리

**FR-1. 시술 관리 대상 카테고리 변경**
- 현재: 중분류 피부과(404)에 해당하는 스팟만 노출
- 변경: 대분류 피부과(3068), 대분류 시력교정(3099)에 해당하는 스팟 노출
- 시술 연결 관리에서 스팟 추가 시에도 대분류 피부과(3068), 대분류 시력교정(3099) 카테고리를 가진 스팟만 조회 대상으로 제한

**FR-2. 시술 연결 관리 탭 하단 카테고리 필터 추가**
- 추가된 스팟의 대분류 카테고리 기준 필터 탭 배치
- 각 탭에 카테고리 한국어 이름 및 해당 카테고리에 속한 스팟 수 병기 (예: "피부과 (12)", "시력교정 (5)")

**FR-3. 시술 연결 관리 상세 모달 — '시술 가격 정보 비공개' 토글 추가**
- 모달 내 토글 UI 배치
- 토글 true: 연결된 시술 및 부가정보 데이터가 유저 페이지 상세에서 숨김 처리
- 토글 false: 기존과 동일하게 노출

**FR-4. 시술 연결 관리 상세 모달 — 시술 정보 Collapse UI 변경**
- 각 시술 정보를 collapse(아코디언) 형태로 변경
- 디폴트 상태: 접혀 있음 (닫힘)
- 클릭하여 펼치면 시술 상세 정보 노출

**FR-5. 유저 페이지 검색 결과 카드 — K뷰티 시술 정보 매칭 시 노출 포맷 정리**

검색어와 매칭되는 K뷰티 시술 정보가 있을 때(1순위 분기), 매칭된 시술에 연결된 부가정보 개수에 따라 아래와 같이 노출한다.

| 매칭된 시술의 부가정보 수 | 노출 형태 |
|---|---|
| 1개 | `시술명 \| 부가정보 \| 가격` |
| 2개 이상 | `{시술명} products total {n} items` |

- 부가정보 1개: `시술명 | 부가정보 | 가격` 형식으로 노출
- 부가정보 2개 이상: `{시술명} products total {n} items` 형식으로 노출
    - `n` = 해당 시술에 연결된 부가정보의 수
    - 검색어와 매칭되는 옵션이 2개 이상일 때의 기존 노출 형태와 동일한 UI 재사용

### 백엔드 요구사항

**BR-1. 시술 관리 대상 카테고리 로직 변경**
- 어드민 시술 관리 데이터 조회 시 필터 조건을 중분류 피부과(404)에서 대분류 피부과(3068), 대분류 시력교정(3099)으로 변경
- 어드민 시술 연결 관리에서 스팟 추가 시 대분류 피부과(3068), 대분류 시력교정(3099) 카테고리를 가진 스팟만 조회되도록 필터 적용
- 유저 페이지 스팟 상세에서 시술 정보 노출 조건을 중분류 피부과(404)에서 대분류 피부과(3068), 대분류 시력교정(3099)으로 변경

**BR-2. '시술 가격 정보 비공개' 필드 추가**
- 스팟-시술 연결 데이터에 비공개 여부 필드 추가
- 비공개 true: 유저 페이지 상세 조회 시 시술 및 부가정보 데이터 미반환
- 비공개 false: 기존과 동일하게 반환
- 검색/리스트 필터 로직에서는 비공개 여부와 무관하게 시술 데이터 활용 (필터 조건으로는 노출)

**BR-3. 카테고리별 스팟 수 집계 쿼리**
- 시술이 연결된 스팟을 대분류 카테고리 기준으로 그룹핑하여 카테고리별 스팟 수 반환
- FR-2의 필터 탭 뱃지 표시에 활용

**BR-4. 검색 결과 — 매칭된 K뷰티 시술의 부가정보 응답 확장**
- 검색 API에서 매칭된 시술(`beautyProcedures`)에 대해, 시술별로 연결된 부가정보 목록(또는 개수)과 대표 부가정보의 가격 정보를 함께 반환
- 부가정보 1개인 경우 FR-5의 `시술명 | 부가정보 | 가격` 노출에 필요한 필드(부가정보명, 가격) 포함
- 부가정보 2개 이상인 경우 FR-5의 `products total {n} items` 노출에 필요한 개수(`n`) 포함
- 비공개 처리된 시술(BR-2)은 검색 결과 노출 로직에서 제외하지 않음 (필터 조건으로는 계속 활용)
