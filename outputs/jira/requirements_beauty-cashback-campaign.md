# 세부 기획안 요구사항 문서 양식

## 배경
### 문제 정의

현재 뷰티캐시백 기능은 모든 스팟에 동일한 환급률(10%)과 단일 가이드 문서를 제공한다. 환급률 "10%"는 프론트 i18n 문자열에 하드코딩되어 있고, 가이드 본문은 `beauty_receipt_notice` DB 테이블에 언어별 1 row로 저장되어 있으나 이를 편집할 어드민 UI가 없어 DB를 직접 수정해야 한다.

향후 확정된 외부 프로모션이 있다.

- **부산관광공사 웰니스의료 프로모션** (2026-05-25 오픈 예정): 부산 피부과 등 특정 스팟에 추가 5% 캐시백(총 15%) 및 프로모션 전용 가이드 제공 필요
- **2026 코리아뷰티페스티벌** (2026-06월 중 오픈 예정): 10억원 규모 캐시백 예산 배정, 무료 예약 피부과 대상 캐시백 혜택 제공

위 프로모션을 기존 기능으로는 스팟별 환급률 차등 적용 / 스팟별 가이드 문서 차등 적용이 불가하다. 하드코딩 대응은 프로모션이 반복될수록 배포 리스크와 휴먼에러 위험이 누적된다. 캐시백은 금액이 크고 유저에게 잘못 안내될 경우 환불/배상 리스크가 크다.

### 가설적 임팩트

- 운영팀이 개발 요청 없이 어드민에서 직접 캠페인 가이드를 생성/수정/적용할 수 있어 프로모션 대응 리드타임 단축
- 스팟별 환급률/가이드가 DB에서 일관되게 관리되어 유저 오안내로 인한 배상 리스크 감소
- 향후 유사 프로모션 진행 시 재사용 가능한 구조 확보

## 구현
### 유저 플로우

**어드민 운영자**
1. 어드민 스팟 상세의 '뷰티캐시백' 영역에서 '뷰티캐시백 여부' 토글을 ON으로 전환한다 (토글 ON 시 기본 가이드가 자동 적용됨)
2. 토글 ON 상태에서 노출되는 '캠페인 관리' 버튼을 클릭해 가이드 관리 모달을 연다
3. 모달 내에서 다음 작업을 수행한다
   - 현재 이 스팟에 적용된 가이드 확인
   - 전체 가이드 리스트 조회 (각 가이드별 연결된 스팟 수 표시)
   - 신규 가이드 생성 (이름, 환급률, 멤버십 환급률, 본문, 언어별 번역 — 자동번역 기능 재사용)
   - 기존 가이드 수정/삭제
   - 가이드별 연결된 스팟 목록 확인
   - 이 스팟에 적용할 가이드 선택 및 저장 (기본 가이드 포함)
4. 프로모션 종료 시 같은 모달에서 수동으로 스팟의 가이드를 '기본 가이드'로 되돌리거나, 해당 가이드의 "연결된 스팟 일괄 해제" 기능을 사용한다

**최종 유저 (웹)**
1. 스팟 상세 페이지에서 뷰티캐시백 뱃지를 본다
   - 해당 스팟에 연결된 가이드의 환급률이 동적으로 표시됨 (예: "Earn 15% Back")
   - 멤버십 환급률 안내도 가이드 값에 따라 표시됨
2. 뱃지 또는 "Beauty Cashback Guide" 버튼 클릭 시 모달이 열리며, 해당 스팟에 연결된 가이드 본문이 표시된다

### 프론트엔드 요구사항

**어드민 - 스팟 상세 '뷰티캐시백' 영역**

- 파일: `frontend/apps/admin/src/components/spot/SpotBasicInfo/ToggleSpotBeautyCashBackSwitch.tsx` 및 인근 컴포넌트
- 변경 내용
  - '뷰티캐시백 여부' 토글이 ON인 경우 '캠페인 관리' 버튼을 토글 옆에 노출
  - 토글이 OFF로 변경되면 해당 스팟의 가이드 연결 자동 해제 (기본 가이드 포인터도 제거)
  - 토글 ON 시 별도 설정이 없으면 기본 가이드가 적용되는 것으로 간주 (DB 상 `beauty_guide_id`는 NULL 유지)

**어드민 - 뷰티캐시백 캠페인 관리 모달 (신규)**

'캠페인 관리' 버튼 클릭 시 열리는 단일 모달에서 아래 기능을 모두 제공한다.

1. 현재 스팟 적용 가이드 섹션
   - 이 스팟에 현재 적용된 가이드 이름 및 환급률 표시 (기본 적용 시 "기본 가이드" 표기)
   - '적용 가이드 변경' 드롭다운: 기본 가이드 + 생성된 가이드 중 선택하여 저장
   - 저장 시 해당 스팟의 `beauty_guide_id` 업데이트 (기본 가이드 선택 = NULL)

2. 가이드 리스트 섹션
   - 컬럼: 이름, 환급률(%), 멤버십 환급률(%), 연결 스팟 수, 작업(편집/삭제)
   - 기본 가이드는 삭제 불가, 이름 변경 불가 (환급률/본문/번역은 편집 가능)
   - '신규 가이드 생성' 버튼: 생성 폼으로 전환

3. 가이드 생성/수정 폼 (모달 내 뷰 전환)
   - 입력 필드: 이름(내부 식별용), 환급률(숫자), 멤버십 환급률(숫자), 본문(markdown)
   - 언어별 번역 탭: 운영 중인 모든 언어(ko, en, ja, zh_CN, zh_TW, id, vi, th, ms 등) 필수 입력
   - 자동번역 버튼: 어드민 기존 자동번역 파이프라인 재사용 (스팟/블로그 등에서 사용 중)
   - 모든 언어 번역값이 입력되지 않으면 저장 불가 validation

4. 가이드 상세 뷰 (편집 버튼 클릭 시)
   - 기본 정보 (이름/환급률/멤버십 환급률/본문/번역)
   - "적용 중인 스팟" 섹션: 스팟코드, 스팟명 리스트, pagination 제공
   - "연결된 스팟 일괄 해제" 버튼: 확인 후 해당 가이드에 연결된 모든 스팟을 기본 가이드로 전환
   - 삭제 시 연결된 스팟이 있으면 에러 또는 일괄 해제 후 삭제 확인 요구

**웹 - 스팟 상세 뷰티캐시백 뱃지**

- 파일: `frontend/apps/web/domain/travel/subdomain/spot/detail/SpotDetailHighlightInfo.tsx` (`BeautyCashbackContent` 함수, line 216-275)
- 변경 내용
  - 하드코딩된 "10%" 제거: `descriptionText.split('10%')` 방식 폐기
  - 스팟에 연결된 가이드의 환급률을 GraphQL로 조회하여 동적으로 표시
  - i18n 키 수정 예시: `beauty-cashback.title-flag` → "Earn {rate}% Back", `beauty-cashback.description` → "Get {rate}% back on beauty reservations"
  - 멤버십 안내 문구(`membership.beauty-cashback-description`)도 가이드의 멤버십 환급률 값으로 치환
  - 모든 운영 언어 재번역 (자동번역 파이프라인 사용)

**웹 - 뷰티캐시백 가이드 모달**

- 파일: `frontend/apps/web/domain/shared/BeautyCashbackGuideModal/BeautyCashbackGuideModal.tsx`
- 변경 내용
  - 기존 `GetBeautyReceiptNotice` 쿼리를 스팟 기준으로 조회하도록 파라미터 추가 (`spotCode` 또는 `spotId`)
  - 스팟에 연결된 가이드의 본문을 반환
  - 연결된 가이드가 없거나 스팟 정보가 없는 호출(예: 마이페이지 영수증 안내)은 기본 가이드 본문을 반환

### 백엔드 요구사항

**1. 테이블**

- 기본 가이드 테이블 (기존 뷰티 캐시백 가이드 테이블 활용)
  - 환급률과 멤버십 환급률을 숫자로 보관하는 필드 필요
  - 기존 고정값(10%, 15%)을 기본값으로 가지도록 마이그레이션
- 뷰티캐시백 가이드 테이블 (신규)
  - 어드민이 생성한 가이드를 저장 (내부 식별용 이름, 환급률, 멤버십 환급률, 본문)
- 뷰티캐시백 가이드 번역 테이블 (신규)
  - 가이드 1개 ↔ 지원 언어 N개의 번역값을 저장
- 스팟 테이블 확장
  - 스팟이 어떤 가이드를 사용하는지 참조하는 필드 필요 (비어있으면 기본 가이드 적용)

**2. 쿼리**

- 가이드 리스트 조회: 어드민 모달에서 전체 가이드 + 각 가이드별 연결된 스팟 수를 보기 위함
- 가이드 상세 조회: 가이드 기본 정보 + 언어별 번역 + 연결된 스팟 리스트 (pagination)
- 스팟 기준 가이드 조회: 프론트 스팟 상세/가이드 모달에서 사용. 해당 스팟에 연결된 가이드를 반환하고, 없으면 기본 가이드 반환

**3. 뮤테이션**

- 가이드 생성: 이름/환급률/본문/전체 언어 번역값을 한 번에 저장
- 가이드 수정: 기본 정보 및 번역값 수정
- 가이드 삭제: 연결된 스팟이 있으면 차단 또는 일괄 해제 후 삭제
- 스팟 ↔ 가이드 연결 변경: 어드민 모달의 '적용 가이드 변경' 동작에 사용. 기본 가이드 선택 시 연결 해제로 처리
- 가이드에 연결된 스팟 일괄 해제: 프로모션 종료 시 해당 가이드에 물린 스팟 전부를 기본 가이드로 되돌림

**4. 조회 로직**

- 스팟의 뷰티캐시백 여부가 꺼져 있으면 뷰티캐시백 UI 자체를 노출하지 않음
- 켜져 있는 경우
  - 스팟에 연결된 가이드가 있으면 해당 가이드의 환급률/본문을 사용
  - 연결된 가이드가 없으면 기본 가이드의 환급률/본문을 사용

**5. 참고 사항**

- 환급금 계산/지급은 유관 운영팀이 수동으로 처리하므로 환급 계산 로직 변경 불필요
- 환급률은 오직 프론트 뱃지·가이드 노출용 수치로만 사용됨
- 기간 기반 자동 전환은 이번 범위에서 제외. 운영자가 어드민 모달에서 수동 연결/해제. 향후 기간 필드 추가로 확장 가능
- 멤버십 환급률은 가이드별로 별도 관리 (현재 전역 15%도 가이드에 따라 달라질 수 있음)
