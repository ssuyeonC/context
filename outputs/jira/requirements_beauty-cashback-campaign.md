# 세부 기획안 요구사항 문서 양식

## 배경
### 문제 정의

현재 뷰티캐시백 기능은 모든 스팟에 동일한 환급률(10%)과 단일 가이드 문서를 제공한다. 환급률 "10%"는 프론트 i18n 문자열에 하드코딩되어 있어 환급률을 바꾸려면 코드 수정 + 배포가 필요하고, 가이드 본문은 `beauty_receipt_notice` DB 테이블에 **언어별 1 row** 형태로만 저장되어 있어 캠페인별로 다른 본문을 둘 수 없는 구조이다.

향후 확정된 외부 프로모션이 있다.

- **부산관광공사 웰니스의료 프로모션** (2026-05-25 오픈 예정): 부산 피부과 등 특정 스팟에 추가 5% 캐시백(총 15%) 및 프로모션 전용 가이드 제공 필요
- **2026 코리아뷰티페스티벌** (2026-06월 중 오픈 예정): 10억원 규모 캐시백 예산 배정, 무료 예약 피부과 대상 캐시백 혜택 제공

기존 구조로는 스팟별 환급률 차등 적용 / 스팟별 가이드 본문 차등 적용이 불가하며, 환급률이 i18n에 박혀 있는 한 캠페인이 반복될수록 매번 배포가 필요해 배포 리스크와 휴먼에러가 누적된다. 캐시백은 금액이 크고 유저에게 잘못 안내될 경우 환불/배상 리스크가 크다.

### 가설적 임팩트

- 환급률·캠페인 메타가 데이터로 분리되어 캠페인 추가·환급률 변경 시 **배포 없이 DB INSERT/UPDATE만으로 대응 가능**
- 스팟-캠페인 매핑으로 환급률·가이드 본문을 스팟마다 차등 적용 → 부산·페스티벌과 같은 **부분 적용 프로모션이 구조적으로 가능**
- 슬랙 알림에 적용 캠페인명·환급률이 명시되어 CS/운영팀의 환급 처리 정확도 향상 (오안내·오환급 리스크 감소)

### 본 단계의 범위 제한

- 어드민 캠페인 관리 UI는 만들지 않는다. 운영팀의 캠페인 추가·수정·매핑 변경 요청은 **매주 정기 미팅 공유 → 엔지니어 DB 직접 반영** 흐름으로 처리한다
- 본 운영 흐름은 캠페인 빈도가 낮은(연 2-3건 수준) 동안 유효하다. 빈도가 누적되거나 매핑 변경 요청이 잦아지면 후속으로 어드민 모달 또는 시트 sync 도입을 재검토한다

## 구현
### 유저 플로우

**어드민 운영자**

본 단계에서는 어드민에 캠페인 관리 UI를 신설하지 않는다. 운영팀이 캠페인 신규 추가/수정/매핑 변경이 필요할 때마다 **매주 정기 미팅에서 변경사항을 공유**하고, 엔지니어가 DB에 직접 반영한다.

1. 어드민 스팟 상세의 '뷰티캐시백' 영역에서 '뷰티캐시백 여부' 토글을 ON/OFF로 전환한다 (기존 동작 유지)
2. 토글 ON 상태에서는 해당 스팟에 매핑된 캠페인이 있으면 그 캠페인의 환급률·본문이 적용되고, 매핑이 없으면 기본 캠페인이 적용된다 (어드민에는 현재 적용 캠페인명·환급률이 읽기 전용으로 표시되어 운영자가 매핑 상태를 즉시 확인 가능)
3. 토글 OFF 시에는 단순히 웹/슬랙에 뷰티캐시백 UI·알림이 노출되지 않을 뿐이며, 캠페인 매핑(`beauty_cashback_spot` row)은 그대로 유지된다 (다시 ON으로 돌렸을 때 이전 매핑이 그대로 적용됨). 매핑 자체의 추가/변경/해제는 엔지니어 DB 반영으로만 처리한다
4. 신규 캠페인 추가, 환급률 조정, 본문 수정, 스팟 매핑 변경은 운영팀의 정기 미팅 공유 → 엔지니어 DB 반영 흐름으로 처리한다 (DB 현황은 매주 미팅 때 엔지니어가 운영팀과 동기화)

**최종 유저 (웹)**

1. 스팟 상세 페이지에서 뷰티캐시백 뱃지를 본다
   - 해당 스팟에 매핑된 캠페인의 환급률이 동적으로 표시됨 (예: "Earn 15% Back")
   - 멤버십 환급률 안내도 캠페인 값에 따라 표시됨
2. 뱃지 또는 "Beauty Cashback Guide" 버튼 클릭 시 모달이 열리며, 해당 스팟에 매핑된 캠페인의 본문이 표시된다 (매핑 없는 경우 기본 캠페인 본문)

### 프론트엔드 요구사항

**어드민 - 스팟 상세 '뷰티캐시백' 영역**

- 파일: `frontend/apps/admin/src/components/spot/SpotBasicInfo/ToggleSpotBeautyCashBackSwitch.tsx` 및 인근 컴포넌트
- 변경 내용
  - '뷰티캐시백 여부' 토글 ON 상태에서 해당 스팟에 적용된 **캠페인명·환급률을 읽기 전용 텍스트로 표시** (운영자 디버깅·확인용. 편집 컨트롤 없음)
  - 토글 OFF 시 캠페인 매핑은 건드리지 않는다 (매핑 row는 그대로 두고, 뷰티캐시백 UI·슬랙 알림 노출만 차단). 매핑 해제는 엔지니어 DB 반영으로 처리
  - 본 단계에서는 캠페인 관리 모달 / 가이드 CRUD UI / 자동번역 UI를 만들지 않는다

**웹 - 스팟 상세 뷰티캐시백 뱃지**

- 파일: `frontend/apps/web/domain/travel/subdomain/spot/detail/SpotDetailHighlightInfo.tsx` (`BeautyCashbackContent` 함수, line 216-275)
- 변경 내용
  - 하드코딩된 "10%" 제거: `descriptionText.split('10%')` 방식 폐기
  - 스팟에 매핑된 캠페인의 환급률을 GraphQL로 조회하여 동적으로 표시
  - i18n 키 수정 예시: `beauty-cashback.title-flag` → "Earn {rate}% Back", `beauty-cashback.description` → "Get {rate}% back on beauty reservations"
  - 멤버십 안내 문구(`membership.beauty-cashback-description`)도 캠페인의 멤버십 환급률 값으로 치환
  - 모든 운영 언어 재번역 (자동번역 파이프라인 사용)

**웹 - 뷰티캐시백 가이드 모달**

- 파일: `frontend/apps/web/domain/shared/BeautyCashbackGuideModal/BeautyCashbackGuideModal.tsx`
- 변경 내용
  - 기존 `GetBeautyReceiptNotice` 쿼리를 스팟 기준으로 조회하도록 파라미터 추가 (`spotCode` 또는 `spotId`)
  - 스팟에 매핑된 캠페인의 본문을 반환
  - 매핑이 없거나 스팟 정보가 없는 호출(예: 마이페이지 영수증 안내)은 기본 캠페인 본문을 반환

### 백엔드 요구사항

**1. 테이블**

```
beauty_cashback_campaign (신규)
  id, name, refund_rate, membership_refund_rate, is_default, created_at, updated_at
  - is_default: TINYINT(1). 전역 1건만 존재 (기본 캠페인)
  - 기본 캠페인은 삭제 불가, name 변경 불가 (refund_rate/본문/번역은 수정 가능)
  - period(start/end) 컬럼은 두지 않음

beauty_receipt_notice (기존 테이블 확장)
  + campaign_id (FK → beauty_cashback_campaign.id, ON DELETE CASCADE)
  - 기존 컬럼(id, language, notice)은 유지
  - 한 캠페인 ↔ 언어별 N row(번역) 관계

beauty_cashback_spot (신규)
  id, campaign_id (FK → beauty_cashback_campaign.id, ON DELETE CASCADE),
  spot_id (FK → spot.id, ON DELETE CASCADE),
  created_at, updated_at
  - UNIQUE(spot_id): 한 스팟에 동시에 두 캠페인 매핑 금지
  - INDEX(campaign_id): 캠페인별 매핑 스팟 조회용
```

**2. 마이그레이션 (기존 → 신규)**

1. 신규 테이블 2개(`beauty_cashback_campaign`, `beauty_cashback_spot`) 생성
2. `beauty_receipt_notice`에 `campaign_id` 컬럼 추가 (nullable로 추가 후 백필 → NOT NULL 전환)
3. 기본 캠페인 1 row 시드: `name='기본'`, `refund_rate=10.00`, `membership_refund_rate=15.00`, `is_default=1`
4. 기존 `beauty_receipt_notice` 모든 row의 `campaign_id`를 시드된 기본 캠페인 id로 백필
5. 기존에 뷰티캐시백 토글 ON 상태였던 모든 스팟은 자동으로 "기본 캠페인 적용" 상태가 된다 (`beauty_cashback_spot`에 row 미생성)

**3. 쿼리**

- **스팟 기준 캠페인 조회**: 스팟 상세 뱃지·가이드 모달·슬랙 알림에서 사용
  - 입력: `spot_code` 또는 `spot_id`, `language`
  - 동작: `beauty_cashback_spot`에 매핑 row 있으면 해당 캠페인 + 해당 언어의 본문 반환, 없으면 기본 캠페인 + 해당 언어의 본문 반환
  - 반환: 캠페인 id, name, refund_rate, membership_refund_rate, notice(본문)
- 기존 `GetBeautyReceiptNotice` 쿼리는 스팟 파라미터를 받도록 확장하거나, 신규 쿼리(`GetBeautyCashbackBySpot` 등)로 추가 후 기존 쿼리는 마이페이지/기본 호출 케이스용으로 유지

**4. 뮤테이션**

본 단계에서는 캠페인 CRUD / 매핑 변경 / 일괄 해제 등 신규 뮤테이션을 만들지 않는다. 캠페인·매핑에 대한 모든 변경(생성·수정·삭제·매핑 해제 포함)은 엔지니어가 DB로 직접 처리한다. 어드민의 기존 뷰티캐시백 토글 동작도 변경하지 않는다 — 토글 OFF는 노출 차단 의미만 가지며 `beauty_cashback_spot` row를 건드리지 않는다.

**5. 조회 로직**

- 스팟의 뷰티캐시백 여부가 꺼져 있으면 뷰티캐시백 UI 자체를 노출하지 않음
- 켜져 있는 경우
  - `beauty_cashback_spot`에 매핑이 있으면 해당 캠페인의 환급률/본문을 사용
  - 매핑이 없으면 기본 캠페인(`is_default=1`)의 환급률/본문을 사용

**6. CS 알림 연동 (Slack)**

- 대상 채널: `#-cs-noti-뷰티캐시백-영수증리뷰`
- 발송 주체: `creatrip-app` 봇 (기존 [피부과 업셀 영수증 리뷰 알림] 템플릿)
- 변경 내용
  - 기존 템플릿에 **'적용 캠페인'**, **'캐시백 환급률'** 2개 필드 추가 노출
    - 적용 캠페인: 해당 예약 스팟에 매핑된 캠페인 `name` (매핑 없으면 "기본")
    - 캐시백 환급률: 캠페인의 `refund_rate`(%) 값 (예: `15%`)
  - 멤버십 결제 건의 경우 멤버십 환급률도 함께 표기 (예: `15% (멤버십 +5%)`)
- 데이터 소스: 백엔드 요구사항 3번 "스팟 기준 캠페인 조회" 로직 재사용
- 기존 필드 중 `지급 상한: 인당 10만원` 하드코딩은 본 범위에서는 유지

**7. 운영 흐름 (참고)**

- 운영팀은 캠페인 신규 추가/수정/매핑 변경 사항을 매주 정기 미팅 전까지 정리하여 공유한다
- 엔지니어는 매주 정기 미팅 시 현재 DB 상의 캠페인·매핑 현황을 운영팀에 공유하여 양측 인식을 동기화한다
- 미팅에서 합의된 변경사항은 엔지니어가 DB에 직접 INSERT/UPDATE/DELETE로 반영한다 (어드민 UI 거치지 않음)
- 본 운영 흐름은 캠페인 빈도가 낮은 동안 유효하며, 빈도가 누적되거나 매핑 변경이 잦아지면 후속으로 어드민 모달 또는 시트 sync 도입을 재검토한다

**8. 참고 사항**

- 환급금 계산/지급은 유관 운영팀이 수동으로 처리하므로 환급 계산 로직 변경 불필요
- 환급률은 오직 프론트 뱃지·가이드·슬랙 알림 노출용 수치로만 사용됨
- 기간 기반 자동 전환은 본 범위에서 제외 (`beauty_cashback_campaign`에 period 컬럼 두지 않음). 캠페인 시작/종료는 엔지니어 수동 매핑/해제로 처리
- 멤버십 환급률은 캠페인별로 별도 관리 (현재 전역 15%도 캠페인에 따라 달라질 수 있음)
- 어드민 캠페인 관리 모달, 자동번역 UI, 가이드 일괄 해제 기능은 본 범위에서 제외
