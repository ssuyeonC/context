# 세부 기획안: 어드민 광고 설정 개선

## 배경

### 문제 정의

- 현재 어드민에서 스팟/블로그의 광고 설정이 토글(on/off)로만 관리되어, 광고 기간을 설정하거나 현재 상태를 직관적으로 파악하기 어려움
- 광고 상태가 "진행 중인지, 예정인지, 미진행인지" 리스트에서 구분되지 않음
- 언어별로 광고 설정이 개별 관리되어 운영 효율이 떨어짐

### 가설적 임팩트

- 광고 기간 설정 및 상태 관리 자동화로 운영 효율 개선
- 리스트에서 광고 상태를 한눈에 파악 가능

---

## 구현

### 유저 플로우

- 어드민 유저는 스팟/블로그 상세 페이지에서 리스트 상단 광고와 리스트 중단 광고에 대해 각각 광고 기간을 설정할 수 있다.
- 광고 기간을 설정하면, 해당 기간 동안 자동으로 광고가 진행되고 기간이 종료되면 자동으로 해제된다.
- 하나의 언어에서 광고를 설정하면, 기간과 설정 여부가 다른 언어에도 동일하게 반영된다.
- 어드민 유저는 스팟/블로그 리스트 페이지에서 각 항목의 광고 상태(진행 중, 진행 예정, 미진행)를 확인할 수 있다.

### 프론트엔드 요구사항

#### [어드민 스팟 상세] 페이지 & [어드민 블로그 상세] 페이지

**광고 관리 섹션** (리스트 상단 광고 / 리스트 중단 광고 각각)

- **광고 미설정 상태**
  - "광고 설정하기" 버튼: 클릭 시 기간설정 모달 오픈
  - "광고 기록" 버튼: 과거 광고 이력 조회

- **광고 설정됨 상태**
  - 상태 표시: status dot + 상태 텍스트
    - 진행 중: `진행 중 (YYYY/MM/DD HH:MM - YYYY/MM/DD HH:MM)`
    - 진행 예정: `진행 예정 (YYYY/MM/DD HH:MM - YYYY/MM/DD HH:MM)`
  - "기간 수정하기" 버튼: 클릭 시 기간설정 모달 오픈 (기존 값 프리필)
  - "광고 해제하기" 버튼: 광고 설정 해제 (위험 액션, danger 스타일)

- **기간설정 모달**
  - 캘린더 UI로 시작일시/종료일시 선택
  - 날짜 + 시간(HH:MM) 설정 가능

- **언어 간 연동**
  - 하나의 언어에서 광고 기간/설정 여부를 변경하면, 다른 언어에도 동일하게 반영

#### [어드민 스팟 리스트] 페이지 & [어드민 블로그 리스트] 페이지

- 기존 광고 칼럼의 토글(on/off) → **상태값으로 변경**

  | 상태 | 표시 | 조건 |
  |------|------|------|
  | 진행 중 | status dot (green) + "진행 중" | 현재 시점이 광고 기간 내 |
  | 진행 예정 | status dot (yellow) + "진행 예정" | 광고 설정됨 + 시작일 미도래 |
  | 미진행 | status dot (gray) + "미진행" | 광고 미설정 또는 종료 |

---

### 백엔드 요구사항

#### 1. 언어 간 광고 설정 연동 (현재 → 변경)

현재 광고 설정은 **언어별 번역 테이블에 개별 저장**되어 있어, 언어마다 따로 설정해야 함:

| 대상 | 현재 저장 위치 | 필드 |
|------|---------------|------|
| 스팟 | `spot_translation` (언어별) | `is_advertised_in_top_list`, `is_advertised_in_middle_list` |
| 블로그 | `blog_translation` (언어별) | `is_advertised_on_homepage` |

**변경**: 광고 설정/수정/해제 시 **해당 스팟/블로그의 모든 언어 번역 레코드에 동일하게 반영**되어야 함
- 기존 mutation(`setSpotTransAdvertised`, `toggleBlogTransStatus`)에서 단일 `spotTransCode`/`blogTransCode`만 업데이트하는 로직을 확장
- 하나의 언어에서 광고를 설정하면, 같은 스팟/블로그의 다른 언어 번역 레코드도 함께 업데이트
- 기간(시작일시/종료일시) 역시 모든 언어에 동일하게 저장

#### 2. 데이터 저장 — 광고 기간 필드 추가

- `spot_translation` 테이블에 광고 유형별 시작일시/종료일시 컬럼 추가
  - 리스트 상단 광고: `ad_top_start_at`, `ad_top_end_at`
  - 리스트 중단 광고: `ad_middle_start_at`, `ad_middle_end_at`
- `blog_translation` 테이블에도 동일하게 시작일시/종료일시 컬럼 추가
- 기간이 설정되면 해당 유형의 `is_advertised_*` 값도 함께 관리

#### 3. 광고 상태 조회

- 광고 상태는 시작일시/종료일시 기반으로 서버에서 계산하여 반환
  - 진행 중: `ad_start_at <= now < ad_end_at`
  - 진행 예정: `now < ad_start_at`
  - 미진행: 설정 없음 또는 `now >= ad_end_at`
- 리스트 조회 API에서 광고 상태값 포함하여 반환

#### 4. 광고 설정/수정/해제 API

- 광고 기간 설정: 시작일시, 종료일시 입력
- 광고 기간 수정: 기존 설정 업데이트
- 광고 해제: 기간 데이터 초기화
- 설정 변경 시 **해당 스팟/블로그의 모든 언어에 동일하게 반영**
- 기존 광고 개수 제한(카테고리별 최대 6~8개 등) 검증 로직 유지
- 권한: 어드민만 가능

#### 5. 광고 이력 관리

- 광고 설정/해제 이력 저장 (과거 광고 기록 조회용)
