# 세부 기획안 요구사항 문서

## 배경

### 문제 정의

파트너 어드민 스팟 수정 요청 화면(`/travel-partner/spot/{id}`)의 "메인 이미지" 영역에 노출되는 이미지가, 일반 어드민의 스팟 상세 페이지 "이미지 편집" 영역에서 관리되는 이미지와 일치하지 않는다.

원인은 두 영역이 서로 다른 데이터 소스를 참조하기 때문이다.

- 파트너 어드민 수정 요청 화면: 스팟 단위(언어 무관)로 이미지를 저장하는 **레거시 데이터 소스**를 참조
- 일반 어드민 이미지 편집 영역: 약 2~3개월 전 **언어별로 이미지를 분리 저장하는 새 데이터 소스**로 마이그레이션됨

그 결과 일반 어드민에서 이미지를 추가·교체·삭제해도 파트너 어드민의 수정 요청 화면에는 반영되지 않으며, 파트너가 보는 이미지와 운영팀이 관리하는 이미지가 어긋난다.

2026-05-07 모델로스튜디오 파트너 인터뷰에서 동일한 현상이 보고되었고, 코드 추적으로 위 원인이 확인되었다.

### 가설적 임팩트

- 파트너의 수정 요청 정확도·신뢰도가 개선되어 "어드민에서 안 되니 카톡으로 부탁한다"는 우회 경로가 줄어든다.
- 사진 직접 수정에 대한 강한 의향(모델로 인터뷰 참조)을 어드민으로 흡수하기 위한 **선행 조건**으로 작용한다.
- 정량 임팩트는 추후 측정. (참고 자료: `interview-modello-studio-result-0507.md`)

## 구현

### 유저 플로우

**파트너 등급 계정 — 어드민 `/travel-partner/spot/{id}` 진입**

1. 기존의 "메인 이미지" 영역이 제거되고, **"이미지" 영역**으로 교체된다. 이 영역에서 파트너는 일반 어드민과 동일한 방식으로 다중 이미지 편집(추가·교체·삭제·우선순위 변경·메인 이미지 지정)을 할 수 있다.
2. 영역 상단에 **언어 선택 드롭다운**이 노출된다.
   - 옵션: 해당 스팟이 보유한 언어 번역(`spot_translation`)이 존재하는 언어만 노출
   - 정렬: 어드민의 언어 선택 탭과 동일한 순서 (한국어 → 일본어 → 대만어 → 홍콩어 → 중국어 → 베트남어 → 태국어 → 영어 → 프랑스어 → 스페인어 → 독일어 → 러시아어 → 이탈리아어 → 인도네시아어 → 몽골어)
   - 디폴트: 위 순서 중 첫 번째로 존재하는 언어 (대개 한국어)
   - 공개 외국어가 0개일 경우: 최초 스팟 등록 시 입력된 한국어 이미지를 노출
3. 파트너가 드롭다운에서 언어를 바꾸면 해당 언어의 이미지로 영역이 갱신된다.
4. 영역 하단에 안내 문구를 노출: **"메인 이미지는 언어 별로 설정할 수 있습니다."**
5. 파트너는 화면에서 변경을 자유롭게 누적하고, 페이지의 "수정 요청" 버튼을 눌러 일괄 제출한다. 이미지 변경은 다른 수정 항목(가격·주의사항 등)과 함께 단일 수정 요청에 묶여 운영팀에 전달된다.
6. 운영팀이 수정 요청을 검수·승인한 시점에 실제 데이터에 반영된다. (이미지 변경은 자동 승인 대상이 아니며 항상 운영팀 검수를 거친다)

**유저 페이지**

- 영향 없음.

### 프론트엔드 요구사항

**(1) 영역 교체**

- 파트너 어드민 수정 요청 화면의 기존 "메인 이미지" 영역을 제거하고, 그 자리에 **"이미지" 영역**을 신규 배치한다.
- 영역 라벨은 "이미지"로 한다. 영역 내에서 메인 이미지를 지정할 수 있으므로 별도의 "메인 이미지" 영역은 두지 않는다.
- 이미지 편집 UI는 일반 어드민의 이미지 편집 컴포넌트를 재사용한다.

**(2) 언어 선택 드롭다운**

- 위치: 영역 상단
- 옵션 구성·정렬·디폴트는 유저 플로우 2번 항목을 따른다.
- 언어 변경 시 해당 언어의 이미지로 화면이 갱신된다.

**(3) 안내 문구**

- 위치: 영역 하단
- 문구: "메인 이미지는 언어 별로 설정할 수 있습니다."

**(4) 권한·저장 흐름**

- 파트너의 편집 액션은 즉시 반영되지 않는다. 변경 사항은 화면 상에서 누적 상태로 표시되며 (어떤 변경이 요청될 예정인지 파트너가 인지할 수 있도록 시각적으로 구분), 페이지의 "수정 요청" 버튼을 통해 일괄 제출된다.
- 다른 수정 요청 항목(가격·주의사항 등)과 동일한 흐름이다.

### 백엔드 요구사항

**(1) 데이터 소스 전환**

- 파트너 어드민 수정 요청 화면이 읽는 이미지 데이터 소스를 레거시(`image` 테이블)에서 새 소스(`spot_trans_has_media` 테이블, 언어별)로 전환한다.
- 본 작업의 데이터 마이그레이션 범위는 **해당 화면 한정**이다. 레거시 테이블 자체의 폐기 및 다른 참조처 전수 마이그레이션은 별도 트랙으로 분리 — `requirements_spot-image-table-deprecation.md` 참조.

**(2) 수정 요청 시스템 통합**

- 이미지 변경은 기존 수정 요청 시스템의 **"기본 정보 수정(`SPOT_INFO`)" 카테고리 / 이미지 필드(`SPOT_IMAGES`)**에 흡수한다. (이 필드 타입은 이미 정의되어 있으며 본 작업에서 신규 update_type을 신설하지 않는다)
- 페이로드 구조는 기존 단일 이미지 배열에서 **언어별 이미지 리스트의 목표 상태**로 확장한다. (예: 언어 키별로 이미지 배열을 담는 형태)
- 승인 처리 시 페이로드를 언어별 이미지 추가/삭제/순서/메인 지정 명령으로 분해하여 실제 데이터에 반영한다.
- 자동 승인 정책: 이미지 변경은 자동 승인 대상이 아님 — 기존 정책 유지.

**(3) 검수 화면**

- 운영팀이 수정 요청을 검수하는 화면에서, 이미지 변경 내역은 기존 "기본 정보 수정" 검수 패턴(원본 vs 요청 양쪽 비교 카드형)을 그대로 사용한다.
- 검수자가 언어별 비교를 확인할 수 있도록 검수 화면에도 언어 선택 UI를 함께 노출한다.

**(4) 진행 중 수정 요청의 자동 반려 처리**

- 본 기능 배포 시점에, 다음 조건을 모두 만족하는 **검수 대기 중인 수정 요청을 일괄 자동 반려**한다.
  - 변경 내용 분류가 "기본 정보 수정"에 해당
  - 메인 이미지 변경 건이 포함되어 있음
- 자동 반려 시 다음 문구를 반려 사유로 입력한다.
  > "이미지 수정 요청 기능이 업데이트되어 언어 별로 이미지를 관리할 수 있도록 개선되었습니다. 다시 요청해주세요."
- 메인 이미지 변경이 포함되지 않은 진행 중 요청(가격·주의사항만 변경 등)은 영향 받지 않으며 정상 검수 흐름을 따른다.
- 처리 결과는 별도 로그로 보관해 추후 추적 가능하도록 한다.

**(5) 동시 변경 충돌 처리**

- 다른 수정 요청 건과 동일한 정책을 따른다. 현재 정책은 검수자의 수동 확인 — 별도의 자동 충돌 감지 로직을 본 티켓에서는 추가하지 않는다.

### 영향 범위

- 파트너 어드민 스팟 수정 요청 화면 (이미지 영역, 언어 드롭다운, 안내 문구)
- 운영팀 수정 요청 검수 화면 (언어별 이미지 비교 확장)
- 파트너 수정 요청 백엔드 시스템 (`SPOT_IMAGES` 필드 핸들러를 새 데이터 소스 기준으로 전환)
- 일회성 마이그레이션 스크립트 (조건 충족 진행 중 요청 자동 반려)
- 유저 페이지 영향 없음
