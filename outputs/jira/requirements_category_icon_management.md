# 대분류 카테고리 아이콘 관리 기능 추가

## 배경

### 문제 정의

- 현재 대분류 카테고리(MAIN_RESERVATION) 아이콘이 프론트엔드에 하드코딩되어 있어, 아이콘 변경이나 신규 카테고리 추가 시 코드 배포가 필요
- 백엔드에는 이미 `mapIconPath` 필드와 `mapIconUrl` resolver가 존재하지만, 프론트엔드에서 활용하지 않고 하드코딩된 React 컴포넌트(18개)를 사용 중
- 어드민에서 카테고리 아이콘을 관리할 수 없어 운영 유연성이 떨어짐

### 가설적 임팩트

- 어드민에서 아이콘을 직접 관리할 수 있게 되어 카테고리 추가/변경 시 개발 리소스 불필요
- COM-2248(메인 페이지 내비게이션 관리 시스템)의 선행 작업으로, 내비게이션에 카테고리 연결 시 어드민 관리 아이콘이 바로 활용됨

## 구현

### 유저 플로우

1. 어드민이 카테고리 관리 페이지에서 대분류 카테고리를 선택한다.
2. 아이콘 업로드 영역에 이미지를 등록한다.
3. 저장 시 아이콘이 업로드되고, 유저 페이지에서 해당 아이콘이 노출된다.

### 프론트엔드 요구사항

**어드민**

- 카테고리 수정 모달에 아이콘 업로드 기능 추가
- 아이콘 미리보기 노출
- 아이콘 사이즈는 기존 하드코딩된 아이콘 사이즈 기준으로 설정

**유저 페이지**

- 기존 하드코딩된 아이콘 매핑(`icon.tsx`의 `categoryIcons` 객체) 제거
- 백엔드에서 내려주는 `mapIconUrl`을 사용하도록 변경
- 아이콘이 활용되는 모든 페이지에서 어드민 관리 아이콘 노출 (메인 페이지 내비게이션, 스팟 상세 등)

### 백엔드 요구사항

- 기존 `mapIconPath` 필드 및 `mapIconUrl` resolver 활용
- 아이콘 업로드 API (이미 존재하는 경우 확인 필요)
- 기존 하드코딩된 아이콘을 DB로 마이그레이션 (`mapIconPath`에 기존 아이콘 이미지 등록)
