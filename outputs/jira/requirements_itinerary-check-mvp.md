# 여행 일정 검증 MVP (간소화 파일럿)

## 배경
### 문제 정의

한국 여행을 준비하는 외국인의 가장 큰 불안은 "내 일정이 현실적인지 모르겠다"는 것

- r/koreatravel 10,000건 분석 결과, 검증 요청(19.6%) + 가능성 확인(14.7%) = 34.3%로 추천 요청(24.6%)보다 10pp 높음
- SupTravel AI가 "AI Itinerary Checker"를 이미 유료로 운영 중 — 시장 선점 압박 존재

목표: 최소한의 구현으로 "여행 일정 검증" 수요를 검증하는 MVP 랜딩페이지를 구축한다.

핵심 전략:
- 어드민 없이, 유저 입력을 Google Sheets로 수집하여 수동 운영
- 텍스트 + 이미지 입력을 지원하여 유저의 자유로운 일정 공유를 유도
- 이미지는 S3에 업로드하고 URL을 시트에 기록

### 가설적 임팩트

검증 요청 규모 추정:
- 현재 일 플랜 생성 34~50건
- 플랜 생성자 중 검증 요청 비율 30~50% 가정 → 일 10~25건 수준
- 1~2명이 수동으로 처리 가능한 범위

성공 기준 (런칭 후 4주 평가):

| 판단 | 조건 | 대응 |
|------|------|------|
| 성공 (Go) | 일 10건+ 요청 | 정식 기능(어드민 + 알림 + 피드백) 개발 착수 |
| 보류 | 일 5~10건 요청 | 진입점 확대 + 2주 연장 관찰 |
| 중단 | 일 5건 미만 요청 | 파일럿 종료, 리소스 재배분 |

## 구현
### 유저 플로우

1. 유저는 랜딩페이지에 접속한다 (별도 URL, 비로그인 상태에서도 이용 가능).
2. 유저는 이메일을 입력한다 (필수).
3. 유저는 여행 일정을 텍스트 혹은 이미지로 입력한다.
   - 텍스트: 최소 50자 이상
   - 이미지: 1장
4. Submit 클릭 시:
   - 이미지가 입력된 경우, S3에 업로드하여 URL을 획득한다.
   - 입력 데이터를 Google Sheets에 기록한다 (입력일 / 이메일 / 텍스트 / 이미지 URL / 상태값).
   - "접수 완료" 확인 화면을 노출한다.
5. 운영팀은 Google Sheets에서 요청을 확인하고, 이메일로 직접 검증 결과를 회신한다.
6. 운영팀은 '상태값'을 '미완료'에서 '완료'로 변경한다.

### 프론트엔드 요구사항

**랜딩페이지 (단일 HTML 또는 경량 페이지)**

- 헤더: Creatrip 로고
- 히어로 섹션: "Is your Korea itinerary actually doable?" 타이틀 + 서비스 설명
- 잔여 수량 카운터: "X of 100 remaining" (Google Sheets에 적재된 데이터 수 기반으로 산출, 100건 도달 시 Submit 버튼 비활성화 및 마감 안내 문구 노출)
- How it works 섹션: 3단계 안내 (일정 공유 → 전문가 리뷰 → 이메일 피드백)
- 입력 폼:
  - 이메일 입력 필드 (필수, 이메일 형식 검증)
  - 텍스트 에어리어 (최소 50자, 실시간 글자 수 표시)
  - 이미지 업로드 영역 (1장, 업로드 프리뷰 표시)
  - 텍스트와 이미지 중 하나 이상 입력 필수
  - Submit 버튼 (이메일 유효 + 텍스트 50자 이상 또는 이미지 1장 이상일 때 활성화, 시트 데이터 100건 도달 시 비활성화)
- 접수 완료 화면: "24시간 내 이메일로 결과를 보내드립니다" 안내
- 모바일 퍼스트 반응형 디자인

**이미지 업로드 UI**

- 드래그 앤 드롭 또는 클릭하여 파일 선택
- 지원 형식: JPG, PNG, WEBP (최대 10MB)
- 업로드 시 썸네일 프리뷰 표시
- 이미지 삭제 가능
- 1장 제한

### 백엔드 요구사항

**이미지 업로드 API**

- S3 연동 이미지 업로드 엔드포인트
  - 입력: 이미지 파일 (multipart/form-data)
  - 출력: 업로드된 이미지의 S3 URL
  - 제한: JPG/PNG/WEBP, 최대 10MB/장
  - 기존 이미지 업로드 인프라가 있다면 재활용

**Google Sheets 연동**

- Google Apps Script Web App으로 데이터 수신
- 시트 컬럼 구조:

| 컬럼 | 설명 |
|------|------|
| 입력일 | 요청 생성 시각 (ISO 8601) |
| 이메일 | 유저 이메일 |
| 텍스트 | 유저가 입력한 여행 일정 텍스트 |
| 이미지 | S3 업로드 URL |
| 상태값 | 미완료 (기본값) / 완료 (운영팀이 검증 결과 회신 후 변경) |

- 프론트에서 직접 Apps Script URL로 POST 요청 (no-cors 모드)

**요청 건수 조회 API**

- Google Apps Script Web App에 GET 엔드포인트 추가
  - 시트의 현재 데이터 행 수(= 총 요청 건수)를 반환
  - 응답 예시: `{ "count": 42 }`
- 프론트에서 페이지 로드 시 호출하여:
  - 잔여 수량 카운터 렌더링 (100 - count)
  - count가 100 이상이면 Submit 버튼 비활성화 + 마감 안내 문구 노출

**슬랙 알림 (선택)**

- 새 요청 수신 시 슬랙 웹훅으로 #itinerary-check 채널에 알림
- 포함 정보: 이메일, 텍스트 요약 (앞 100자), 이미지 수, 시트 링크
