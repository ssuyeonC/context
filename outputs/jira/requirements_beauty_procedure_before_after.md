# 세부 기획안: 연결된 시술 데이터에 Before&After 이미지 관리 기능 추가

## 배경

### 문제 정의

- 피부과 스팟 상세 페이지를 조회하는 유저는 안내되는 시술의 효과를 직관적으로 확인할 수 없음
- 시술별 가격 테이블은 존재하지만, 시술 결과를 시각적으로 보여주는 Before & After 이미지가 없어 유저의 구매 의사결정을 돕지 못하는 상태

### 가설적 임팩트

- Before & After 이미지를 통해 시술 효과를 직관적으로 전달하여 유저의 구매 전환율 개선 기대
- 피부과 스팟의 상품 신뢰도 향상

---

## 구현

### 유저 플로우

- 유저는 피부과 스팟 상세 페이지를 조회할 때, 시술 정보와 함께 해당 시술의 효과를 Before & After 이미지를 통해 직관적으로 확인할 수 있다.
- 시술 이미지는 해당 시술이 연결된 스팟 단위로 저장된다.

### 프론트엔드 요구사항

#### 유저페이지

**[스팟 상세] 페이지**

- "시술별 가격 테이블" 영역에 Before & After 이미지 노출
- 이미지 노출 영역에 고정 안내 문구 표시
  - 번역값 키: `spot.detail.procedure-image-caution`
- Before & After 이미지가 등록되지 않은 시술은 이미지 영역 미노출

#### 어드민

**여행 SETTINGS → [K뷰티 시술 관리] → "시술 연결 관리" 탭 → [스팟 연결 관리] 모달**

- 노출되는 각 시술에 "시술 이미지 관리" 섹션 추가
- 각 시술마다 이미지 2개씩 업로드 (필수 아님)
  - Before 이미지 1개 + After 이미지 1개
- 이미지 업로드/삭제/교체 기능

---

### 백엔드 요구사항

#### 1. 데이터 저장

- 현재 시술-스팟 연결 데이터(`spot_beauty_pricing`)에는 이미지 정보가 없음
- 각 시술-스팟 연결에 Before 이미지 1개, After 이미지 1개를 저장할 수 있어야 함
- 이미지는 필수가 아님 (0개 또는 2개)
- 1개만 등록된 상태에서 저장 시 오류 반환 (Before와 After는 반드시 쌍으로 등록되어야 함)

#### 2. 조회 API (유저페이지용)

- 스팟 상세 페이지에서 시술 가격 목록(`spotBeautyPricings`) 조회 시, 각 시술의 Before & After 이미지를 함께 반환
- 이미지가 없는 시술은 빈 배열 반환

#### 3. 관리 API (어드민용)

- 시술-스팟 연결에 Before 또는 After 이미지 업로드
  - 같은 유형의 이미지가 이미 존재하면 교체
- 업로드된 이미지 삭제
- 권한: 어드민만 가능
