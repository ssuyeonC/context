# 세부 기획안 요구사항 문서

## 배경

### 문제 정의

- 어드민 스팟 리뷰 페이지(`/spotReview`)에서 답글을 작성할 때, **답글 작성자 타이틀이 작성자의 권한(role)에 의해 자동으로 결정**된다.
  - 관리자(admin / super-admin)가 작성 → 유저 페이지에 **"관리자 답변"** 으로 노출
  - 파트너가 작성 → 유저 페이지에 **"상점 답변"** 으로 노출
- 운영 상황에 따라 관리자(CX/운영팀)가 **상점을 대신해서 답글을 작성**해야 하는 케이스가 발생한다. 이때 현재 로직으로는 자동으로 "관리자 답변"으로 노출되어, 유저 입장에서는 상점이 답변하지 않은 것처럼 보이거나 어색한 노출이 발생한다.
- 관리자가 답글 작성 시점에 **유저 페이지 노출 이름(파트너/관리자)을 선택**할 수 있어야 한다.

### 가설적 임팩트

- 관리자가 상점 대신 답변할 때도 **자연스러운 "상점 답변"** 으로 노출되어, 유저가 받는 답변의 **일관성과 신뢰감**이 향상된다.
- CX/운영팀이 파트너 응대 누락 케이스를 대신 커버할 때 **별도 우회 절차 없이** 어드민에서 바로 처리 가능하다.

## 구현

### 유저 플로우

- 어드민 사용자가 `/spotReview` 페이지에서 특정 리뷰의 답글 작성 모달을 연다.
- 모달 내에서 답글 본문을 작성한다.
- **유저 권한이 관리자(admin / super-admin)인 경우**:
  - 모달 내에 **"노출 이름 선택" 라디오 버튼**이 표시된다.
    - `파트너로 노출` — 유저 페이지에서 "상점 답변"으로 표시
    - `관리자로 노출` — 유저 페이지에서 "관리자 답변"으로 표시 (**기본값**)
  - 관리자가 라디오 버튼을 선택한 후 답글을 등록한다.
- **유저 권한이 파트너인 경우**:
  - 라디오 버튼은 **노출되지 않는다** (기존 동작 유지).
  - 작성한 답글은 자동으로 "상점 답변"으로 노출된다.
- 등록된 답글은 유저 페이지(스팟 상세 - 리뷰 영역)에서 선택된 노출 타입에 맞는 **타이틀**과 함께 표시된다.

### 프론트엔드 요구사항

- **어드민 — `/spotReview` 답글 작성 모달**
  - 답글 작성 모달 내부에 **노출 이름 선택 라디오 버튼** 추가.
    - 옵션: `파트너로 작성` / `관리자로 작성`
    - 기본 선택값: **`관리자로 작성`**
    - 노출 조건: **현재 로그인 사용자의 권한이 `ADMIN` 또는 `SUPER_ADMIN`인 경우에만** 노출
  - 파트너 권한으로 로그인된 경우 라디오 버튼은 노출하지 않고 기존 UI 유지.
- **어드민 — 답글 수정 모달**
  - 관리자 권한일 경우, 답글 수정 시에도 동일한 라디오 버튼을 노출하여 **노출 타입을 변경**할 수 있도록 한다.
  - 기존 값을 기본 선택값으로 표시.
- **어드민 — 리뷰/답글 목록 화면 (내부 식별 표시)**
  - 어드민 내부 화면에서는 답글에 **실제 노출 타입(파트너/관리자)을 표시**한다.
  - 예: 관리자가 `파트너로 작성`한 답글의 경우, 어드민 목록에는 "파트너로 작성됨"(혹은 그에 준하는 배지/태그)을 별도로 보여줘 운영 추적이 가능하도록 한다.
  - 유저 페이지에는 노출되지 않는다 (어드민 한정).
- **답글 등록 동작**
  - 등록 mutation 호출 시 라디오 선택값(노출 타입)을 함께 전송.
- **유저 페이지 — 스팟 상세 리뷰 영역**
  - 답글 데이터에 포함된 노출 타입 필드에 따라 타이틀을 분기 렌더링.
    - `PARTNER` → "상점 답변"
    - `ADMIN` → "관리자 답변"
  - 기존에 작성된 답글(필드 없음)은 작성자의 권한 기준으로 기존과 동일한 타이틀 노출 (마이그레이션 정책 참조).

### 백엔드 요구사항

- **데이터 모델**
  - 리뷰 답글 엔티티에 **노출 타입(display author type)** 필드 추가.
    - 가능한 값: `PARTNER`, `ADMIN`
    - 신규 작성 시 클라이언트가 전달한 값을 저장.
- **API / 비즈니스 로직**
  - 답글 작성 mutation에 `displayAuthorType` 입력 파라미터 추가 (선택값, 미전달 시 작성자 권한으로 자동 결정).
  - 답글 **수정 mutation에도** `displayAuthorType` 입력 파라미터 추가 (관리자만 변경 가능).
  - 권한 검증 (작성/수정 공통):
    - 작성자/수정자가 **파트너**이면 입력값과 무관하게 `PARTNER`로 강제 저장 (악의적 우회 방지).
    - 작성자/수정자가 **관리자(ADMIN / SUPER_ADMIN)** 인 경우에만 `PARTNER` / `ADMIN` 모두 허용.
  - 답글 조회 시 `displayAuthorType` 필드를 함께 반환.
  - 어드민용 답글 조회 응답에는 노출 타입 외에도 **실제 작성자 권한** 정보를 함께 제공하여, 어드민 화면에서 "관리자가 파트너로 작성한 케이스"를 식별할 수 있도록 한다.
- **기존 데이터 마이그레이션**
  - 기존에 작성된 답글에 대해 작성자의 권한 기준으로 `displayAuthorType` 백필.
    - 관리자 작성 → `ADMIN`
    - 파트너 작성 → `PARTNER`
- **테스트 케이스**
  - 관리자가 `관리자로 작성` 선택 → `ADMIN`으로 저장, 유저 페이지에 "관리자 답변"
  - 관리자가 `파트너로 작성` 선택 → `PARTNER`로 저장, 유저 페이지에 "상점 답변", 어드민 내부 화면에는 "파트너로 작성됨" 식별 표시 노출
  - 관리자가 답글 수정 시 노출 타입 변경 → 변경된 타입으로 저장 및 노출
  - 파트너가 답글 작성 (라디오 미노출) → `PARTNER`로 저장
  - 파트너가 클라이언트 임의로 `ADMIN` 전송 → 백엔드에서 `PARTNER`로 강제 저장
  - 파트너가 답글 수정 시 노출 타입 변경 시도 → 백엔드에서 무시하고 `PARTNER` 유지
  - 마이그레이션 후 기존 답글의 노출 타이틀이 변경되지 않아야 함 (회귀 없음)
