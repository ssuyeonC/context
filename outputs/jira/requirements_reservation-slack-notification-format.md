# 예약 슬랙 알림 메시지 포맷 개선

## 배경
### 문제 정의

예약 승인/취소/부분환불 시 Slack 채널(`#cs_noti-예약승인내역`, `#cs_noti-예약취소내역`)로 발송되는 알림 메시지를 CS 담당자가 **복사하여 카카오톡으로 전달**하는 운영 워크플로우가 존재한다.

현재 메시지는 Slack 인용구 마크다운(`> `)과 줄 끝 trailing space(` \n`)를 포함하고 있어, 복사-붙여넣기 시 다음 문제가 발생한다:

- `>` 문자가 카카오톡에 그대로 노출되어 메시지가 깨져 보임
- 각 항목 앞에 불필요한 공백이 포함됨
- 개인정보 구분선(`——————————`)이 과도하게 길어 가독성 저하

### 가설적 임팩트

- CS 운영 효율 개선: 복사 후 수동으로 `>` 및 공백을 제거하는 추가 작업 제거
- 파트너/고객 커뮤니케이션 품질 향상: 깔끔한 포맷으로 전달되어 전문성 있는 인상

## 구현
### 유저 플로우

1. 예약 상태 변경(승인 요청/취소/부분환불) 발생
2. Slack 채널에 알림 메시지 발송
3. CS 담당자가 메시지 전체를 복사
4. 카카오톡에 붙여넣기하여 파트너/고객에게 전달
5. **개선 후**: 별도 편집 없이 바로 전달 가능

### 프론트엔드 요구사항

해당 없음 (백엔드 Slack 메시지 포맷 변경만 해당)

### 백엔드 요구사항

**대상 파일**: `backend/apps/trip/src/modules/reservation/slack/noti-reservation/noti-reservation-slack-parser.service.ts`

#### 1. 인용구 마크다운 제거

모든 메시지 줄 앞의 `> ` 접두사를 제거한다.

| 적용 대상 | 행 범위 |
|-----------|---------|
| `makeMessage` — CONFIRM_REQUIRED case | 159~177행 |
| `makeMessage` — PARTIAL_REFUND case | 188~210행 |
| `makeMessage` — CANCEL case | 217~245행 |
| `infoString` 조립 (추가정보) | 127행 |
| `changeReservationDateInfoString` 조립 | 148행 |
| `itemInfo` 메서드 | 254행 |
| `generatePersonalReservationString` 메서드 | 262~264행 |

#### 2. Trailing space 제거

모든 줄 끝의 ` \n` (공백+개행)을 `\n` (개행만)으로 변경한다.

**변경 전**: `` `> 예약처 : ${spotName} \n` ``
**변경 후**: `` `예약처 : ${spotName}\n` ``

#### 3. 구분선 축소

개인정보 영역의 구분선을 축소한다.

**변경 전**: `——————————`
**변경 후**: `----`

#### 4. 콜론 뒤 공백 통일

`예약자 이름 :` → `예약자 이름 : ` (콜론 뒤 공백 추가, 167/197/232행)

#### 변경 전후 비교 (CONFIRM_REQUIRED 스레드 기준)
