# 좋아요 모달 내 플랜 생성 진입점 추가 (좋아요 리스트 진입률 개선)

## 배경

### 문제 정의
- COM-1895에서 좋아요 아이템으로 플랜을 생성하는 기능을 제공하지만, 좋아요 리스트 페이지 자체의 진입률이 낮아 해당 기능의 활용도가 제한될 수 있음
- 현재 모바일에서 좋아요 클릭 시 노출되는 모달에는 상단에 "Go to Likes >" 텍스트 링크만 존재하며, 좋아요 리스트로 이동할 동기 부여가 약함
- 좋아요 클릭 일 평균 108명, 그 중 앱 비중 55%로 좋아요 행동 자체가 빈번하지 않아 빠른 시점에 플랜 연결 CTA를 노출할 필요가 있음

### 가설적 임팩트
- 좋아요 2개 이상 보유 유저에게 플랜 생성 CTA를 노출하면, 좋아요 리스트 페이지 진입률이 개선될 것
- 좋아요 리스트 진입률 개선은 COM-1895(좋아요 아이템으로 플랜 생성) 기능의 전환율 증가로 이어질 것
- 좋아요 → 플랜 연결이 강화되면, 좋아요 행동의 리텐션 리프트(2.17x)가 플랜 생성 리프트와 시너지를 낼 것

## 구현

### 유저 플로우

모바일 환경에서 상품에 좋아요를 클릭하면, 해당 유저의 전체 좋아요 수(방금 누른 것 포함)에 따라 모달 상단 영역이 다르게 노출된다.

**Case 1: 좋아요 수 = 1개 (첫 좋아요)**

현재와 동일 — 변경 없음

```
┌─────────────────────────────────────┐
│ ✓ Added to your 'Likes' list          Go to Likes >
├─────────────────────────────────────┤
│ How about this product instead?     │
│ [추천 상품 카드들]                    │
└─────────────────────────────────────┘
```

**Case 2: 좋아요 수 >= 2개**

좋아요 개수 표시 + 플랜 생성 CTA 버튼 노출

```
┌─────────────────────────────────────┐
│ ✓ Added to your 'Likes' list       │
│ You have N liked items — ready     │
│ to plan your trip?                  │
│ [  Build a Plan from Likes  ]       │
├─────────────────────────────────────┤
│ How about this product instead?     │
│ [추천 상품 카드들]                    │
└─────────────────────────────────────┘
```

- 좋아요 2개부터 바로 플랜 CTA 노출
- 기존 "Go to Likes >" 텍스트 링크 대신 CTA 버튼이 같은 역할 수행

**CTA 동작:**
- "Build a Plan from Likes" (여행 계획하기) CTA 탭 시 → 좋아요 리스트 페이지로 이동
- 좋아요 리스트 페이지에서 COM-1895의 "여행 계획하기" 버튼을 통해 플랜 생성 플로우 진입
- 모달에서 바로 플랜 생성 바텀시트를 띄우지 않음 (좋아요 모달 위에 바텀시트가 이중으로 뜨는 UX 문제 방지)

### 프론트엔드 요구사항

- 좋아요 클릭 시 노출되는 모달 컴포넌트에서 유저의 전체 좋아요 수를 조회
- 좋아요 수에 따른 모달 상단 영역 조건부 렌더링 (Case 1 / Case 2)
- Case 2의 CTA 버튼 탭 시 좋아요 리스트 페이지(/my/likes)로 라우팅
- 모달 하단의 "다른 상품 추천" 영역은 모든 케이스에서 변경 없이 유지
- 임계값(현재 2개)은 설정값으로 관리하여 런칭 후 조정 가능하도록 구현

### 백엔드 요구사항

- 좋아요 모달 렌더링 시 유저의 전체 좋아요 수를 반환하는 API가 필요한 경우 대응 (기존 API에서 count 제공 여부 확인 필요)
