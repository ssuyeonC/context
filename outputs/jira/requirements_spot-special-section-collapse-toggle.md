# 스팟 상세 'Special Event/Special Price' 영역 디폴트 펼침 및 접힘 토글 추가

## 배경

### 문제 정의

- 스팟 상세 페이지의 'Special Event'와 'Special Price' 영역이 현재 디폴트 접힘 상태로 노출된다. 사용자는 영역을 클릭해 펼쳐야만 내용을 확인할 수 있어, 운영자가 의도한 프로모션·할인 정보의 가시성이 낮다.
- 디폴트 노출 상태가 모든 스팟에 동일하게 접힘으로 고정되어 있고, 운영자가 스팟·영역 단위로 노출 상태를 조절할 수 있는 수단이 없다. 강조하고 싶은 영역과 그렇지 않은 영역을 구분해 운영하지 못한다.

### 가설적 임팩트

- 디폴트 펼침으로 전환되면 사용자가 추가 클릭 없이도 'Special Event/Special Price' 내용을 인지할 수 있어, 프로모션·할인 정보의 노출 빈도가 증가한다.
- 어드민 측에 '접힘' 토글이 추가되면 운영자가 스팟·영역 단위로 노출 상태를 직접 결정할 수 있다. 예) 본문이 길어 페이지 길이를 부담스럽게 만드는 케이스만 선택적으로 접을 수 있다.

## 구현

### 유저 플로우

**사용자 — 스팟 상세 페이지**

- 스팟 상세 페이지에 진입한다.
- 'Special Event' 영역이 어드민에서 등록되어 있고 '접힘' 토글이 OFF인 경우 → 영역이 **펼쳐진 상태**로 노출된다. 사용자는 클릭 없이 내용을 바로 확인할 수 있다.
- 'Special Event' 영역의 '접힘' 토글이 ON인 경우 → 영역이 **접힌 상태**로 노출된다. 헤더만 보이며, 사용자가 클릭하면 펼쳐서 내용을 확인할 수 있다 (기존 동작 유지).
- 'Special Price' 영역도 동일한 규칙을 따른다.

**운영자 — 어드민의 Special Event/Special Price 영역 등록·수정 화면**

- 어드민에서 스팟의 'Special Event' 또는 'Special Price' 영역을 등록/수정하는 입력 폼에 '접힘' 토글이 노출된다.
- '접힘' 토글의 디폴트 값은 **OFF (펼침)** 이다.
- 토글을 ON으로 변경하면 사용자 페이지에서 해당 영역이 접힌 상태로 노출된다.
- 토글 변경은 저장과 동시에 사용자 페이지에 반영된다.

### 프론트엔드 요구사항

**FR-1. 스팟 상세 페이지 'Special Event/Special Price' 영역 노출 상태 분기**

- 영역의 '접힘' 값에 따라 초기 노출 상태가 결정된다.
  - '접힘' OFF → 영역 펼침 상태로 진입. 헤더 + 본문 노출.
  - '접힘' ON → 영역 접힘 상태로 진입. 헤더만 노출, 사용자가 클릭 시 펼침.
- 영역 펼침/접힘 토글 인터랙션은 기존 동작 유지 (영역 헤더 클릭 시 토글).

**FR-2. 어드민 'Special Event/Special Price' 입력 폼에 '접힘' 토글 추가**

- 영역의 정보 입력 필드들 옆 또는 상단에 '접힘' 토글을 추가한다.
- 라벨 워딩: '접힘'.
- 디폴트 값: OFF.
- 도움말 텍스트: "ON으로 설정하면 사용자 화면에서 영역이 접힌 상태로 노출됩니다." (워딩은 디자인 단계에서 확정).

### 백엔드 요구사항

**BR-1. 'Special Event/Special Price' 영역 데이터에 '접힘' 필드 추가**

- 두 영역 각각에 boolean 형태의 '접힘' 필드를 추가한다.
- 기본값은 false (펼침).
- 어드민에서 토글 ON 저장 시 true, OFF 저장 시 false로 저장된다.

**BR-2. 스팟 상세 응답에 '접힘' 필드 포함**

- 사용자 스팟 상세 응답에 두 영역 각각의 '접힘' 값이 포함되어 FE가 초기 노출 상태를 결정할 수 있도록 한다.

**BR-3. 기존 데이터 마이그레이션**

- 신규 필드의 기존 데이터 디폴트 값은 **false (펼침)** 으로 일괄 초기화한다.
- 운영자가 의도적으로 접힘 상태로 두고 싶은 케이스는 어드민에서 개별 토글 ON으로 재설정한다.

## 비범위 (Out of Scope)

- 'Special Event/Special Price' 영역 외 스팟 상세의 다른 섹션에 대한 동일한 토글 적용은 1차 범위 외.
- '접힘' 토글의 디자인 디테일(스타일, 위치, 도움말 워딩 등)은 디자인 단계에서 별도 정의.
- 디폴트 펼침 전환에 따른 사용자 행동 변화 측정(클릭률, 전환율 등) 트래킹 설정은 별도 작업으로 분리.

## 검증 포인트

- 스팟 상세 페이지에서 '접힘' OFF로 저장된 'Special Event/Special Price' 영역이 진입 즉시 펼친 상태로 노출되는지.
- 스팟 상세 페이지에서 '접힘' ON으로 저장된 영역이 진입 즉시 접힌 상태로 노출되는지. 헤더 클릭 시 정상적으로 펼쳐지는지.
- 어드민 입력 폼에서 '접힘' 토글의 디폴트 값이 OFF로 표시되는지.
- 어드민에서 토글을 ON/OFF로 변경 후 저장하면 사용자 페이지에 정확히 반영되는지.
- 기존 데이터의 '접힘' 값이 모두 false (펼침)로 마이그레이션되었는지.
