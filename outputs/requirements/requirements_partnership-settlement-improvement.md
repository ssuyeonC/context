# 파트너십 사이트 정산 화면 개선

## 배경

### 문제 정의

현재 파트너십 사이트의 정산 화면에서는 **누적 정산액만 확인 가능**하여, 파트너사가 실제로 가장 궁금해하는 정보를 직관적으로 파악하기 어려움.

- "이번 달에 얼마를 받는지" 알 수 없음
- "아직 정산되지 않고 남아 있는 금액이 얼마인지" 알 수 없음
- 기간 필터가 없어 특정 월 기준 정산 내역 확인 불가
- 이번 달/다음 달 정산 예정 금액을 구분해서 보기 어려움
- 파트너사의 정산 관련 문의가 발생하는 원인

### 가설적 임팩트

- 정산 관련 파트너사 문의 감소 (운영팀 업무 부담 경감)
- 파트너십 사이트 실사용성 향상 → 파트너 만족도 제고
- 정산 투명성 확보로 파트너 신뢰도 강화

## 구현

### 유저 플로우

1. 파트너가 파트너십 사이트에 로그인하여 정산 화면에 진입한다
2. 상단 요약 영역에서 **누적 정산액 / 이번달 정산금액 / 다음달 정산예정금액** 3개 카드를 확인한다
3. 월별 필터를 사용하여 특정 월의 정산 내역을 조회한다
4. 각 정산 건의 상태(완료/예정/대기)를 확인한다
5. 기준일 안내 문구를 통해 금액 산정 기준을 이해한다

### 프론트엔드 요구사항

#### 1. 상단 요약 카드 (3개)

| 카드 | 표시 내용 | 산정 기준 |
|------|----------|----------|
| 누적 정산액 | 계약 시작~현재까지 정산 완료된 전체 금액 | status = PROCESSED인 모든 인보이스 합산 |
| 이번달 정산금액 | 이번 달에 실제 지급될 금액 | 전월 예약 확정 건 기준 (예: 3월 화면 → 2월 확정 건) |
| 다음달 정산예정금액 | 다음 달에 지급 예정인 예상 금액 | 당월 예약 건 기준 (예: 3월 화면 → 3월 예약 건) |

- 각 카드에 금액과 함께 기준 설명 문구 노출
  - 예: "이번달 정산금액 = 전월 예약 확정 건 기준"

#### 2. 월별 기간 필터

- 월 단위 선택 가능한 필터 추가 (예: 2026.01 / 2026.02 / 2026.03)
- 필터 변경 시 하단 정산 내역 테이블이 해당 월 기준으로 갱신
- 기본값: 현재 월

#### 3. 정산 상태 라벨

각 정산 건에 상태 배지를 표시:

| 상태 | 설명 | 표시 색상 (예시) |
|------|------|----------------|
| 정산 완료 | 지급 완료된 건 | {TBD - 디자인 확정 필요} |
| 정산 예정 | 정산 요청되어 처리 대기 중인 건 | {TBD} |
| 정산 대기 | 아직 정산 요청 전인 건 | {TBD} |

#### 4. 기준일 안내

- "이번달 정산금액"과 "다음달 정산예정금액" 카드 하단에 산정 기준 안내 문구 노출
- 예: "이번달 정산금액은 전월(2월) 예약 확정 건을 기준으로 산정됩니다"

### 백엔드 요구사항

#### 1. 정산 요약 조회 API

파트너별 정산 요약 정보를 반환하는 쿼리 추가 (또는 기존 쿼리 확장).

**반환 데이터:**

```
{
  cumulativeSettlementAmount: number   // 누적 정산액 (PROCESSED 인보이스 합산)
  currentMonthAmount: number           // 이번달 정산금액
  currentMonthBasis: string            // 이번달 기준월 (예: "2026-02")
  nextMonthExpectedAmount: number      // 다음달 정산예정금액
  nextMonthBasis: string               // 다음달 기준월 (예: "2026-03")
}
```

**산정 로직:**
- 누적 정산액: `affiliate_invoice` 테이블에서 해당 파트너의 `status = PROCESSED`인 건의 `settlementAmount` 합산
- 이번달 정산금액: 현재 월 인보이스의 `settlementAmount` (전월 확정 건 기준으로 이미 생성됨)
- 다음달 정산예정금액: `affiliate_balance` 테이블에서 `expectedSettlementMonth = 다음달`이고 `status = CONFIRMED`인 건의 `amountKrw` 합산

#### 2. 월별 정산 내역 조회 API

기존 `myAffiliateSettlementHistory` 쿼리에 월별 필터 파라미터 추가 (또는 신규 쿼리).

**추가 파라미터:**
- `yearMonth`: string (예: "2026-03") — 선택한 월 기준 필터

**반환 데이터:**
- 해당 월의 정산 건 목록 (상품명, 예약일, 정산금액, 상태 포함)

#### 3. 정산 상태 매핑

기존 `AffiliateInvoiceStatus` enum을 파트너 노출용 상태로 매핑:

| 내부 상태 | 파트너 노출 상태 |
|----------|----------------|
| PROCESSED | 정산 완료 |
| SETTLEMENT_SCHEDULED | 정산 예정 |
| PENDING | 정산 대기 |
| CLOSED | 정산 마감 |
| REJECTED | 정산 반려 |
