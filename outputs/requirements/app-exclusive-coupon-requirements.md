# 앱 전용 쿠폰 — 요구사항 문서

> **Status**: 요구사항 미확정 — 확정 후 수정 필요

## 배경

### 문제 정의

- 앱 사용자 확대를 위해 앱에서만 사용 가능한 전용 쿠폰을 도입하여 앱 설치/전환을 유도
- 기존 온라인 쿠폰 시스템을 최대한 활용하되, 현재 쿠폰에 플랫폼 제한(앱/웹) 개념이 없어 확장 필요

### 가설적 임팩트

DAU 모델 분석 기준, 앱 유저는 모든 리텐션 버킷에서 웹 유저 대비 구매 전환율이 1.9~4.3배 높다.

- 웹→앱 전환 유저의 **1회 방문당 증분 가치: ~6,679원** (CVR × 인당 거래액 가중 평균)
- 전환 유저가 기존 앱 유저의 50% 수준으로만 행동한다고 보수적으로 가정 시, **연간 증분 가치: ~51,732원/인**
- 쿠폰 비용(5,000원)은 연간 증분 가치의 9.7%로, 사용 시에만 발생 (미사용 쿠폰은 비용 아님)

## 구현

### 유저 플로우

1. 유저가 회원가입하면 앱 전용 쿠폰이 자동 발급된다
   - 기존 회원에게는 일괄 발급 스크립트로 발급한다

2. 유저가 `/mypage/coupon`에서 발급받은 쿠폰을 조회한다
   - 앱 환경: 앱 전용 쿠폰이 정상 노출된다
   - 비앱 환경: 앱 전용 쿠폰이 비활성화된 UI로 노출되며, "앱에서만 사용 가능" 안내 문구가 함께 노출된다
   - 비앱 + 모바일 환경: 앱 다운로드를 유도하는 버튼/배너가 함께 노출된다

3. 유저가 `/reserve/payment/{id}`에서 쿠폰을 적용하려 한다
   - 앱 환경: 앱 전용 쿠폰을 선택하여 정상 적용할 수 있다
   - 비앱 환경:
     - 앱 전용 쿠폰이 비활성화된 UI로 노출된다
     - 해당 상품에 적용 가능한 앱 전용 쿠폰이 있으면, "앱에서만 사용 가능한 쿠폰을 사용하면 {금액}만큼 더 저렴하게 이용할 수 있어요" 안내 문구가 노출된다

### 프론트엔드 요구사항

**어드민 — 쿠폰 관리**
- 쿠폰 생성/수정 폼에 "앱 전용" 토글 추가
- 쿠폰 목록에서 앱 전용 여부 확인 가능 (뱃지 또는 컬럼)

**`/mypage/coupon` — 내 쿠폰 페이지**
- `useWebviewEnvironment()` 훅으로 앱/비앱 환경 분기
- 앱 환경: 기존과 동일하게 노출
- 비앱 환경: 앱 전용 쿠폰을 비활성화(dimmed) 상태로 노출 + "앱에서만 사용 가능" 문구 표시
- 비앱 + 모바일 환경: 앱 다운로드 유도 버튼/배너 추가 노출 (앱스토어/플레이스토어 이동)
- 기존 "Unavailable" 탭 워딩을 "Expired" 등 실제 상태를 반영하는 워딩으로 변경 (i18n 키값 수정)

**`/reserve/payment/{id}` — 결제 페이지**
- 앱 환경: 앱 전용 쿠폰 정상 선택/적용 가능 (기존 로직과 동일)
- 비앱 환경:
  - 앱 전용 쿠폰 비활성화 표시
  - 적용 가능한 앱 전용 쿠폰이 있으면, 기존 `getDiscountedAmount()` 로직으로 할인 금액을 계산하여 안내 문구 노출

### 백엔드 요구사항

**`isAppOnly` 필드 추가**
- `OnlineCoupon` 엔티티에 `isAppOnly: boolean` 컬럼 추가 (default: false)
- DB 마이그레이션 생성
- GraphQL 스키마에 `isAppOnly` 필드 노출
- 쿠폰 생성/수정 mutation에서 `isAppOnly` 설정 가능

**쿠폰 사용 검증 — 앱 환경 체크**
- `verifyCanUse()` 로직에 앱 환경 검증 추가
- `isAppOnly=true`인 쿠폰은 비앱 환경에서 적용 시 에러 반환
- 앱 여부를 백엔드에 전달하는 방법 필요 (쿠폰 적용 mutation 파라미터 또는 요청 헤더)

**회원가입 시 자동 발급**
- 회원가입 이벤트 핸들러에서 하드코딩된 쿠폰 ID로 발급
- 쿠폰 ID는 배포 전에 확인하여 상수 파일에 반영
- 발급 실패 시 회원가입에 영향 없도록 비동기 처리

**기존 회원 일괄 발급**
- 전체 회원을 페이지네이션(100건 단위)하며 쿠폰 발급하는 배치 스크립트
- 중복 발급 방지 (이미 발급된 회원 스킵)
- 발급 진행 상황 로깅 (처리 건수, 에러 건수)
- 일회성 CLI 커맨드 또는 마이그레이션 스크립트

## 배포 순서

1. **(배포 전)** 어드민에서 쿠폰 생성 (이 시점엔 `isAppOnly` 필드가 없으므로 일반 쿠폰으로 생성) → 쿠폰 ID 확인
2. **(배포 전)** 해당 쿠폰 ID를 코드 상수 파일에 하드코딩 + 회원가입 자동 발급 로직 구현
3. **배포** (`isAppOnly` 필드/마이그레이션 + 검증 로직 + 자동 발급 로직 + 프론트 UI 포함)
4. **(배포 후)** 어드민에서 해당 쿠폰의 `isAppOnly` 토글을 true로 변경
5. **(배포 후)** 기존 회원 대상 일괄 발급 스크립트 실행

> **4 → 5 순서 중요**: `isAppOnly`를 먼저 켜야 발급된 쿠폰이 바로 앱 전용으로 동작한다
