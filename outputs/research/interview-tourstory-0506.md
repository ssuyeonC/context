| 4-7 *(시연 요청)* | 만약 지금 한 상품의 가격을 바꾸신다고 가정하면 어디서 어떻게 하실 것 같으세요? 화면 같이 보면서요? | **막히는 동선이 곧 마찰점** |  |

## 영역 5. 예약 처리 경험 (5~7분, Type D 분기)

> 투어스토리는 **미사용**에 가까우므로 5-B 분기 위주.

| # | 질문 | 의도 | 응답 |
| --- | --- | --- | --- |
| 5-B-1 | 예약은 평소에 어떻게 확정되고 있나요? 운영팀이 대신 처리한다는 사실을 알고 계세요? | **운영팀 의존 인지** |  |
| 5-B-2 | 자체 예약 시스템에서 운영하실 텐데, 크리에이트립으로 들어온 예약은 별도로 입력하시는 건가요? 어떻게 합쳐 보세요? | **이중 운영 패턴** |  |
| 5-B-3 | 만약 직접 예약을 어드민에서 확인하고 승인할 수 있다면, 도움이 될까요 부담일까요? | 의향 |  |
| 5-B-4 | 직접 예약을 처리하지 않는 가장 큰 이유가 무엇이라고 생각하세요? | 핵심 미사용 이유 |  |
| 5-B-5 *(추가, 다스팟 특화)* | 50개+ 상품의 예약을 매일 다 들여다보는 게 현실적이신가요? 어떻게 하면 가능할까요? (요약 대시보드? 알림 필터?) | **스케일 솔루션 가설** |  |
| 5-B-6 *(추가)* | 다른 OTA(클룩, KKday, 트립닷컴, 마이리얼트립)도 같이 쓰세요? 어드민 사용 빈도 비교하시면 어떠세요? | **OTA 어드민 비교** |  |
| 5-B-7 *(데이터 정정, 알림톡)* | 투어스토리는 카카오 알림톡 채널이 OFF 상태라 알림톡으로 예약 알림을 받지 않으세요. 그게 의도하신 건가요, 모르고 계셨던 건가요? 만약 켜진다면 57개 스팟 알림이 한꺼번에 와도 괜찮으세요? | **알림톡 미수신 인지 + 다스팟 알림 폭주 우려** |  |

## 인터뷰 후 정리 템플릿

| 차원 | 항목 | 메모 |
| --- | --- | --- |
| 행동 | 누가 / 언제 / 어디서 / 어떻게 (구체 동작) | |
| 워크어라운드 | 어드민에 없어서 카톡·전화로 처리하는 것 | |
| 시간/사람 | 작업당 분, 사람 수, 처리 주기 | |

특별히 추출해야 할 것:
- **다스팟 운영자가 어드민을 쓰지 않는 진짜 이유** (Type D 일반화)
- **일정 관리 셀프 vs 콘텐츠 수정 위임의 분기점** (영역별 채널 선택)
- **시즌 사이클 콘텐츠 변경 부담** (벌크 기능 가설)
- **자체 시스템 ↔ 크리에이트립 이중 운영 구조** (싱크 자동화 욕구)
- **OTA 어드민 비교** (경쟁사 어드민에서 잘되는 것 발견)
