## 핵심 발견 (Top Findings)

### 1. 상품/옵션별 스케줄 개별 관리 니즈 (★강한 신규 기능 후보)

현재 모델로는 다른 플랫폼(클룩 등) 예약과 시간이 겹쳐도 **크리에이트립 어드민에서 상품별로 닫을 수 없어 통째로 처리**하는 우회 운영을 하고 있음.

- 운영 패턴 A: 한 상품의 시간이 점유되면 전체 일정을 닫음
- 운영 패턴 B: 일단 열어두고 크리에이트립 예약이 들어오면 카톡으로 시간 조율 유도
- 본인이 **인지하지 못한 채** 통째로 닫고 있을 가능성 — 상품별 스케줄 관리 UX가 발견되지 않거나 직관적이지 않음
- 부수적으로 "상품별 예약 현황을 한눈에 보고 토글하는" UI 니즈도 언급. **레퍼런스: 네이버 플레이스**

### 2. 사진 직접 수정에 대한 강한 니즈 (★어드민 흡수 1순위)

- "사진 직접 수정 의향 **높음**" (글 수정 니즈는 낮음 — 사진과 분리해서 인식)
- 직접 번역도 가능하므로 직접 수정도 되면 좋겠다는 발화
- 다만 "잘못 수정될까 부담은 조금 있음" — **검수·프리뷰·되돌리기 같은 안전망**이 함께 있어야 채택 가능

### 3. 스팟 수정 요청 ↔ 어드민 사진 불일치 (버그성 마찰)

- 어드민의 **스팟 수정 요청 메뉴에 노출되는 사진**과 **어드민에서 관리 중인 사진**이 일치하지 않음
- 이게 사진 수정 시도에서 시행착오·혼란의 원인이 된 정황
- → **빠른 수정 필요** (단순 동기화 이슈일 가능성)

### 4. 콘텐츠 수정 = 100% 카톡 우회 확인 (가설 H2 지지)

- 데이터로 보였던 "콘텐츠 수정 0건" 패턴의 실제 운영 모습:
  - "카톡 요청" — 가격/사진/주의사항 모두 카톡으로 운영팀(창우)에게 전달
  - 어드민에 스팟 정보 수정 요청 메뉴가 있다는 건 **알지만 잘 안 씀**
  - 최근 사례(크리스마스 이벤트 수정)도 본인이 직접 어드민에 들어가지 않고 매니저에게 전달
- 즉 **인지 부재가 아니라 의식적 회피** — 카톡이 더 확실하거나 빠르다고 체감

### 5. 변경 반영 지연에 대한 우회 운영

- 어드민 요청 후 반영까지 "꽤 걸린다"고 체감
- 반영 지연 동안 잘못된 정보로 예약이 들어오면 → **개별 안내** 혹은 **크리에이트립 손님에 한해 해당 정책을 적용하지 않는 방식**으로 운영 우회
- 매장의 정상 운영을 위해 시스템이 아닌 사람으로 메우는 패턴 확인

### 6. 카톡 우위의 이유 = 잦은 변경에서 카톡이 더 빠름

- 사진 스튜디오 특성상 예약 변경/취소 요청이 잦음
- 다른 플랫폼 예약과 시간 충돌이 잦아 손님과 직접 시간 조율이 필요
- 이런 조율은 카카오 알림톡보다 **카톡 대화가 더 효율적** → **예약 변경 요청 UX 자체에 대한 개선 시그널**
