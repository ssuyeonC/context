# 순시키 파트너 인터뷰 질문지 (2026-04-30)

## 인터뷰 컨텍스트

순시키는 데이터 기준 어드민 처리율 96~97%로 **Type A(어드민 주력 대형 파트너)** 모범 사례. 단, 일정 조율 시 "특별한 불편 없다"고 답했으나 내부 직원은 스팟 수정 영역에서 마찰을 인지하고 있음. 이번 인터뷰는 다음 두 갈래를 동시에 푼다.

1. **97% 비결 추출** — 다른 파트너 온보딩에 적용할 행동 패턴
2. **숨은 마찰 발굴** — 이미 적응해 의식하지 못하는 스팟 수정 워크어라운드

## 진행 원칙

- **추상 질문 금지** — "불편하세요?" 대신 "최근 ○○하셨을 때 어떻게 하셨어요?"
- **마지막 사례** — 가장 최근 일어난 구체 케이스를 끝까지 따라간다
- **현장 시연** — 가능하면 어드민 화면을 같이 보면서 "지금 한 번 보여주세요"
- **우회 경로 추적** — 카톡/전화로 운영팀에 부탁하시는 것 = 어드민에 없는 기능
- **시간/사람 측정** — 누가, 몇 분/며칠 걸리는지 숫자로 받아 적기

## 인터뷰 후 정리 템플릿

응답을 다음 3가지 차원으로 정리해두면 다른 파트너 인터뷰와 비교 가능.

| 차원 | 항목 | 메모 |
| --- | --- | --- |
| 행동 | 누가 / 언제 / 어디서 / 어떻게 (구체 동작) | |
| 워크어라운드 | 어드민에 없어서 카톡·전화로 처리하는 것 | |
| 시간/사람 | 작업당 분, 사람 수, 처리 주기 | |

특별히 추출해야 할 것:
- **97% 비결** (Type D 파트너 온보딩 가설 인풋)
- **스팟 수정 마찰의 구체 사례** (운영팀 가설 검증 또는 반증)
- **어드민에 없지만 필요한 기능 후보** (우회 경로에서 발견)
