# 아리한복 파트너 인터뷰 질문지 (2026-05-07 예정)

## 본 인터뷰의 목적

> **파트너의 어드민 사용성 개선을 통한 크리에이트립 관리자의 리소스 효율화.**
> 아리한복은 자동승인 ON + 카톡 매니저 의존으로 어드민에 거의 들어가지 않아도 운영이 굴러가는 구조다. 매니저(성민님)와의 사전 확인을 통해 **수정 요청 0건의 원인이 "매니저 지시에 의한 의식적 회피"**임이 밝혀졌고(톤앤매너·번역 이슈), **알림톡 OFF는 매니저 공지 미완료** 때문임이 확인됐다. 인터뷰의 초점은 가설 검증에서 **"매니저 우회의 디테일과 톤앤매너 이슈의 실체, 카톡방 콜라보 운영의 구조, 파트너의 다른 플랫폼 운영 경험과의 비교"**로 이동한다.

## 인터뷰 컨텍스트

아리한복은 **자동승인 의존형 다지점·콜라보 파트너**(성민 매니저 담당). 데이터 기준 어드민 직접 처리율 **약 5%(7건/138건)**, 단 **자동 승인(`is_auto_confirm`) 옵션을 켜둠**으로써 142건이 자동 확정 — 사실상 "어드민에 들어가지 않아도 되는 운영 구조"를 채택한 케이스. 게다가 어드민의 스팟 수정 요청 기능은 **5개 스팟 합산 0건** — 매니저가 톤앤매너·번역 이슈 때문에 직접 처리하도록 안내한 결과.

### 재정의된 검증 가설

1. **톤앤매너·번역 이슈의 실체** — 매니저가 우회시킨 이유. 사장님이 직접 본 어떤 콘텐츠가 문제였는지, 어드민이 어떻게 바뀌면 직접 쓸 수 있을지
2. **카톡방 콜라보 운영 구조** — 메이크업 파트너와의 카톡방이 누구로 구성되는지, 일정 충돌·취소 시 어떻게 흐르는지, 어드민이 이 구조를 흡수 가능한지
3. **자동승인+알림 OFF 결합의 실제 사고** — 콜라보 3스팟은 파트너 메일도 없고 알림톡도 OFF. 예약이 있어도 모르고 지나친 경험이 실제로 있었는지
4. **다른 플랫폼과의 비교 우위/열위** — 다수 플랫폼 운영 중. 다른 곳은 어떻게 정보 수정·번역을 제공하는지, 크리에이트립의 강·약점 인식
5. **알림톡 공지가 닿은 후의 채택 의향** — 매니저 공지가 진행되면 실제로 쓸 것인가, 수신자(대표 폰 / 데스크) 설계

## 진행 원칙

- **추상 질문 금지** — "불편하세요?" 대신 "최근 ○○하셨을 때 어떻게 하셨어요?"
- **시연 위주** — 어드민 화면을 직접 띄워 "이 화면 보신 적 있으세요?"부터
- **매니저 우회를 부정 없이 따라가기** — 매니저 카톡이 편하다 하면 "왜 편한지" 더 묻기. 어드민으로 옮기라는 압박 금물
- **톤앤매너 이슈의 실체 캐기** — 매니저가 우회시킨 이유가 톤앤매너·번역인 게 사전 확인됨. 사장님이 직접 본 어떤 콘텐츠가 마음에 안 드셨는지, 어떻게 다르길 바라시는지 구체화
- **카톡방 구조 따라가기** — 메이크업 업체와의 카톡방이 누구로 구성되는지, 일정 충돌 시 어떻게 흐르는지 끝까지 추적
- **사전 확인 정보는 검증 + 디테일 캐기 모드로** — 매니저로부터 들은 내용을 "정말 그러신가요" 확인하는 모드보다, **"그렇다고 들었습니다. 그 안에서 구체적으로는 어떠세요?"**로 한 겹 깊이 들어가기
- **시간/사람 측정** — 누가, 몇 분/며칠 걸리는지 숫자로
- **다른 플랫폼 비교 적극 활용** — 다수 플랫폼 운영 중. 다른 곳은 어떻게 하는지로 크리에이트립의 강·약점 도출

| 3-6-a *(알림톡 채택 의향)* | 매니저님께서 알림톡 안내를 거래처에 곧 드릴 예정이라고 들었어요. 만약 카카오 알림톡으로 예약 알림이 오게 된다면, 쓰실 의향이 있으세요? 어떤 형태가 좋을 것 같으세요? (대표 폰 / 매장 데스크 / 메이크업 업체 카톡방 연동) | **알림톡 채택 의향 + 수신 설계** |  |
| 3-7 *(시연)* | (어드민 화면 같이 띄워보며) 이 화면 어떠세요? 처음 들어오면 어디 뭐 있는지 한눈에 들어오세요? 헷갈리는 부분이 있다면? | **첫인상 사용성 진단** |  |

## 영역 4. 수정 요청·운영 변경 흐름 (10~15분) — **핵심 영역**

> 사전 확인 결과 **매니저가 톤앤매너·번역 이슈로 어드민 기능 사용을 피해달라 안내** → 5개 스팟 0건. 이 영역에서 톤앤매너·번역 이슈의 실체와 사장님이 직접 어드민을 쓸 수 있는 조건을 발굴.

### 4-A. 매니저 우회 디테일 & 톤앤매너 이슈 (필수)

| # | 질문 | 의도 | 응답 |
| --- | --- | --- | --- |
| 4-A-1 | 최근 3개월 안에 한복 가격, 운영시간, 메이크업 옵션, 사진 같은 걸 바꾼 적 있으세요? 어떤 거였어요? | **마지막 사례 확보** |  |
| 4-A-2 | 그 케이스, 누가 결정했고 실제로 어디로 알리셨어요? (성민님 카톡 / 어드민 / 메이크업 업체 카톡방) | 사내·콜라보 분담 |  |
| 4-A-3 | 요청부터 실제 사이트에 반영되기까지 며칠 걸렸어요? | 체감 처리 시간 |  |
| 4-A-4 | 그동안 유저 노출은 어떻게 됐어요? 잘못된 정보로 예약 들어온 적은요? | 비즈니스 임팩트 |  |
| 4-A-5 *(톤앤매너 핵심 — 직접 화법)* | 매니저(성민님)께서 "어드민에 직접 정보 수정하시지 마시고 저한테 말씀하세요"라고 안내드렸다고 들었어요. 그 이유가 톤앤매너랑 번역 때문이라고 하시던데, 실제로 사장님께서 어드민이나 사이트에서 마음에 안 드는 콘텐츠를 보신 경험이 있으세요? 어떤 거였는지 구체적으로요. | **톤앤매너 이슈의 실체** |  |
| 4-A-6 *(톤앤매너 — 번역)* | 한복 설명이나 메이크업 옵션 같은 게 외국어로 번역돼서 사이트에 노출되잖아요. 그 번역을 직접 보신 적 있으세요? 어떤 점이 마음에 안 드셨어요? | **번역 이슈의 실체** |  |
| 4-A-7 *(톤앤매너 — 가설)* | 만약 어드민에서 사장님이 직접 콘텐츠를 수정하시면 번역이 자동으로 돼서 나가게 돼 있어요. 그 자동 번역 결과를 사장님이 검토·수정하실 수 있다면, 직접 어드민에서 처리해보실 의향이 있으세요? | **어드민 채택 조건 발굴** |  |
| 4-A-8 *(다른 플랫폼 비교)* | 다른 플랫폼에도 들어가 계시잖아요. 거기서는 정보 수정이나 번역을 어떻게 처리하세요? 거기는 직접 하시는 편이세요, 거기도 담당자를 통하시는 편이세요? | **비교 우위/열위 단서** |  |
| 4-A-9 | 가격을 바꿔야 하거나 새 한복을 추가해야 할 때, 매니저께 카톡을 보내시면 그 뒤 진행 상황은 어떻게 아세요? 단계별로 알려주시나요, 본인이 사이트 가서 확인하세요? | **카톡 우회 진행 가시성** |  |

### 4-B. 일반 흐름 & 다지점 일괄 욕구

| 4-2 | 그 시간 동안 답답하셨던 적이 있으세요? 그럴 땐 어떻게 하셨어요? | 카톡 우회의 트리거 |  |
| 4-3 | 한복·콜라보 상세 페이지(사진, 한복 종류, 메이크업 옵션) 내용을 직접 수정하고 싶으셨던 적이 있으세요? 어떤 부분을요? | **콘텐츠 직접 수정 의향** |  |

## 영역 5. 예약 처리 경험 (5~7분)

> 자동승인 ON + 카톡방 운영 방식. 핵심은 "예약 인지 채널 추적 + 자동승인 약점 검증 + 다른 플랫폼 비교".

| # | 질문 | 의도 | 응답 |
| --- | --- | --- | --- |
| 5-1 *(예약 인지 채널)* | 한복 본점·촬영상품은 자동확정+메일, 콜라보 3건은 자동확정+카톡방 운영이라고 들었어요. 그러면 일정 충돌이 나거나 당일 예약이 들어오면 누가 가장 먼저 알아차리세요? | **예약 인지 채널 추적** |  |
| 5-2 *(자동승인 약점)* | 자동확정된 예약이 알고 보니 처리 못 하는 일정이었던 적이 있으세요? 그땐 어떻게 하셨어요? | **자동승인 약점 노출** |  |
| 5-3 *(콜라보 알림 누락 사고)* | 콜라보 카톡방으로 예약이 흘러오는데, 카톡방에서 놓쳐서 메이크업 업체랑 일정이 어긋났던 적은요? 그땐 누가 캐치하셨어요? | **카톡방 누락 사고 검증** |  |
| 5-4 | 예약 변경 요청(시간/인원/한복 종류)이 들어오면 누가 어디서 처리하나요? | 마찰점 |  |
| 5-5 | 노쇼는 어떻게 처리하세요? 어드민에서 노쇼 처리해보신 적 있으세요? | 노쇼 흐름 |  |
| 5-6 *(다른 플랫폼 비교)* | 다른 플랫폼들도 같이 운영 중이시잖아요. 거기와 비교해서 크리에이트립이 어떤 점이 더 편하시고, 어떤 점이 더 답답하세요? | **비교 프레임 — 강·약점 도출** |  |

## 영역 6. 마무리 (3~5분)

| # | 질문 | 의도 | 응답 |
| --- | --- | --- | --- |
| 6-1 | 마지막으로, 크리에이트립과 일하실 때 "이것만 좀 바뀌면 좋겠다" 하는 것이 있다면? | 의식적 우선순위 |  |
| 6-2 | 오늘 얘기 안 한 것 중에 사실 좀 거슬리는 거 하나만 꼽으신다면 뭐일까요? | **의식 안 한 마찰 노출** |  |
| 6-3 *(마법 질문)* | 내일부터 마법으로 어드민 한 가지가 바뀐다면 뭐가 바뀌면 좋겠어요? | 무의식 욕구 |  |
| 6-4 *(용어 점검)* | 어드민에 "승인 대기", "확정", "정산 예정", "사용완료" 같은 단어들이 나오는데, 한눈에 무슨 뜻인지 들어오세요? 헷갈리는 단어 있으세요? | 용어 직관성 |  |

## 인터뷰 후 정리 템플릿

| 차원 | 항목 | 메모 |
| --- | --- | --- |
| 행동 | 누가 / 언제 / 어디서 / 어떻게 (구체 동작) | |
| 워크어라운드 | 어드민에 없어서 카톡·전화로 처리하는 것 | |
| 시간/사람 | 작업당 분, 사람 수, 처리 주기 | |
| 사용성 반응 | 화면·용어에 대한 첫인상·즉각 반응 | |

특별히 추출해야 할 것:
- **톤앤매너·번역 이슈의 실체** (사장님이 어떤 콘텐츠를 어떻게 보고 어떻게 느끼셨는지 구체 사례)
- **어드민 채택 조건** (어떤 기능·UX가 보장되면 사장님이 직접 쓸 의향이 있는지)
- **카톡방 콜라보 운영의 구조와 한계** (멤버십, 흐름, 누락 사고 사례, 어드민 흡수 가능성)
- **자동승인 ON + 카톡방 운영 결합의 실제 사고 사례** (당일 충돌·누락·취소 발생 시 누가 어떻게 회복하는지)
- **다른 플랫폼과의 비교 우위/열위** (정보 수정·번역·알림·정산 관점)
- **알림톡 채택 의향과 수신자 설계** (대표 폰 vs 매장 데스크 vs 카톡방 연동)
- **다지점·콜라보 일괄 운영 욕구** (벌크 기능 가설)
- **첫인상 사용성·용어 직관성** (시연 반응)
- **어드민에 없지만 필요한 기능 후보** (우회 경로에서 발견)
