# 홈 화면 아이콘 진입점 개선 기획

## 배경

- 현재 메인 홈과 스팟 홈에 10개의 카테고리 진입점이 노출 중
- 최대 15개까지 확대 필요
- '더보기' 등의 별도 버튼은 지양

### 대상 카테고리 (15개)

피부과, 메디컬 클리닉, 헤어, K-뷰티, 사진&스튜디오, 미식, 액티비티&체험, 명소티켓, K-POP&공연, 일일 투어, 할인 쿠폰, 웰니스, 여행 에센셜, 장기체류&배움, 약국

---

## 핵심 전제: 한국 여행 탐색은 발견형 탐색에 가깝다

### 1. 여행 상품 탐색은 목적형보다 발견형에 가깝다

- 구체적인 특정 항목에 대한 니즈가 비교적 약하다
- 여행지는 기본적으로 익숙하지 않은 곳이므로, 무엇이 있는지 모른다
- SIM, 교통편 등의 항목에 대해서는 목적형으로 작용한다
- 여행에 필수적인 요소는 확정적으로 정해두지만 그 외의 공백에 대해서는 발견적으로 탐색한다
  - 여행지에서의 매일, 24시간을 채우는 컨텐츠는 여행 계획 수립 시 모두 목적 하에서 수립된 것은 아니다
  - 관광학에서는 이를 **"Planned Serendipity"**(계획된 세렌디피티)로 개념화 — 여행자가 계획 과정에서 의도적으로 발견의 여지를 남겨두는 행동 (Mieli, 2023)
  - 스마트폰 상시 연결성이 "현장에서 충분히 찾을 수 있다"는 기대를 형성하여, 사전 확정을 줄이고 유연성을 높이는 방향으로 행동이 변화
- 특히 한국 여행의 경우, 전체 방한 관광객의 **약 45%가 첫 방문** → 한국에서 뭘 할 수 있는지 자체를 모름 (한국관광공사)
- 한국 여행 관련 최대 커뮤니티(r/koreatravel, 179K members) 내 대부분의 질문이 '뭘 해야 할지 모르겠다'는 열린 질문에 가까움

### 2. 메인 화면의 진입점이 발견형 탐색에 더 적합하려면?

- 진입점 자체가 유저의 호기심을 불러일으킬 수 있어야 함
- 기계적 분류(탭/그룹) 대신 유저의 흥미를 일으킬 수 있는 요소 & 전략적으로 노출해야 하는 요소 우선 배치
- **진입점이 숨겨지지 않아야 함** — 탭/접힘 뒤에 배치되면 발견 자체가 불가능
- **구체성이 호기심을 만든다** — "뷰티"보다 "피부과"가 더 강한 반응을 유발 ("외국인이 한국 피부과를 갈 수 있어?"라는 의문 → 클릭)

### 근거 자료

| 출처 | 핵심 내용 |
|---|---|
| Vogt & Fesenmaier (1998) | 여행자의 5가지 정보 니즈 중 4개(hedonic, innovation, aesthetic, sign)가 발견형. 실용적 정보 탐색은 일부에 불과 |
| Janiszewski (1998) | Goal-directed(목적형) vs Experiential(발견형) 탐색 구분. 여행은 hedonic consumption으로 experiential 모드에 해당 |
| Mieli (2023) | "Planned Serendipity" — 관광객의 정보 행동은 계획과 자발성의 이분법이 아닌 공존. 유연한 계획이 핵심 테마 |
| 한국관광공사 (2024) | 방한 관광객 재방문율 54.7% → 첫방문 약 45%. 62%가 여가·관광 목적, 60%가 미식 목적 |
| r/koreatravel (179K) | 가장 흔한 질문: "first time, what should I do?", "itinerary check", "hidden gems" — 대부분 발견형 열린 질문 |

### 배민 스타일 탭 그룹핑을 기각한 이유

배달의민족의 탭 모델(음식배달/픽업/장보기쇼핑 등)은 **목적형 탐색에 최적화된 구조**다. 유저가 이미 자신의 인텐트를 알고 있기 때문에 탭이 효율적 필터로 작동한다. 발견형 탐색에 탭을 적용하면:

- 유저가 자기 인텐트를 미리 분류해야 하는 인지 부담 발생
- 탭 뒤에 숨은 카테고리는 발견 자체가 되지 않음
- "이런 것도 있었어?"라는 세렌디피티가 차단됨

## 디자인 안 비교

### 1안: 수평 스크롤 그리드 (Horizontal Scroll Grid)

2행 x 5열 아이콘 그리드를 수평 스크롤하는 방식. 오른쪽 peek 힌트로 스크롤 가능성을 암시.

```
[icon] [icon] [icon] [icon] [icon] [ico>
[icon] [icon] [icon] [icon] [icon] [ico>
              dot indicators
```

**채움 순서**: 세로(column-first) — 1,3,5... / 2,4,6...

**스크롤 방식 옵션**:
- 자유 스크롤: 아이콘이 중간에서 잘린 채 멈출 수 있음
- 컬럼 단위 snap: 자유 스크롤과 유사한 느낌이면서 정지 상태가 깔끔
- **오버래핑 페이지 snap (채택 검토)**: 페이지 간 2~3열 겹침으로 2페이지 빈 공간 해소

오버래핑 snap 시 2열 오버랩이 최적:
```
Page 1:  col 1-5 (items 1-10)
Page 2:  col 4-8 (items 7-15, 빈 칸 1개뿐)
```

#### 장점

- 세로 공간 최소 (현재와 동일한 2행 유지), first fold 하단 콘텐츠가 밀리지 않음
- 구현·운영 단순 (아이콘 + 라벨만 관리, 별도 이미지 에셋 불필요)
- 스크롤 구조 단순 (수평 스크롤 1개, 조작 혼동 없음)
- 확장 용이 (카테고리 16~20개로 늘어나도 페이지만 추가)

#### 단점

- Discovery 유도력 약함 (아이콘 + 텍스트 라벨만으로는 감정적 반응을 일으키기 어려움)
- 숨겨진 아이콘 존재 (2페이지 아이콘은 스와이프하지 않으면 발견 불가)
- 시각적 차별화 어려움 (15개 아이콘이 모두 동일한 크기·형태)

---

### 2안: Hero 카드 + 아이콘 그리드 (Hero Cards + Grid)

상단에 사진 기반 Hero 카드 행, 하단에 아이콘 그리드를 배치하는 방식.

```
[══K-Beauty══] [══Hanbok══] [══Hair S>   ← Hero 카드 (사진)
[icon] [icon] [icon] [icon] [icon] [ico>
[icon] [icon] [icon] [icon] [icon] [ico>
              dot indicators
```

#### 장점

- Discovery 유도력 강함 (사진 기반 카드가 감정적·심미적 반응을 직접 유발)
- 시각적 위계 명확 (Hero → 아이콘 → HOT 필로 정보 구조가 계층적)
- 전략적 노출 가능 (Hero에 시즌별·전략적 카테고리 배치 가능)
- 구체성이 호기심을 만든다 (사진으로 구체적 이미지를 형성, 아이콘만으로는 불가능한 탐색 동기 부여)

#### 단점

- 세로 공간 증가 (Hero 카드 행 ~80px만큼 Travel 섹션이 길어짐)
- 이중 수평 스크롤 (Hero 카드 + 아이콘 그리드 모두 수평 스크롤 → 터치 타겟 혼동·인지 부하 가능)
- 운영 비용 (Hero 카드 이미지 큐레이션·시즌 교체 등 지속적 관리 필요)
- 중복 처리 설계 필요 (Hero 카테고리와 아이콘 그리드 간 관계 정의 필요)

### 비교 요약

| 기준 | 1안 (스크롤 그리드) | 2안 (Hero + 그리드) |
|---|---|---|
| Discovery 유도력 | 보통 | **강함** |
| 세로 공간 효율 | **최소** | 증가 (~80px) |
| 조작 복잡도 | **단순** (스크롤 1개) | 주의 필요 (스크롤 2개) |
| 운영 비용 | **낮음** | 중간 |
| 확장성 | **높음** | 중간 |
| 전략적 카테고리 강조 | 어려움 | **가능** |
| 비주얼 임팩트 | 약함 | **강함** |

**핵심 트레이드오프**: 1안은 **효율·단순성**, 2안은 **발견력·임팩트**. 발견형 탐색을 우선시하면 2안이 유리하고, 공간 효율과 운영 부담을 우선시하면 1안이 유리하다.

### 1페이지 배치 전략 (공통)

발견형 탐색에서 1페이지에 노출되는 아이콘의 선별이 전환율을 좌우한다.

| 원칙 | 설명 | 예시 |
|---|---|---|
| 의외성 우선 | "여행앱에 이것도?" 반응 유발 | 피부과, 약국, 웰니스 |
| 킬러 카테고리 | 클릭률/매출 기여도 높은 항목 | K-뷰티, 명소티켓 |
| 목적형은 뒤로 | 이미 알고 찾아오는 항목은 2페이지도 OK | SIM/교통 등 여행 에센셜 |
